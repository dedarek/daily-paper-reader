## 摘要
本文对面向战场物联网（IoBT）任务控制的策略感知大语言模型检索增强生成（PA-LLM-RAG）框架进行了对抗性安全研究。我们提出了查询无关的语义检索投毒攻击，这是一种新型攻击方式，通过将语义构造的规则注入IoBT知识库，在所有操作员查询类型中实现高检索排名，且无需知晓运行时提示。该攻击通过单条注入规则（1.6%投毒率）实现了85%的LLM上下文污染，并在7.7%投毒率时饱和，表明即使最小的知识库妥协也足以破坏任务决策。为应对这一威胁，我们提出了CLD-KB（知识库网络分层防御），一种双检测器异常检测框架，结合一类支持向量机边界检测与新颖的基于成员的类别扩散分析，利用IoBT策略的三类别分类法在恶意规则到达决策LLM前识别它们。CLD-KB在投毒检测和知识保留方面显著优于五种基线方法（DBSCAN、LOF、K-Means、孤立森林和一类SVM），每次任务仅增加7毫秒计算开销，从而成为LLM驱动的IoBT任务系统中有效且可部署在边缘的防御方案。

## Abstract
This paper presents an adversarial security study of the Policy-Aware LLM Retrieval-Augmented Generation (PA-LLM-RAG) framework for Internet of Battlefield Things (IoBT) mission control. We propose Query-Agnostic Semantic Retrieval Poisoning, a novel attack that injects semantically crafted rules into the IoBT knowledge base achieving high retrieval ranking across all operator query types without requiring knowledge of runtime prompts. The attack achieves 85% LLM context corruption from a single injected rule (1.6% poisoning rate) and saturates at 7.7% poisoning, demonstrating that even minimal knowledge base compromise is sufficient to corrupt mission decisions. To counter this threat, we propose CLD-KB (Cyber-Layered Defense for Knowledge Base), a dual-detector anomaly detection framework combining One-Class SVM boundary detection with a novel Member-Based Category Spread analysis that exploits the three-category IoBT policy taxonomy to identify poisoned rules before they reach the decision LLM. CLD-KB significantly outperforms five baseline methods including DBSCAN, LOF, K-Means, Isolation Forest, and One-Class SVM in both poisoning detection and knowledge preservation, with only 7ms computational overhead per mission, establishing it as an effective and edge-deployable defense for LLM-driven IoBT mission systems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究背景**：战场物联网（IoBT）中的任务控制需要严格的安全和实时性。传统方法采用策略感知的大语言模型检索增强生成（PA-LLM-RAG）框架，通过结构化的政策知识库（包含工作流、交战规则、能力三类规则）来约束LLM决策，并利用独立Judge LLM进行预执行验证。然而，该框架假设知识库内容完全可信，未考虑知识库投毒攻击。
- **核心问题**：在对抗环境下，攻击者若获得知识库写入权限，可注入精心构造的规则，使这些规则在语义检索中排名靠前，从而污染LLM上下文，导致错误的作战决策。现有RAG防御方法（如TrustRAG、RobustRAG、RAGForensics）适用于开放域文档集合，但无法利用IoBT领域特定的结构化策略分类信息，且计算开销大，不适合边缘部署。
- **整体含义**：本文首次系统研究了面向IoBT任务控制的LLM-RAG框架在知识库投毒下的安全性，提出了一种查询无关的语义检索投毒攻击，并设计了高效的边缘可部署防御机制CLD-KB，填补了该领域的安全空白。

## 2. 论文提出的方法论

### 核心思想
- **攻击（Query-Agnostic Semantic Retrieval Poisoning）**：在未知运行时操作员查询的情况下，通过将多种资产类型和周边词汇打包到单个规则中，使得其语义嵌入接近所有查询类型的质心，从而在所有查询下都能获得高余弦相似度得分，进入LLM上下文。
- **防御（CLD-KB：Cyber-Layered Defense for Knowledge Base）**：利用IoBT政策知识库固有的三分类结构（WF、ROE、CAP），设计双检测器异常检测框架。第一检测器使用One-Class SVM（OC-SVM）学习合法规则分布的边界，检测全局离群点；第二检测器采用新颖的“基于成员的类别扩散（Member-Based Category Spread）”分析，利用合法规则更接近其自身类别成员而远离其他类别成员的特性，检测跨类别语义中毒规则。

### 关键技术细节
- **攻击形式化**：
  - 知识库 \(K = \{r_1,\dots,r_N\}\)，每条规则 \(r_i\) 有归一化嵌入 \(e_i = \phi(r_i) \in \mathbb{R}^{768}\)。对于查询 \(q\)，检索器返回 top-\(k\) 个余弦相似度最高的规则。
  - 攻击者注入 \(M\) 条中毒规则 \(P\)，使得对于任意查询 \(q\sim Q\)，至少有一条中毒规则 \(p_j \in P\) 出现在 top-\(k\) 中。
  - 构造方法：计算训练查询集的语义质心 \(\bar{e}_Q = \frac{1}{|Q|}\sum_{q\in Q}\phi(q)\)，并构造规则嵌入使其与 \(\bar{e}_Q\) 的相似度高于阈值 \(\tau_{\text{ret}}\)。通过对IoBT操作词汇（如gate、drone、UAV、UGV等）进行高频率跨类别组合实现。
  - 投毒率 \(\rho = \frac{M}{N+M}\)，本文评估了从1.6%（1条规则）到25%（20条规则）的范围。

- **CLD-KB防御**：
  - **Detector 1: One-Class SVM**：使用RBF核，设置 \(\nu=0.2, \gamma=0.1\)，在合法规则嵌入上训练。阈值 \(\tau_{\text{svm}}\) 设为训练集决策分数第3百分位数，保证训练集上最多3%假阳性。测试规则若决策分数低于该阈值则被标记。
  - **Detector 2: Member-Based Category Spread**：将合法规则按类别分组 \(E_c = \{\phi(r_i): \text{cat}(r_i)=c\}, c\in\{WF, ROE, CAP\}\)。对候选规则 \(r\)，计算到每一类别的最小余弦距离 \(d_{\min}(r,c)=\min_{e\in E_c}(1-\phi(r)\cdot e)\)，则类别扩散度 \(\text{spread}(r) = \max_c d_{\min}(r,c) - \min_c d_{\min}(r,c)\)。合法规则 \(\text{spread} > 0.20\)，中毒规则 \(\text{spread} < 0.15\)，阈值设为 \(\tau_{sp}=0.17\)。
  - **组合检测**：任一检测器标记即认为中毒。采用过滤-补充机制：检索器返回 top-10 候选，依次检查，收集合法规则直到达到目标 \(k_{\text{target}}=3\)，只将合法规则送入LLM。同时，被标记的规则被永久黑名单清除，实现渐进自修复。

## 3. 实验设计

- **数据集/场景**：

  - 操作员提示：20条IoBT任务提示，涵盖安全巡逻、资源分配、威胁响应、检查站管理、兵力部署、多智能体协调等场景（附录A列出）。

- **Benchmark和对比方法**：
  - 基线方法：DBSCAN、LOF、K-Means、Isolation Forest、One-Class SVM。

  - 攻击有效性评估：无防御时，统计每个查询上下文中平均中毒规则数。

## 6. 论文的主要结论与发现

1. **攻击有效性**：查询无关语义检索投毒攻击在1.6%投毒率（单条规则）下即可达到85%的LLM上下文污染；在7.7%投毒率时饱和，平均每上下文包含2.65条中毒规则；此后增加投毒率效果不再提升。
2. **防御有效性**：CLD-KB在所有投毒率（1.6%~25%）下保持100%上下文完整性，且零假阳性。相比之下，K-Means和Isolation Forest的CIR随投毒率升高从45%降至10%~20%；One-Class SVM虽CIR≈100%但产生一个假阳性；DBSCAN和LOF完全失效（F1=0）。
3. **计算开销**：CLD-KB每次任务仅增加7ms延迟，适合资源受限的IoBT边缘环境。
4. **方法论价值**：利用领域特定策略分类结构进行异常检测是有效且轻量的防御思路，优于通用异常检测方法。

## 7. 优点

- **问题新颖性**：首次针对IoBT任务控制中的LLM-RAG框架提出知识库投毒攻击与防御，填补了该领域安全空白。
- **攻击设计巧妙**：查询无关特性使得攻击者无需运行时查询知识即可实现高污染，具有现实威胁性。

## 8. 不足与局限
