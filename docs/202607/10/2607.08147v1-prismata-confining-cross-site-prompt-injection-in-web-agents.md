## 摘要
自主Web代理有望自动化日常浏览任务，但继承了Web最古老的攻击面之一。跨站脚本攻击已证明，在即使良性的页面上，混合可信与不可信内容也是危险的。代理通过将自然语言解释为指令，重新引入了这一风险，允许第三方和用户生成的内容通过提示注入劫持代理。核心挑战在于，推导任务特定的安全策略需要对页面结构进行推理，而该结构又与攻击者的内容纠缠在一起。我们提出Prismata，一种为Web代理执行上下文最小权限的防御机制，既限制代理能看到什么，也限制它能做什么。Prismata的动态信任推导为页面内容生成权限标签，并具有结构约束保证，这受经典完整性模型启发，限制了任何标签错误，使得标签只能降低权限，且错误标记被限制。Prismata的机械约束通过编辑内容和限制代理能力来强制执行这些标签。重要的是，这些机制不需要开发者注释，因此Prismata支持长尾网站。在最近发布的Web代理攻击（包括自适应变体）中，Prismata显著降低了攻击成功率，同时保持了良性任务的效用。

## Abstract
Autonomous web agents promise to automate everyday browsing tasks, but inherit one of the web's oldest attack surfaces. Cross-Site Scripting proved that mixing trusted and untrusted content is dangerous, even on benign pages. Agents resurface this risk by interpreting natural language as instructions, allowing third-party and user-generated content to hijack the agent via prompt injection. The core challenge is that deriving a task-specific security policy requires reasoning over page structure that is entangled with the attacker's content.   We present Prismata, a defense enforcing contextual least privilege for web agents, constraining both what the agent sees and what it can do. Prismata's dynamic trust derivation produces permission labels for page content, with structural confinement guarantees, inspired by classical integrity models, that bound any labeling errors so that labels can only decrease in privilege and mislabelings are bounded. Prismata's mechanical confinement enforces these labels by redacting content and restricting agent capabilities. Importantly, these mechanisms require no developer annotations, so Prismata supports the long tail of websites. Across recent published web agent attacks, including adaptive variants, Prismata substantially reduces attack success while preserving benign task utility.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：自主Web代理将自然语言视为指令，导致第三方或用户生成内容（如评论、广告）可通过**跨站提示注入（XSP）** 劫持代理，执行恶意操作（如泄露用户数据）。
- **背景**：类似于经典跨站脚本攻击（XSS），但现有XSS防御无法直接迁移，因为提示注入是自然语言而非代码，输入过滤器无法区分指令与数据。模型级防御（如指令层次训练）易被自适应攻击绕过，系统级防御（如CaMeL）需要预定义行动计划，难以处理数据依赖的Web任务。核心挑战在于**Web纠缠问题**：要推导安全策略需先理解页面结构，但页面结构本身与不可信内容混杂。
- **研究动机**：提出一种无需开发者标注、可动态推导最小权限策略的系统级防御，同时限制代理的观测和操作能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **上下文最小权限原则**：基于用户任务，限制代理能看到的页面内容和能执行的操作。
- **动态信任推导 + 机械约束**：利用DOM树关键路径和结构线索，自动为页面元素分配权限标签，然后通过剪切内容和降级操作来强制执行。

### 2.2 关键技术细节
- **系统架构**（图3）：Prismata作为透明防御层，置于Web代理与浏览器之间，截获每个页面的渲染DOM，进行信任推导和约束，再将受限观测传给代理。包含三个组件：
  1. **动作门（Action Gate）**：对每个可交互元素，提取其从DOM根到该元素的**关键路径**，仅基于路径和用户任务，由LLM判断该元素是否在任务范围内。攻击者内容若不在路径上则无法影响判断（Case 1）。
  2. **Biba解析（Biba Parsing）**：递归遍历关键路径，逐层分类内容来源（开发者、用户、托管方、外部），利用结构线索（如标题、aria-label、class名）在暴露不可信内容前识别边界，并遵循**无读下、无写上**策略（Biba完整性模型）。不可信子树被剪枝或限制为只读（Case 2）。
  3. **机械约束（Mechanical Confinement）**：根据标签确定性移除或禁用元素，拒绝超出权限的动作调用。
- **安全分析**：三种情况（表1）：
  - Case 1：注入不在关键路径上 → 动作门标签无暴露。
  - Case 2：注入在路径上但有结构线索 → Biba解析在暴露前限制权限。
  - Case 3：注入在路径上且无结构线索 → 残留风险（约0.1%路径，可降至0.017%）。
- **无需开发者标注**：仅依赖DOM结构和通用HTML约定。
