## 摘要
本文对用于战场物联网（IoBT）任务控制的策略感知大语言模型检索增强生成（PA-LLM-RAG）框架进行了对抗性安全研究。我们提出了一种新的攻击方法——查询无关语义检索投毒，该方法通过向IoBT知识库注入语义精心构造的规则，在所有操作员查询类型中均能获得高检索排名，且无需了解运行时提示。该攻击通过单条注入规则（1.6%的投毒率）实现了85%的LLM上下文污染，并在7.7%的投毒率下达到饱和，表明即使最小的知识库破坏也足以干扰任务决策。为应对这一威胁，我们提出了CLD-KB（知识库网络分层防御），一种双检测器异常检测框架，结合一类支持向量机（One-Class SVM）边界检测与一种新颖的基于成员的类别扩散分析，利用IoBT策略的三类别分类法在投毒规则到达决策LLM之前识别它们。CLD-KB在投毒检测和知识保留方面显著优于五种基线方法（包括DBSCAN、LOF、K-Means、孤立森林和一类SVM），且每次任务仅增加7毫秒的计算开销，为LLM驱动的IoBT任务系统提供了一种有效且可边缘部署的防御方案。

## Abstract
This paper presents an adversarial security study of the Policy-Aware LLM Retrieval-Augmented Generation (PA-LLM-RAG) framework for Internet of Battlefield Things (IoBT) mission control. We propose Query-Agnostic Semantic Retrieval Poisoning, a novel attack that injects semantically crafted rules into the IoBT knowledge base achieving high retrieval ranking across all operator query types without requiring knowledge of runtime prompts. The attack achieves 85% LLM context corruption from a single injected rule (1.6% poisoning rate) and saturates at 7.7% poisoning, demonstrating that even minimal knowledge base compromise is sufficient to corrupt mission decisions. To counter this threat, we propose CLD-KB (Cyber-Layered Defense for Knowledge Base), a dual-detector anomaly detection framework combining One-Class SVM boundary detection with a novel Member-Based Category Spread analysis that exploits the three-category IoBT policy taxonomy to identify poisoned rules before they reach the decision LLM. CLD-KB significantly outperforms five baseline methods including DBSCAN, LOF, K-Means, Isolation Forest, and One-Class SVM in both poisoning detection and knowledge preservation, with only 7ms computational overhead per mission, establishing it as an effective and edge-deployable defense for LLM-driven IoBT mission systems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：战场物联网（IoBT）依赖异构的自主资产（无人机、地面车辆、机器人等）进行协作任务。传统指令控制方法缺乏实时适应性。作者先前提出了PA-LLM-RAG框架，通过在边缘部署检索增强生成（RAG）系统，将LLM决策锚定在结构化操作策略知识库中，并配合Judge LLM进行预执行验证。该框架假设知识库只包含合法规则，但在对抗环境下这一假设不成立。
- **核心问题**：知识库投毒攻击——攻击者向RAG系统的知识库注入精心伪造的文档，使其在语义检索中排名靠前，从而污染LLM的上下文，导致错误的使命决策。该攻击面在IoBT场景中尚未被研究。本文旨在：①表征针对PA‑LLM‑RAG的检索投毒攻击；②提出有效的防御机制。
- **研究意义**：证明即使最小规模的知识库破坏（单条规则，1.6%投毒率）也能导致85%的LLM上下文污染，严重威胁任务安全；同时提出一种轻量级、可边缘部署的防御方案，填补了领域空白。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 攻击方法：查询无关语义检索投毒（Query-Agnostic Semantic Retrieval Poisoning）

- **攻击者能力**：白盒访问知识库（已知嵌入模型`nomic-embed-text`和知识库类别结构WF/ROE/CAP），但不能修改现有规则、拦截查询或访问LLM权重。
- **目标**：使注入的规则在所有可能的操作员查询中都能进入检索结果top‑k。
- **关键思想**：攻击者不需要知道运行时查询，而是针对查询分布的质心构建规则。通过分析IoBT操作词汇，提取高频跨类别术语（如`gate, drone, UAV, UGV, robot, coverage, perimeter, checkpoint, security`），注入规则同时包含所有资产类型和外围词汇，并明确指示集中到某个单一目标门。这种跨类别词汇构造使注入规则的嵌入与所有三个策略类别（WF, ROE, CAP）的中心距离相等。
- **形式化**：设知识库 $K=\{r_1,...,r_N\}$，嵌入 $e_i=\phi(r_i)\in\mathbb{R}^{768}$。查询 $q$ 的检索返回 $C(q)=\text{top-}k_{r_i\in K}\text{sim}(q,r_i)$。攻击者注入 $M$ 条规则 $P=\{p_1,...,p_M\}$，要求 $\max_{p_j\in P}\text{sim}(q,p_j)\ge\min_{r_i\in C(q)}\text{sim}(q,r_i)$，对所有 $q\sim Q$ 成立。攻击者使用查询分布质心 $\bar{e}_Q=\frac{1}{|Q|}\sum_{q\in Q}\phi(q)$ 来构造规则。
- **投毒率控制**：$M = \lfloor \rho\cdot N / (1-\rho)\rfloor$，$\rho=M/(N+M)$。实验评估 $\rho=1.6\%\sim25\%$（单条到20条规则）。

### 2.2 防御方法：CLD-KB（Cyber-Layered Defense for Knowledge Base）

- **设计动机**：攻击构造使投毒规则嵌入位于合法规则分布的中心附近，传统密度/聚类方法难以检测；但合法规则属于单一类别，嵌入靠近同类成员而远离其他类成员（高扩散度），投毒规则由于跨类别词汇嵌入而到各类成员距离均匀（低扩散度）。
- **架构**：双检测器 + 过滤-补齐策略。在检索器返回top‑10候选规则后，对每条规则依次运行两个检测器，任一标记则屏蔽并永久移除；继续从候选池中取干净规则直到凑足3条传给LLM。
- **检测器1：一类SVM（OC-SVM）**
  - 在合法规则嵌入上训练，使用RBF核，$\nu=0.2,\gamma=0.1$，阈值设在训练得分的第3百分位数（确保零假阳性）。
  - 判断：$f(\tilde{e}_r) < \tau_{svm}$ 则标记为异常。
- **检测器2：基于成员的类别扩散（Member-Based Category Spread）**
  - 按类别将合法规则嵌入分组：$E_c=\{\phi(r_i):\text{cat}(r_i)=c\},c\in\{WF,ROE,CAP\}$。
  - 计算候选规则 $r$ 到每个类别的最小余弦距离：$d_{\min}(r,c)=\min_{e\in E_c}(1-\phi(r)\cdot e)$。
  - 扩散度：$\text{spread}(r)=\max_c d_{\min}(r,c)-\min_c d_{\min}(r,c)$。合法规则扩散度 $>0.20$，投毒规则 $<0.15$。阈值 $\tau_{sp}=0.17$。
- **组合判断**：任一检测器标记即判定为投毒：$\hat{y}(r)=\hat{y}_{svm}(r)\lor\hat{y}_{sp}(r)$。
- **自适应黑名单**：被标记的规则永久移除，后续任务中投毒率单调递减。

## 6. 论文的主要结论与发现

1. **攻击有效性**：单条投毒规则（1.6%投毒率）可导致85%的LLM上下文污染；攻击在7.7%投毒率时饱和（平均每条LLM上下文含2.65条投毒规则），之后无边际收益。
2. **基线方法缺陷**：DBSCAN和LOF完全失效（F1=0）；K-Means和孤立森林仅部分检测（F1≈0.684-0.703）；一类SVM召回率100%但有1个假阳性。
3. **CLD-KB优势**：检测率Recall=1.000，Precision=1.000，F1=1.000，零假阳性；在所有投毒率下保持100%上下文完整性；开销仅7ms/任务。
4. **根本原因**：投毒规则嵌入跨类别均匀扩散（扩散度<0.15），而合法规则扩散度>0.20，形成清晰分离间隙。
5. **自适应黑名单效果**：随任务进行，知识库逐渐自我清洗，攻击面递减。
