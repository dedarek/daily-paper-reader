## 摘要
语言模型智能体读取攻击者可写的上下文以解决任务。工具执行需要对受保护接收字段、接收解释的有效载荷以及调用事件进行单独权限检查。上下文到执行完整性（CXI）是针对此场景的执行边界系统。策略标记受保护接收字段，类型化发布将狭窄的验证值从可写上下文传送到特定目的地，不透明数据槽将证据作为数据保留，而确定性门仅在字段权限、精确效果授权和调用权限均绑定到同一动作清单时才允许调用。我们在开放权重字段投影运行、AgentDojo实况片段、代码智能体精确效果基准测试、清单绑定账本故障、提案压力控制以及托管/API兼容性轨迹上评估了CXI。AgentDojo涵盖720个实况片段和1739次LLM调用；代码智能体基准测试涵盖400个仓库片段，具有精确效果授权和租约绑定执行，产生了231个安全任务完成，且未观察到字段、效果或调用逃逸。账目报告解析结果、授权结果和任务质量结果以及准入完整性结果。在评估的接收字段中，仅当字段、效果和调用权限绑定到同一动作清单时，CXI才允许执行。

## Abstract
Language-model agents read attacker-writable context to solve tasks. Tool execution needs a separate authority check for protected sink fields, sink-interpreted payloads, and the invocation event. Context-to-Execution Integrity (CXI) is an execution-boundary system for this setting. Policies mark protected sink fields, typed releases carry narrow validated values from writable context to specific destinations, opaque data slots keep evidence as data, and a deterministic gate admits a call only after field authority, exact-effect authorization, and invocation authority all bind to the same action manifest.   We evaluate CXI on open-weight field-projection runs, AgentDojo live episodes, a code-agent exact-effect benchmark, manifest-bound ledger faults, proposal-pressure controls, and hosted/API compatibility traces. AgentDojo covers 720 live episodes and 1,739 LLM calls; the code-agent benchmark covers 400 repository episodes with exact-effect authorization and lease-bound execution, yielding 231 safe task completions and zero observed field, effect, or invocation escapes. The accounting reports parser outcomes, authorization outcomes, and task-quality outcomes together with the admission-integrity result. Across the evaluated sinks, CXI admits execution only when field, effect, and invocation authority bind to the same action manifest.

# 论文总结：Context-to-Execution Integrity for LLM Agents

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：LLM 智能体从攻击者可写的上下文（如邮件、Web 页面、CI 日志、检索片段）中读取信息并执行工具调用。现有系统缺乏细粒度的权限边界，导致“权限洗白”（authority laundering）——攻击者影响的值（如“approve”）可以不经授权地决定受保护的字段（如批准状态）或触发高危副作用。
- **动机**：仅靠 schema 验证（检查 JSON 格式）不够，因为形状正确的值可能缺乏对应字段的授权。需要一种执行边界机制，在副作用发生前确保字段、效果和调用事件都拥有与同一动作清单相匹配的权限。

## 2. 方法论：核心思想与关键技术细节
- **核心思想**：**上下文到执行完整性（CXI）**。将动作授权拆分为三个独立但必须绑定的部分：
  - **字段权限**：受保护接收字段（如 tool、operation、approval_state）的值只能来自可信状态（T）或经策略授权的类型化发布（D）。
  - **精确效果授权**：对接收端解释的有效载荷（如补丁、SQL、shell 命令），需通过适配器计算其精确语义效果，并绑定到当前快照、主体、策略纪元等。
  - **调用权限**：调用事件本身（是否执行、重试、批处理、调度）需要消耗一个清单绑定的可消费能力（invocation capability），并获得执行租约。
- **关键技术细节**：
  - **类型化发布（Typed Release, D）**：由可信的解分类器（declassifier）从可写上下文（W）中提取并验证，仅针对一个目标字段授予权限（如 `FilePath` 只授权 `arguments.file_path`）。
  - **不透明数据槽（Opaque Data Slot, O）**：复制未信任文本但不赋予任何权限；重新进入后继边界时视为可写上下文。
  - **确定性门控（Gate）**：经过六个步骤（规范化动作 → 检查字段权限 → 检查效果承诺 → 授权调用 → 消费能力 → 执行）。门控保证同一动作清单绑定所有授权。
  - **动作清单（Action Manifest）**：包含 nonce、动作摘要、适配器 ID、快照、策略纪元、字段证据摘要、效果授权摘要、调用能力摘要。任何变更后执行失败。
- **算法流程（文字说明）**：
  1. 模型提出结构化动作候选。
  2. 门控将其规范化为动作清单（m）。
  3. 对每个受保护字段 f，检查其影响集 Π[f] 是否仅包含可信或针对该字段的 D 原子，且至少有一个原子通过精确值摘要或精确效果承诺授权该字段值。
  4. 对每个效应型载荷，通过适配器生成精确效果承诺并验证其匹配。
  5. 解析调用能力（invoke capability），必须与 m 的调用摘要、策略纪元、快照等匹配。
  6. 在线性化账本中消费该能力，获得执行租约。
  7. 只有租约持有者才能通过被中介的接收端执行。
  - 任何步骤失败则拒绝，无副作用的回退。

## 3. 实验设计

  - **代码智能体精确效果基准**：400 个仓库片段（repository episodes），测试精确效果授权和清单绑定。

## 6. 主要结论与发现
- CXI 在评估的所有接收端上实现了**零字段、效果和调用逃逸**。
- AgentDojo 中 199 个安全任务完成，代码智能体中 231 个安全任务完成（73 个解析失败、61 个无效果授权、35 个质量失败，这些都不是逃逸）。
- 托管/API 路径阻止了 85 个不安全选择，零门控逃逸。
- 门控成功区分了“提案压力”（不安全的提议）与“执行逃逸”：尽管模型经常提出未授权字段，但门控阻止了实际执行。
- 禁止不安全的上下文直接影响受保护字段；类型化发布保留了可用性（例如将 W 中的路径合法提取为 FilePath 类型发布给文件路径字段）。

## 7. 优点
- **细粒度授权**：将字段、效果和调用事件分别授权，且强制绑定同一清单，解决了“权限洗白”问题。
- **精确效果授权**：不是比较字符串，而是比较接收端解释后的语义效果（如补丁在具体仓库基上的状态增量），防止重放和路径替换。
- **可消费能力**：线性化账本消费能力确保同一动作不能重复执行或双花。
- **双路径兼容**：开放权重路径提供最强证据（后端证据记录、注意力掩码），托管/API 路径提供现场建设证据（主机可见的构造边界），但共享同一门控检查。
- **明确的分界线**：将策略、验证器、运行时等假设明确列出，不声称无法保证的方面（如完全的非干涉性或任务质量）。

## 8. 不足与局限
- **依赖健全的策略与完整中介**：如果策略分类不完整、缺少某个受保护字段、或存在绕过门控的接收端路径，则保证失效。
- **任务质量不在保证范围内**：CXI 仅确保授权正确，不确保补丁是正确的、解析成功的、或任务满足用户目标。论文报告了 35 个授权执行但任务失败的例子。
- **不能应对验证器错误**：如果效果适配器未能覆盖接收端解释的所有效果，则可能允许未授权的副作用。
- **托管/API 路径提供较弱的证据**：无法检查提供者内部的注意力掩码或 KV 缓存，因此不能完全排除提供者内部的非授权影响。
- **未实验大规模部署的精确开销**：虽然报告了微秒级别的 mask 构建，但模型推理成本和环境限制未详细讨论。
- **可扩展性**：策略编写是人工工作，需要为每个新接收端花费人力审查，论文中仅审计了 24 个接收端。
- **信任假设列表长**：包括运行时、工具、授权状态、解分类器等，任何一个被攻破则破坏 CXI。
