## 摘要
本文针对战场物联网（IoBT）任务控制的策略感知大语言模型检索增强生成（PA-LLM-RAG）框架进行了对抗性安全研究。我们提出了查询无关的语义检索投毒攻击，这是一种新型攻击方式，通过向IoBT知识库中注入语义精心设计的规则，在所有操作员查询类型下均能获得较高的检索排名，且无需了解运行时提示。该攻击通过单条注入规则（1.6%的投毒率）实现了85%的LLM上下文污染，并在投毒率达到7.7%时饱和，表明即使极小的知识库破坏也足以破坏任务决策。为应对这一威胁，我们提出了CLD-KB（知识库网络层防御），这是一个双检测器异常检测框架，结合了单类支持向量机边界检测与一种新颖的基于成员的类别扩散分析，利用IoBT策略的三类别分类法在决策LLM之前识别受污染规则。CLD-KB在投毒检测和知识保留方面显著优于五种基线方法（包括DBSCAN、LOF、K-Means、孤立森林和单类SVM），且每个任务仅增加7毫秒的计算开销，使其成为面向LLM驱动的IoBT任务系统的有效且可部署于边缘的防御方法。

## Abstract
This paper presents an adversarial security study of the Policy-Aware LLM Retrieval-Augmented Generation (PA-LLM-RAG) framework for Internet of Battlefield Things (IoBT) mission control. We propose Query-Agnostic Semantic Retrieval Poisoning, a novel attack that injects semantically crafted rules into the IoBT knowledge base achieving high retrieval ranking across all operator query types without requiring knowledge of runtime prompts. The attack achieves 85% LLM context corruption from a single injected rule (1.6% poisoning rate) and saturates at 7.7% poisoning, demonstrating that even minimal knowledge base compromise is sufficient to corrupt mission decisions. To counter this threat, we propose CLD-KB (Cyber-Layered Defense for Knowledge Base), a dual-detector anomaly detection framework combining One-Class SVM boundary detection with a novel Member-Based Category Spread analysis that exploits the three-category IoBT policy taxonomy to identify poisoned rules before they reach the decision LLM. CLD-KB significantly outperforms five baseline methods including DBSCAN, LOF, K-Means, Isolation Forest, and One-Class SVM in both poisoning detection and knowledge preservation, with only 7ms computational overhead per mission, establishing it as an effective and edge-deployable defense for LLM-driven IoBT mission systems.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 攻击方法：查询无关语义检索投毒

- **核心思想**：攻击者无需知道运行时查询内容，只需注入一条或多条跨类别词汇的规则，使其嵌入与查询分布质心的余弦相似度足够高，从而在任意查询下都能进入top‑k检索结果。
- **关键技术细节**：
  - 白盒威胁模型：攻击者知晓嵌入模型（nomic‑embed‑text）和知识库类别结构，可写入但不能修改已有规则或拦截查询。
  - 构造策略：分析IoBT操作词汇（gate、drone、UAV、UGV等），在每条投毒规则中嵌入所有资产类型和所有门岗词汇（北、南、东、西门），同时混合三类策略的关键词（工作流程指令、交战规则紧迫性、能力资产引用），使嵌入向量近似处于三类质心等距离位置。
  - 注入规则数 \( M = \lfloor \rho \cdot N / (1-\rho) \rfloor \)，其中 \( N=60 \) 为合法规则数，\( \rho \) 为投毒率（1.6%~25%）。
- **为何标准防御失败**：密度方法（DBSCAN、LOF）将投毒规则误判为内点；聚类方法（K‑Means）和集成方法（孤立森林）只能部分检测，因为投毒规则落在合法分布边界内。

### 2.2 防御方法：CLD-KB

- **整体架构**：双检测器异常检测框架，在检索模块和LLM之间运行。RAG返回top‑10候选规则，每个规则经两个检测器联合判定，任一标记为恶意则永久黑名单移除；直到收集到3条通过验证的规则才送入LLM上下文。
- **检测器1：One‑Class SVM**
  - 在仅含合法规则的训练集上拟合核函数边界，ν=0.2，RBF核γ=0.1。
  - 阈值设为训练得分第3百分位数，确保不超过3%误报。
  - 规则若决策函数值低于阈值则标记。
- **检测器2：Member‑Based Category Spread（成员类别散布）**
  - 利用合法规则严格属于WF/ROE/CAP三类的几何特性：合法规则与同类成员距离近，与其他类远，散布值高；投毒规则因跨类别词汇嵌入，与所有类别成员距离相近，散布值低。
  - 定义一个规则到类别\(c\)的最小余弦距离 \( d_{\min}(r,c) \)，散布值 \( \text{spread}(r) = \max_c d_{\min} - \min_c d_{\min} \)。
  - 阈值 \( \tau_{sp}=0.17\)，低于此值标记为恶意。
- **组合判决**：若任一检测器标记，规则即被拒绝并永久黑名单。
- **自适应黑名单**：每次被标记的规则永久移除，后续任务投毒率单调递减，知识库自我清洁。

## 3. 实验设计：数据集/场景、benchmark、对比方法

- **知识库**：60条合法规则（每类20条：WF、ROE、CAP），加上人工生成的24条对抗规则。
- **任务场景**：基于Gazebo Harmonic仿真（WSL2），包含6个智能体（3架无人机、2辆UGV、1个机器人），5个门岗位置。使用20条操作员提示（覆盖门岗守卫、资源分配、威胁响应、多智能体协调等）。
- **基准对比方法**：DBSCAN、LOF、K‑Means、孤立森林、One‑Class SVM（单独使用）。
- **评估指标**：
  - 规则级检测：召回率、精确度、F1分数。
  - 上下文完整性率（CIR）：最终LLM上下文中无任何投毒规则的任务占比。
- **实验参数**：投毒率从1.6%到25%（共9个等级），每个等级在20条提示上重复3次独立运行。

1. **攻击威胁确凿**：投毒攻击从单条规则（1.6%投毒率）即实现85%上下文污染，在7.7%投毒率时饱和（平均每条上下文含2.65条投毒规则），证明即使是极小规模的投毒也能严重破坏任务决策。
2. **CLD-KB防御有效**：在规则级检测上达到Recall=1.0、Precision=1.0、F1=1.0，零误报；在所有投毒率下保持100%上下文完整性；仅增加7ms/任务延迟。

4. **结构属性可利用**：IoBT策略的三分类几何特征（成员散布值<0.15）是可靠的检测信号，合法规则均>0.20，二者无重叠。
