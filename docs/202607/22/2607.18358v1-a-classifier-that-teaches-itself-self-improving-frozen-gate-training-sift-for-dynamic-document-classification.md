## 摘要
文档分类在实验室中是一个已解决的问题，但在企业应用中却是一个未解难题。阻碍因素很少是模型架构，而是必须在模型构建前进行的标注项目，以及模型一旦存在便让其自主重新训练的机构顾虑。我们提出了SIFT（自改进冻结门训练），一种动态分类器服务，旨在同时解决这两个问题。SIFT通过一条刻意设计的低成本、受限于CPU的流水线提供分类服务，该流水线由SPLADE稀疏编码器输入LightGBM头组成，并且仅将低置信度的少数页面升级至LLM裁判。裁判的判定结果会被写回标注语料库，因此昂贵的模型能够持续教导廉价的模型：升级率随之下降，语料库由生产流量而非前期的标注工作不断积累，且准确率随使用而复利增长。接入新的文档族仅需一个声明式捆绑包、标签空间、锚定短语和裁判术语表，而无需标注项目。更困难的问题在于安全性：自主重训练的分类器可能会发生无声的性能退化。SIFT通过一个由两部分组成的晋升门解决了这一问题：关键标签F1回归检查，加上一个模型从未在其上训练过的冻结黄金回归集，其中任一检查未通过都会否决模型晋升。这将“无需人工每月重训练”从鲁莽之举变为了常规操作。我们描述了该架构、自喂养语料库循环、冻结门晋升机制，以及一个说明性的多领域部署案例，并讨论了边际标注成本趋于零的分类器的经济学效益。

## Abstract
Document classification is a solved problem in the laboratory and an unsolved one in the enterprise. The blocker is rarely model architecture; it is the labeling project that must precede a model and the institutional fear of letting a model retrain itself once one exists. We present SIFT (Self-Improving, Frozen-gate Training), a dynamic classifier service, which attacks both. SIFT serves classification from a deliberately cheap, CPU-bound pipeline, a SPLADE sparse encoder feeding a LightGBM head, and escalates only the low-confidence minority of pages to an LLM judge. The judge's verdicts are written back into a labeled corpus, so the expensive model continuously teaches the cheap one: the escalation rate falls, the corpus grows from production traffic rather than from an up-front annotation effort, and accuracy compounds with use. Onboarding a new document family requires only a declarative bundle, label space, anchor phrases, and a judge glossary, not a labeling project. The harder problem is safety: an autonomously retraining classifier can silently regress. SIFT resolves this with a two-part promote gate, a critical-label F1 regression check plus a frozen golden regression set the model is never trained on, either of which vetoes promotion. This turns "retrain monthly without a human" from reckless into routine. We describe the architecture, the self-feeding corpus loop, the frozen-gate promotion mechanism, and an illustrative multi-domain deployment, and we discuss the economics of a classifier whose marginal labeling cost trends toward zero.

## 2. 提出的方法论：核心思想、关键技术细节与流程
- **核心思想**：构建一个“廉价模型处理常规请求 + 昂贵LLM作为裁判教导廉价模型”的级联系统，实现语料库随生产流量自增长；同时引入双重“冻结门控”机制，确保模型在无人值守下的自动迭代绝对安全。
- **关键技术细节与流程**：
  - **声明式领域捆绑**：接入新文档族无需写代码或标注数据，只需上传包含标签空间、锚定短语、正则包和裁判策略的YAML配置文件。
  - **成本分级服务架构**：
    - 廉价层：SPLADE稀疏编码器提取特征 + LightGBM分类头，纯CPU运行，毫秒级响应。
    - 昂贵层：当廉价模型输出的置信度低于阈值时，将样本升级至LLM裁判进行判定。
  - **自喂养语料库循环**：
    - LLM裁判对低置信样本的判定结果（高于最低置信度阈值）会回写到标注语料库中。
    - **零数据冷启动**：通过引导路径，直接用LLM裁判对无标签原始文档进行标注，生成初始语料库。
    - **主动学习**：将LLM标注中最不确定的样本推送给人类专家审核，专家修正的标签作为真实标签入库。
  - **冻结门控晋升机制**：这是实现安全自主重训练的核心，新候选模型必须同时通过两项检查，任一失败即否决晋升：
    1. **关键标签F1回归检查**：业务不可容忍出错的类别被设为关键标签，候选模型在这些标签上的F1分数若低于当前线上模型，则一票否决。
    2. **冻结黄金回归集**：捆绑包中包含一个从未参与训练的固定黄金样本集，候选模型在该集上的表现下降也会触发否决。这防止了模型在漂移分布数据上产生的虚假提升。

## 6. 主要结论与发现
- SIFT成功将企业文档分类的两大痛点转化为常规操作：新文档分类任务的接入从“昂贵的标注项目”变为“上传声明式配置包”；模型保持更新的方式从“季度人工重训”变为“无人值守的自动迭代”。
- 随着系统运行，廉价模型不断吸收LLM裁判的知识，升级率持续下降，**边际标注成本趋近于零**，准确率随使用量复利增长。
- 冻结门控机制有效杜绝了自主重训练带来的关键类别性能退化风险，使得“每月无人工重训练”成为安全合规的默认配置。

## 7. 优点
- **直击产业真痛点**：跳出“刷榜”思维，将研究重心放在制约AI落地的数据标注与安全合规问题上。
- **巧妙的级联与反哺设计**：将LLM定位为“按小时雇佣的教师”而非通用分类器，仅处理廉价模型不会的难题，并将答案反哺给小模型，极大降低了推理成本。
- **工程完备性极高**：双重门控晋升机制（关键标签+Frozen Set）设计精巧，兼顾了业务关键指标与数据分布漂移的防御，具备极高的生产系统可用性。

## 8. 不足与局限
- **模态单一**：目前仅处理OCR提取的文本，不支持图像嵌入，依赖视觉排版的文档分类超出其当前能力。
- **裁判偏见风险**：LLM裁判的判定被直接回写为训练数据，若裁判存在系统性偏见，廉价模型将继承该偏见。虽然冻结集和人工审核是防线，但若偏见属于盲区则难以察觉。
- **冻结集的时效性**：冻结黄金集作为固定标尺，无法覆盖未来出现的新子类型文档，对全新输入的回归 veto 能力受限。
- **冷启动依赖配置质量**：零数据冷启动的质量高度依赖于人工编写的锚定短语和裁判策略，配置粗糙会导致初始语料库噪声大，收敛慢。
- **学术验证缺失**：缺乏在标准数据集上的定量实验和消融研究，难以量化评估各模块（特别是冻结门控）的绝对收益。
