## 摘要
键值（KV）缓存是长上下文大语言模型（LLM）推理中的主要内存瓶颈。目前两大主流的无训练方法族均存在结构性局限：词元选择方法（SnapKV, Ada-KV）从观测窗口对重要性进行评分并驱逐低分词元，但驱逐是不可逆的——因此当重要性信号在查询无关的重用下衰减时，准确率会暴跌11-15个百分点；均匀低秩编码保留了每个词元，但在各处分配相同的秩，造成了预算浪费。我们观察到，这两种失败共享一种解决方案：秩应当被分配，而非被驱逐。我们提出了VarRate，一种无训练的KV编解码器，它根据每个词元的查询显著性为其分配可变的低秩预算，使每个词元都保持非零秩。现有的可比自适应秩编解码器只能通过训练来实现这种分配；而VarRate无需训练。由于没有词元被丢弃，在查询感知选择方法准确率暴跌的情况下，它仅下降3.5-5.5个百分点。在LongBench（16个任务）上20%的匹配预算下，VarRate在Llama-3.1-8B和Qwen2.5-7B上与未压缩模型的准确率差距均保持在0.8个百分点以内。两者平均而言，它是最强的同等内存压缩器。它在两个模型上都显著击败了其均匀秩消融版本。与专为查询无关重用构建的方法KVzip相比，它在四种设置中的三种下准确率相当，总体差距在1个百分点以内，而预填充开销仅约为其八分之一。

## Abstract
The key-value (KV) cache is the main memory bottleneck in long-context large language model (LLM) inference. Two leading training-free families are both structurally limited: token-selection methods (SnapKV, Ada-KV) score importance from an observation window and evict low-scoring tokens, but eviction is irreversible -- so when the importance signal degrades under query-agnostic reuse, accuracy collapses by 11-15 points; uniform low-rank coding keeps every token but spends equal rank everywhere, wasting budget. We observe that both failures share one cure: rank should be allocated, not evicted. We present VarRate, a training-free KV codec that assigns each token a variable low-rank budget by its query salience, keeping every token at a nonzero rank. Comparable adaptive-rank codecs reach this allocation only through training; VarRate requires none. Because no token is dropped, it degrades by only 3.5-5.5 points where query-aware selection collapses. At a matched 20% budget on LongBench (16 tasks), VarRate stays within 0.8 points of the uncompressed model on both Llama-3.1-8B and Qwen2.5-7B. Averaged over the two, it is the strongest matched-memory compressor. It significantly beats its uniform-rank ablation on both models. Against KVzip, a method purpose-built for query-agnostic reuse, it is accuracy-equivalent in three of four settings and within a point overall, at about one-eighth the prefill overhead.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：长上下文大语言模型（LLM）推理时，键值（KV）缓存随上下文线性增长，成为显存的主要瓶颈。现有的两大类免训练压缩方法存在结构性缺陷：
  - **Token选择法（如SnapKV）**：基于当前查询的观测窗口对token重要性评分并**驱逐**低分token。由于驱逐不可逆，当缓存被复用于其他查询（查询无关场景，如多轮对话、前缀缓存）时，重要性信号失效，导致精度暴跌11-15个百分点。
  - **均匀低秩编码（如Palu）**：保留所有token，但为每个token分配**相同**的低秩预算，导致在关键token上秩不够（信息饥饿），在无关token上秩浪费。
- **整体含义**：两种失败的根本原因在于“响应的不可逆性（驱逐）”与“信号的失效”相遇。本文提出核心洞察：**秩应当被分配，而非被驱逐**。通过为每个token分配可变的低秩预算，即使重要性信号失效，被误判的token也仅仅是精度变粗，而非彻底丢失，从而在极低预算下兼顾高保真与高鲁棒性。

## 2. 提出的方法论：核心思想、关键技术细节与算法流程
- **核心思想**：提出VarRate，一种免训练的KV编解码器。它根据每个token的查询显著性分配可变低秩预算，保证所有token的秩不低于非零下限（$r_{min}$），实现“按需分配，绝不丢弃”。
- **关键技术细节与流程**：
  1. **Pre-RoPE联合与残差编码**：
     - 逆旋转位置编码，将所有KV头的键和值拼接为每token向量 $z_t$（维度 $D=2Gd_h$），使单层单基服务所有头。
     - 保留步长为 $\Delta$ 的锚点token和最近 $w$ 个token为精确集 $E$；其余为编码集 $C$。
     - 编码集token通过减去最近 $c$ 个锚点的均值计算残差 $e_t$。
  2. **共享基（免训练PCA）**：
     - 离线使用少量无标签校准文本，对残差进行SVD提取前 $R$ 个右奇异向量，构建共享PCA基 $V$。无需梯度或微调。
  3. **显著性引导的可变码率分配**：
     - **显著性得分**：复用SnapKV的观测窗口注意力权重，经移动平均和Min-Max归一化得到 $\hat{s}_t \in [0,1]$。
     - **注水分配**：在总预算 $B$ 约束下，按显著性分配秩：$r_t = \text{clip}(r_{min} + \lambda \hat{s}_t, r_{min}, R)$。通过调节 $\lambda$（水位线）并截断溢出，使 $\sum r_t = B$。这保证了：1) 总内存与均匀低秩编码相同；2) 没有token被丢弃（$r_{min} > 0$）。
  4. **重建**：
     - 将残差投影到基上得到系数 $c_t = V e_t$，按分配的秩 $r_t$ 截断保留前 $r_t$ 个系数（类似Matryoshka嵌套截断），重建后重新施加RoPE。

## 6. 主要结论与发现
- **精度与内存**：在20%预算（5倍压缩）下，VarRate在两模型上与未压缩模型差距仅0.8分，是同等内存下最强压缩器。
- **鲁棒性**：在查询无关重用场景下，选择法精度暴跌11-15分，VarRate仅降3.5-5.5分。
- **消融核心发现**：显著优于均匀秩消融版（Llama +2.22, Qwen +17.50），证明“按需分配”比“均匀分配”关键；注水分配比二值分配更能保留摘要所需的分散上下文信息。
- **效率**：达到与KVzip相当的查询无关鲁棒性，但预填充开销仅为其1/8（+44% vs +372%）。
- **正交性**：低秩压缩与量化正交可组合，VarRate+3bit量化可进一步将缓存降至9%且精度几乎无损。
- **边界**：极低预算（8%）下共享低秩基会崩溃，此时不可逆驱逐的选择法反而有生存力；量化在保留分散信息（如长文档摘要）上天然优于低秩法。

## 7. 优点
- **洞察深刻**：精准定位了现有方法“驱逐不可逆”与“信号失效”的致命组合，提出“粗化而非丢弃”的解决范式。
- **工程价值高**：完全免训练，离线PCA基构建成本极低，且与量化无缝组合，部署实用性极强。
- **机制清晰**：通过详尽的消融和负对照（如洗牌显著性、位置先验），确凿证明了“内容对齐的信号+阶梯式分配”的必要性，排除了“仅靠递减形状即可”的替代假设。
- **实验严谨**：统计检验严格，对自身局限（如预填充开销、极低预算崩溃、摘要任务弱于量化）坦诚剖析。

## 8. 不足与局限
- **预填充开销**：虽然远低于KVzip，但+44%的预填充开销仍显著高于SnapKV（+2%），在短生成、重检索的负载中代价明显。
- **极低预算崩溃**：在极低预算（如8%）下，共享低秩基无法表征任何token，方法失效，缺乏在此极端场景下的兜底机制。
- **摘要任务先天弱势**：低秩压缩丢弃坐标方向的特性，使其在处理依赖全局分散信息的摘要任务时，天然弱于保留全秩的量化方法。
- **运行时显存收益未兑现**：当前实现采用“重建全KV缓存”模式，解码速度与未压缩相同，实际显存节省需依赖未来开发的融合算子或张量并行。
- **校准集依赖**：虽证明不同校准集结果稳定（波动<1.8分），但仍需离线数据，非完全“零样本”。
