## 摘要
我们提出了ARBITER，一个新颖的大语言模型（LLM）护栏框架，该框架引入了两个关键理念：（i）双假设推理，一种用于LLM护栏的推理方法，在做出安全决策之前，显式地考虑提示词的安全与不安全解释；（ii）多组件监督微调（MC-SFT），一种用于基于推理的护栏的结构化训练损失，它将LLM输出分解为逻辑组件，并根据其重要性进行加权。现有的基于推理的护栏通常依赖于昂贵的流程，例如使用更大或闭源的教师模型生成推理轨迹，以及应用全参数微调。相比之下，ARBITER采用了一种高性价比的自生成策略来生成推理轨迹，并使用基于LoRA的参数高效微调，同时仍能取得比这些昂贵方法更好的性能。此外，ARBITER为不安全决策提供了忠实的证据短语解释，从而实现了一种更透明、可解释的护栏方法。在三个安全审核基准上的实验表明，ARBITER优于现有的基于推理和非推理的护栏基线，在域外评估中取得了明显的提升。

## Abstract
We propose ARBITER, a novel LLM guardrail framework that introduces two key ideas: (i) dual-hypothesis reasoning, a reasoning method for LLM guardrails that explicitly considers both safe and unsafe interpretations of a prompt before making a safety decision, and (ii) multi-component supervised fine-tuning (MC-SFT), a structured training loss for reasoning-based guardrails that decomposes LLM outputs into logical components and weights them according to their importance. Existing reasoning-based guardrails often rely on expensive procedures, such as generating reasoning traces using larger or closed-source teacher models and applying full-parameter fine-tuning. In contrast, ARBITER uses a cost-effective self-generation strategy for reasoning traces and LoRA-based parameter-efficient fine-tuning while still achieving better performance than these expensive approaches. Additionally, ARBITER provides faithful evidence-phrase explanations for unsafe decisions, enabling a more transparent and interpretable guardrail method. Experiments on three safety moderation benchmarks show that ARBITER outperforms existing reasoning-based and non-reasoning guardrail baselines, with clear gains in out-of-domain evaluations.

  1. **依赖昂贵教师模型**：现有基于推理的护栏通常依赖大参数量或闭源模型（如GPT-4o、DeepSeek-R1-671B）生成推理轨迹，成本高且会引入教师模型偏差。

  4. **推理结构局限**：现有方法多生成单一推理轨迹（先有结论再找理由），容易对模糊提示过早下结论，导致误拒或漏判。
- **整体含义**：本文旨在提出一种低成本、高泛化、且决策过程透明可解释的LLM护栏训练框架，使小模型也能通过自我博弈式的对比推理实现强大的安全审核能力。

## 2. 提出的方法论
### 核心思想
提出 **ARBITER** 框架，包含两大创新：**双假设推理** 和 **多组件监督微调（MC-SFT）**。通过让模型自己生成正反两方观点再进行辩论裁决，避免过早承诺单一解释；同时通过解构损失函数，让模型把学习精力集中在最关键的决策和证据上。

### 关键技术细节
- **双假设推理生成**：
  1. **共享前缀缓存**：将指令和提示词预填充为KV缓存，后续步骤复用。
  2. **安全假设**：附加引导词，让模型生成最强有力的“无害/安全”解释及证据。
  3. **不安全假设**：附加引导词，让模型生成最强有力的“有害/不安全”解释及证据（两者独立且可并行解码）。
  4. **审议**：将上述两个假设随机排序后拼接，引导模型对比双方，得出哪个假设更成立的审议文本。
- **自生成策略**：不依赖大模型，直接用待微调的基础模型（Llama-3.1-8B-Instruct）离线生成上述推理轨迹，大幅降低数据构造成本。

### 公式与算法流程（MC-SFT损失）
标准SFT对所有Token平均计算交叉熵，而MC-SFT将输出序列分解为逻辑组件并加权求和：
- **损失公式**：$L_{MC-SFT} = \lambda_r L_r + \lambda_w L_w + \lambda_y L_y + \lambda_e L_e + \lambda_{syn} L_{syn} + \lambda_a L_a$
  - $L_r$：推理部分损失（安全/不安全假设、审议），权重低（$\lambda_r=0.1$）。
  - $L_w$：胜者预测损失，权重中（$\lambda_w=0.5$）。
  - $L_y, L_e$：最终安全标签与解释损失，权重高（$\lambda_y=\lambda_e=1.0$）。
  - $L_{syn}$：JSON语法格式损失，权重低（$\lambda_{syn}=0.1$）。
  - $L_a$：辅助推理对齐损失，权重较低（$\lambda_a=0.2$）。
- **辅助对齐损失 $L_a$**：包含两部分：
  1. $L_{hyp-delib}$：基于余弦相似度，强制审议表征与真实标签对应的假设表征对齐。
  2. $L_{delib-exp}$：针对不安全提示，强制审议表征与最终解释表征对齐。
- **推理阶段**：采用链式解码与KV缓存复用，依次生成假设、审议、胜者标记及受JSON Schema约束的最终结构化输出。

## 7. 优点
- **推理范式创新**：双假设（对抗式）推理机制符合人类面对模糊情境时的审慎决策逻辑，减少了误判。
- **工程落地性强**：自生成策略摆脱了对大模型的依赖，LoRA使得微调与更新成本极低，KV缓存复用和Token预算控制缓解了推理延迟问题。
- **可解释性与忠实性**：不仅输出结构化证据短语，还通过严格的删除扰动实验量化验证了解释的忠实度，增强了系统的可信度。
- **损失函数设计精妙**：MC-SFT根据自生成数据的特性，区分了“需强化的决策信号”与“仅需维持的推理/格式信号”，为基于推理的微调提供了新思路。
