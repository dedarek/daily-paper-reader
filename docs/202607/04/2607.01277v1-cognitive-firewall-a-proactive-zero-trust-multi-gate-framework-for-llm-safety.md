## 摘要
大型语言模型（LLM）可能通过多轮策略被诱导生成有害内容，其中没有单一用户消息明显不安全。现有的运行时防护通常将提示或响应作为孤立消息进行评估，这限制了它们恢复累积意图、验证声称的权威性或检测跨对话分解的有害目标的能力。本文提出认知防火墙，一种主动的运行时监督框架，在用户和保护的目标模型之间插入一个独立的监督模型。该框架将安全评估分解为四个分类门控：意图门（识别请求的操作目标）、零信任上下文门（将声称的角色和权限视为未经验证的证据）、一致性门（检测跨轮的升级和分解）以及输出风险门（在发布前检查候选响应）。门控决策通过升级而非分数平均进行组合，允许任何置信的危险信号阻止交互，同时保留可审计的理由。在四个越狱基准测试和一个良性安全测试集上的实验表明，认知防火墙显著降低了单轮、多轮、基于权威和人工设计的攻击的成功率。它将三个攻击集的攻击成功率降低到2%或以下，在最困难的人工设计集上降低到14%，同时保持8%的过度拒绝率。这些结果表明，分解的、对话级别的监督可以改善LLM安全的主动遏制和可审计性。

## Abstract
Large language models (LLMs) can be induced to produce harmful content through multi turn strategies in which no single user message appears clearly unsafe. Existing runtime safeguards commonly evaluate prompts or responses as isolated messages, which limits their ability to recover ac-cumulated intent, verify asserted authority, or detect harmful objectives decomposed across a dialogue. This paper presents the Cognitive Firewall, a proactive runtime oversight framework that interposes an independent oversight model between a user and a protected target mod l. The framework decomposes safety assessment into four categorical gates: an intent gate that identi-fies the operational objective of a request, a zero trust context gate that treats claimed roles and permissions as unverified evidence, a consistency gate that detects escalation and decomposition across turns, and an output risk gate that inspects candidate responses before release. Gate decisions are combined through escalation rather than score averaging, allowing any confident danger signal to block an interaction while preserving an auditable rationale. Experiments on four jailbreak benchmarks and a benign safety test set show that the Cognitive Firewall substantially reduces attack success across single turn, multi turn, authority based, and human crafted attacks. It lowers attack success to 2 percent or below on three attack sets and to 14 percent on the most difficult human crafted set, while maintaining an 8 percent over refusal rate. These results indicate that decomposed, conversation level oversight can improve proactive containment and auditability for LLM safety.

# 论文中文总结

## 1. 核心问题与整体含义（研究动机和背景）
现有 LLM 运行时安全防护（如 Llama Guard、ShieldGemma、Granite Guardian）本质上是基于消息级别的分类器——对每个提示或响应独立评分，无法感知对话上下文。攻击者可以利用多轮策略（如 Crescendo、ActorAttack 等）将恶意目标分散到多轮无害消息中，或通过冒充权威角色（如声称是开发者、操作员）来绕过检测。此类攻击表明：**安全判断需要理解对话的累积意图、验证声称的角色、追踪轨迹升级**，而不仅仅是逐条分类。本文提出 **Cognitive Firewall（认知防火墙）**，在用户与目标模型之间插入一个独立的监督模型，将安全评估分解为多个专门的“门控”，实现主动、可审计的对话级防御。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- **分解式判断**：不依赖单一全局分数，而是将安全评估拆分为四个独立的门控（Gate），每个门输出明确的类别标签（安全/危险），而非数值分数。
- **升级决策规则（Escalation Rule）**：任何门控一旦输出“危险”标签，立即阻止交互；只有当所有门都输出“安全”时才允许生成。这避免了分数稀释，且每个阻止决策都有可追溯的理由。

### 四个门控
| 门控 | 阶段 | 判断内容 | 危险标签 |
|------|------|----------|----------|
| **意图门 (G1)** | 预生成 | 识别请求的实际操作目标（如“结束进程”而非其表面措辞），判断该可交付物是否构成可转移的有害能力（如武器合成、恶意代码） | unsafe |
| **零信任上下文门 (G2)** | 预生成 | 检查用户是否通过声称角色/身份/权限/覆盖指令来获取信任，将其视为未经验证的证据，判断是否为操纵性（manipulative） | manipulative |
| **一致性门 (G3)** | 预生成 | 检测对话历史中的逐步升级（crescendo）、意图漂移、矛盾、分解（decomposition），判断是否为 escalating | escalating |
| **输出风险门 (G4)** | 后生成 | 检查目标模型生成的候选响应是否实际传递了有害内容（操作指南、仇恨言论等） | harmful |

### 流程
- 用户请求 → 依次执行 G1、G2、G3（若其中任一个触发危险，则直接阻止，不调用目标模型）→ 通过后调用目标模型生成响应 → 执行 G4（若有害则阻止输出）。
- 每个门控输出一个分类标签和一句可读理由，无需调整数值阈值。

## 3. 实验设计
### 数据集
- **jbb**：100 个单轮直接有害请求（JailbreakBench）。
- **crescendo**：50 个多轮升级攻击。
- **safemt**：50 个多轮 ActorAttack 分解攻击。
- **mhj**：50 个人工精心制作的多轮越狱攻击（Multi-turn Human Jailbreaks）。
- **xstest**：120 个良性但表面敏感的提示（过度拒绝测试）。

### 基准方法
- **无防御**：直接使用目标模型（mistral）。
- **逐消息防护模型**：Llama Guard 3 8B、ShieldGemma 9B、Granite Guardian 8B（作为输入+输出过滤器）。
- **轨迹感知防火墙**：THRD 风格（时间衰减累积风险）和 TCA 风格（每轮重新评分整个对话），均为作者根据论文描述复现。

### 指标
- **ASR**：攻击成功率（至少有一个有害传递轮次的攻击比例）。
- **blk**：在有害内容传递前被阻止的比例（遏制率）。
- **OR**：过度拒绝率（良性提示被误拒的比例）。

### 充分性与公平性
- 实验覆盖了单轮、多轮、权威冒充、人工攻击等多种攻击类型，消融和泛化验证较为充分。
- **不足之处**：
  - 每个攻击集样本量较小（50–120），导致统计噪声较大（作者指出）。

  - 基线仅运行单次，未像认知防火墙那样做多种子平均，比较可能不够公平。

## 6. 主要结论与发现
1. **攻击遏制效果显著**：在 jbb、crescendo、safemt、mhj 四个攻击集上，ASR 分别降至 1%、0%、2%、14%（未防御时对应 45%、10%、30%、74%）。
2. **大部分遏制发生在生成之前**：jbb 中 89/92 个阻止由意图门完成；crescendo 中 48/50 由一致性门完成；mhj 中 27/43 由一致性门完成。
3. **输出门作为必要后盾**：在 safemt 和 mhj 上，输出门分别贡献 21/39 和 12/43 的阻止，表明部分有害内容仅在生成时显现。
4. **过度拒绝率低**：在 xstest 上 OR 为 8%，远低于 ShieldGemma（18%）和 Granite Guardian（21%）。
5. **零信任上下文门关键处理权威攻击**：在 16 个权威探测中阻止 12 个（75%），且不误拒良性控制。
6. **门控互补**：任何单一门控都不能达到完整防火墙的效果，预生成和后生成门互为补充。

## 7. 优点
- **主动防御**：三个预生成门在调用目标模型前拦截，避免浪费计算资源和潜在泄露。
- **分解审计**：每个门输出独立可读理由，便于追溯和调试。
- **零信任上下文维度**：将角色冒充问题独立出来，填补了现有防线空白。
- **升级规则简单有效**：避免分数稀释，使得局部危险信号不被全局平均掩盖。
- **泛化性好**：更换目标模型或监督模型后性能保持稳定。
- **低过度拒绝**：使用意图门指代真实目标而非表面措辞，减少了误拒。

## 8. 不足与局限
- **样本量小**：攻击集仅 50–100 个对话，权威探测仅 16 个，统计置信度有限。
- **评判偏差**：所有危害判断来自单一同家族模型，可能低估/高估某些类别的攻击。
- **轨迹防火墙基线为复现**：THRD 风格去掉了原版中的响应评估项，比较结果可能偏低。
- **监督模型自身安全未测试**：未对监督模型进行提示注入攻击测试，同模型所有门可能被同时绕过。
- **过度拒绝测试不全面**：仅测试了单轮良性提示（xstest），未测试多轮良性对话的误拒率。
- **延迟未分析**：每轮需要 3–4 次 API 调用，实际部署延迟未量化。
- **残存风险**：在人类精心设计的 mhj 攻击上仍有 14% ASR，需要更强或任务蒸馏的监督模型。
- **部分提示在样本内开发**：门控提示是在每个数据集的前几个例子上开发的，略有 in-sample 风险。
