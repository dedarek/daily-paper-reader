## 摘要
现代AI智能体实现（如前沿编码智能体）在运行时链式调用多个工具，产生了每个工具的安全护栏无法解决的安全面，因为单独允许的工具在组合时可能违反组织策略。我们提出了动态安全控制组合器（DSCC），一种用于多工具智能体链的组合安全的两阶段方法。在第一阶段，在会话结账时，最严格集（MRS）算法将每个工具的安全策略组合成单一有效策略，具有形式化的单调性不变量：扩展链只能收紧结果，在任何工具执行前阻止不兼容的组合。任何工具调用的输出将其分类约束传播到会话级别的污点状态，因此后续调用必须满足迄今为止看到的最严格约束。在第二阶段，在运行时，系统通过单调污点状态跟踪智能体接触的数据的敏感性，并撤销会话如果累积的暴露会使后续工具调用成为策略违规。这两个阶段共同提供了纵深防御，其中静态组合防止不安全的链启动，而运行时污点跟踪捕获从特定数据中出现的违规。我们提供了一个参考实现，使用由16个NIST SP 800-53对齐策略管理的32个工具，并在两种组合模式下进行评估。在默认的清除模式下，允许的组合被划分为分类级别集群，阻止了79.2%的策略对和95.5%的三元组。替代的污点模式允许在渗漏边界内的混合分类链，分别阻止42.5%和60.5%。我们讨论了对部署多工具智能体的组织的治理影响，包括效用-安全权衡以及实现链感知策略所需的变化。

## Abstract
Modern AI agent implementations such as frontier coding agents chain multiple tools at runtime that create a security surface that per-tool guardrails are unable to address, as individually permitted tools can violate organizational policies when composed. We propose the Dynamic Security Control Compositor (DSCC), a two-phase approach to compositional security for multi-tool agent chains. In Phase 1, at session checkout, a Most Restrictive Set (MRS) algorithm composes per-tool security policies into a single effective policy with a formal monotonicity invariant that extending a chain can only tighten the result, blocking incompatible combinations before any tool executes. Outputs of any tool call propagate their classification constraints into a session-level taint state, so subsequent invocations must satisfy the most restrictive constraints seen so far. In Phase 2, at runtime, the system tracks the sensitivity of data the agent touches through a monotonic taint state and revokes the session if the accumulated exposure would make a subsequent tool call a policy violation. Together, these phases provide defense in depth, where static composition prevents unsafe chains from starting, and runtime taint tracking catches violations that emerge from the specific data used. We provide a reference implementation on 32 tools governed by 16 NIST SP 800-53 aligned policies and evaluate it under two composition modes. In the default clearance mode, permitted combinations are partitioned into classification-level clusters, blocking 79.2% of policy pairs and 95.5% of triples. The alternative taint mode admits mixed-classification chains within the exfiltration boundary, blocking 42.5% and 60.5% respectively. We discuss the governance implications for organizations deploying multi-tool agents, including the utility-security tradeoff and the changes needed to operationalize chain-aware policies.

- **整体含义**：提出动态安全控制组合器（DSCC），通过两阶段纵深防御，在会话启动前静态组合策略、在运行时通过污点跟踪动态检测违规，从而将安全边界从单一工具扩展到整个工具链。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
- 将每个工具附加一个标准化的六元组安全策略（数据分类、控制绑定、传输禁止标志、流向方向、允许网络区域、最大会话TTL）。
- 在会话结账时，通过**最严格集（MRS）算法**将所有工具的策略组合成一个单一有效策略，满足**单调性不变量**：扩展链只会收紧约束，不会放松。
- 在运行时，通过单调污点状态跟踪实际接触数据的敏感性，在即将违规时撤销会话。

### 关键技术细节
1. **工具策略定义**（六元组）：
   - `Bi`: 控制绑定（NIST控制→Allow/Restrict/Deny）
   - `Ci`: 数据流分类（Public < Internal < Confidential < Restricted）
   - `ϕi`: 传输禁止标志（1=数据必须留在内部）
   - `di`: 流向方向（出站/入站/双向/仅内部）
   - `Zi`: 允许网络区域
   - `τi`: 最大会话TTL
2. **MRS算法（Phase 1）**：四步流程
   - **Step 1: 兼容性检查**：检查是否存在矛盾（如一个工具禁止外传，另一个允许公网操作）。
   - **Step 2: 控制解析**：对每个控制c，取所有声明它的工具的最严格级别：`Leff(c) = max_i li(c)`。
   - **Step 3: 数据流组合**：
     - 分类高水位：`Ceff = max(C0, max_i Ci)`
     - 清除检查（clearance模式）：若任何工具`Ci < Ceff`则拒绝；taint模式跳过此检查。
     - 禁止传播：`ϕeff = max_i ϕi`
     - 分类边界规则：若`Ceff ≥ Confidential`且链包含出站工具，拒绝。
     - 网络区域交集：`Zeff = ∩i Zi`，若为空拒绝。
   - **Step 4: DENY 执行**：对已解析的Deny级别控制，检查组合数据流是否违反。
   - 最终输出有效控制集：`E = ({(c,Leff(c))}, Ceff, ϕeff, Zeff, τeff)`。
3. **运行时执行模型（Phase 2）**：
   - 资源分类：每个资源在访问时被分配敏感性标签。
   - 污点状态：`S = (Cs, ϕs)`，单调更新：`Cs ← max(Cs, Cr)`, `ϕs ← max(ϕs, ϕr)`。
   - 三守卫检查（任一触发即撤销会话）：(1) 会话已禁止外传且当前工具出站；(2) 当前资源禁止外传且链包含出站工具；(3) Phase1已禁止外传且当前工具出站。

## 6. 主要结论与发现

- **清除模式（默认）**：阻止79.2%的策略对（95/120）和95.5%的策略三元组；允许的组合严格按分类级别聚类，无跨级组合。
- **污点模式**：阻止42.5%的策略对和60.5%的策略三元组；允许混合分类链，但坚守渗漏边界（高于Public的数据不能触发出站工具）。
- **单调性不变量成立**：扩展链只会收紧约束，附件工具后策略集更严格。
- **模式选择**：清除模式适合严格监管环境（每个工具必须预授权到高水位）；污点模式适合混合分类管道，但牺牲部分预授权安全性。
- **治理启示**：DSCC提供可审计的单一有效策略工件；阻断是恢复性的（可监控、可修正），而漏放是不可恢复的。

## 7. 优点

- **系统性填补空白**：首次形式化定义多工具组合安全，MRS算法和单调性不变量提供理论保证。
- **纵深防御设计**：静态组合阻断离线可知的违规，运行时污点跟踪捕获由实际数据触发的违规，两者互补。
- **实用性强**：基于NIST SP 800-53控制标准，易于企业集成；提供两种模式适应不同安全-效用权衡。
- **可审计性**：每次阻断清晰归因到哪个工具的哪个控制绑定，便于策略优化。
- **模块化与联邦式策略管理**：工具开发者撰写策略，安全团队拥有共享词汇和审计，降低维护成本。

## 8. 不足与局限

- **运行时实验覆盖不足**：Phase 2仅演示三个示例，缺乏系统化评估（如随机资源分配下的阻断率、性能开销测量）。
- **未考虑复杂链和动态工具集**：假设链在会话前是预定义的，但实际智能体可能动态选择工具（如基于Agent推理选择下一个工具），论文仅提到“已知工具集”，未处理全动态场景。
- **策略一致性假设**：假设所有工具开发者准确、诚实地声明策略，但可能存在策略错误或过时；缺乏策略自动验证机制。
- **分类粒度粗**：分类级别只有4级（Public/Internal/Confidential/Restricted），可能不足以刻画复杂数据的多维度敏感度。
- **未讨论用户授权分级**：策略仅基于工具和资源，未考虑调用用户的不同权限级别（例如：高管和实习生访问同一工具时应不同）。
- **性能开销未量化**：未报告Phase 1组合延迟、Phase 2污点更新延迟，对高吞吐场景的适用性不明。
