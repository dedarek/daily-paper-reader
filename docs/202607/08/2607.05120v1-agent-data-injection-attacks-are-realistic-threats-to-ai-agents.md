## 摘要
AI智能体代表用户提示执行操作，消耗外部数据并基于智能体上下文采取行动。先前关于AI智能体安全的研究主要集中于间接提示注入（IPI）。其研究最充分的类别是指令注入，即攻击者控制的不可信数据被解释为指令。为此，已提出许多缓解措施来防止指令注入攻击。本文引入IPI的新类别——智能体数据注入攻击（ADI）。ADI将恶意数据伪装成可信数据，如安全关键元数据（例如资源标识符或数据来源）或智能体上下文数据（例如工具调用和响应格式）。结果，智能体在不知情的情况下基于攻击者控制的数据执行非预期操作。ADI具有与指令注入攻击相似的攻击影响，因为它导致智能体行为异常并执行非预期操作。尽管影响相似，ADI仍未得到充分探索，且容易绕过现有的IPI防御。我们在真实世界的智能体中发现了多个关键漏洞，允许攻击者发起各种攻击：对Web智能体（Chrome中的Claude、Antigravity和Nanobrowser）的任意点击攻击，以及对编码智能体（Claude Code、Codex和Gemini CLI）的远程代码执行和供应链攻击。我们评估了市售模型和AI智能体中的ADI漏洞，发现ADI在独立LLM和AI智能体设置中均有效。ADI暴露了智能体安全中的关键缺口，表明当前AI智能体未采用基本安全原则：当前智能体未将可信数据与不可信数据隔离。

## Abstract
AI agents act on behalf of user prompts, consuming external data and taking actions based on the agent context. Prior research on AI agent security has primarily focused on indirect prompt injection (IPI). Its most well-studied category is instruction injection, where attacker-controlled untrusted data is interpreted as an instruction. In response, many mitigations have been proposed to prevent instruction injection attacks. In this paper, we introduce a new category of IPI, agent data injection attacks (ADI). ADI injects malicious data disguised as trusted data, such as security-critical metadata (e.g., resource identifiers or data origins) or agent context data (e.g., tool call and response formats). As a result, agents unknowingly execute unintended actions based on attacker-controlled data. ADI has similar attack impacts as instruction injection attacks, because it causes agents to misbehave and execute unintended actions. Despite the similar impact, ADI remains underexplored and easily bypasses existing IPI defenses. We found several critical vulnerabilities in real-world agents that allow an attacker to launch various attacks: arbitrary click attacks on web agents (Claude in Chrome, Antigravity, and Nanobrowser), and remote code execution and supply-chain attacks on coding agents (Claude Code, Codex, and Gemini CLI). We evaluate ADI vulnerabilities across off-the-shelf models and AI agents, and find that ADI is effective in both standalone LLMs and AI agent settings. ADI exposes a critical gap in agent security, signifying that current AI agents do not employ a fundamental security principle: current agents do not isolate trusted data from untrusted data.

# 智能体数据注入攻击（ADI）——详细论文总结

## 1. 核心问题与整体含义（研究动机与背景）
- **研究动机**：现有AI智能体（Agent）安全研究主要关注**间接提示注入（IPI）**，其中最受关注的是**指令注入**（Instruction Injection），即攻击者将恶意数据伪装成指令，使智能体执行非预期任务。然而，作者发现存在一种被忽视的攻击面：**智能体数据注入攻击（ADI）**，它利用智能体上下文内部**可信数据与不可信数据之间缺乏隔离**，将恶意数据伪装成可信元数据（如资源标识符、数据来源、工具调用格式等），从而操纵智能体的行为。
- **核心问题**：当前智能体安全防御（如模型硬化、输入/输出护栏、双LLM架构等）主要针对指令注入，但对ADI几乎无效，因为ADI不篡改指令，而是篡改被智能体视为可信的数据。这暴露了智能体安全设计中的基本原则缺失：**没有隔离可信数据与不可信数据**。
- **整体含义**：ADI是一种与指令注入具有同等危害的新威胁，能导致任意点击、远程代码执行、供应链攻击等严重后果，并且容易绕过现有防御，需要重新审视智能体安全架构。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：通过**概率定界符注入（Probabilistic Delimiter Injection）**，在不可信数据字段中注入看似无害但被LLM误解为结构定界符的字符序列，从而伪造额外的可信数据对象（如伪造的邮件对象、伪造的工具调用结果、伪造的用户评论等）。攻击者并不需要精确匹配语法，因为LLM概率性地处理数据，即使是不精确的定界符（如转义引号、花括号等）也可能被误解。
- **关键技术细节**：
  - ADI攻击定义：将不可信数据 \( D_U \) 中注入的恶意数据 \( D_A \) 误认为是可信数据 \( D_T \) 的一部分，即 \( \text{LLM}(I, (D_T, D_U \parallel D_A)) \approx_{\text{ADI}} \text{LLM}(I, (D_T \parallel D_A, D_U)) \)。
  - 攻击实现步骤：
    1. 确定目标智能体使用的数据格式（如JSON、Web DOM、自定义标记格式）。
    2. 在不可信字段（如邮件正文、网页评论、GitHub issue评论）中注入包含**概率定界符**的payload，这些定界符在确定性解析器（如工具）中被视为普通字符，但LLM概率性地将其识别为结构边界。
    3. 利用这些伪造的结构边界创建虚假的可信数据（如伪造的“sender”字段、伪造的元素ID、伪造的工具调用结果）。
  - 三种典型攻击变体：
    - **元素ID注入**（Web agent）：伪造网页摘要中的UI元素条目，使智能体误以为存在一个“Read More”按钮，实际点击却是“Buy Now”等敏感按钮。
    - **来源注入**（Coding agent）：伪造GitHub issue评论中的作者名与角色（如maintainer），诱使智能体执行恶意命令。
    - **工具调用注入**（Coding agent）：伪造工具调用块（包括伪造的`read_pr_commit`调用和结果），使智能体根据伪造的良性代码变更合并本应拒绝的恶意PR。
- **关键创新**：提出**概率定界符注入**这一概念，不同于传统确定性定界符注入（如SQL注入、XSS）要求精确匹配，它利用LLM的概率性理解，使攻击更容易且难以防御。

## 6. 主要结论与发现
- **独立LLM高度脆弱**：在所有6个模型上，概率定界符注入在JSON上实现31.3%–43.3%的ASR，在Web DOM上实现33.3%–100.0%的ASR。不同定界符变体（如转义引号、花括号、括号等）均能成功攻击。
- **现有防御对ADI无效**：针对指令注入的防御（如模型硬化、输入/输出护栏、双LLM等）对ADI的ASR高达22.2%–50.0%，只有CaMeL Strict（严格数据流跟踪）实现了0% ASR，但效用大幅下降至36.5%。
- **随机化对键值格式有效但对非键值格式有限**：随机化（如随机元素ID）将JSON的ASR降至0%–3.0%，但在Web DOM上当攻击者猜中正确的nonce时ASR回升至60%–100%。对于非键值格式（如Markdown）无法保护。
- **清洗会严重降低效用**：清洗策略虽然能降低ASR，但良性效用从81%–100%降至67%–81%，因为合法内容中的定界符也被清除。
- **现实世界攻击成功**：在Claude in Chrome、Antigravity、Nanobrowser上实现了任意点击；在Claude Code、Codex、Gemini CLI上实现了远程代码执行和供应链攻击。

## 7. 优点
- **首次系统定义ADI攻击类别**：清晰区分指令注入与数据注入，提出了形式化定义和攻击机制。
- **揭示根本性安全缺口**：指出现有智能体缺乏可信/不可信数据隔离，这是传统软件安全（数据执行保护、SQL注入防御）早已解决的原则问题。
- **充分的现实世界验证**：攻击了多个主流商用智能体，并成功进行了PoC演示。
- **全面的防御评估**：涵盖7种防御策略，并分析了各自的局限性和效用-安全权衡。
- **开源基准和工具**：发布概率定界符注入基准和扩展的AgentDojo基准，促进后续研究。

## 8. 不足与局限
- **防御评估覆盖不全**：未评估如结构化输入模板、专用分隔符令牌、更先进的PAC防御等。对CaMeL Normal实现中的bug（数据流标注丢失）导致ASR偏高，但作者已报告。
- **随机化局限**：仅对键值格式有效，对非结构化格式（如Markdown）无力，且攻击者可能通过观察或猜测绕过。
- **攻击者能力假设**：假设攻击者知道智能体使用的数据格式，虽然作者给出了多种恢复方法（开源分析、逆向、越狱提示），但未系统评估在所有场景下的可行性。
- **智能体评估规模有限**：基于AgentDojo，可能无法完全反映真实生产环境的复杂性（如长上下文、多轮交互、动态网页）。
- **未考虑防御与攻击的对抗性演化**：防御设计（如数据流跟踪）可能面临新的绕过技术（如语义伪装），论文未深入讨论。
- **效用-安全权衡未充分优化**：CaMeL Strict虽然达到0% ASR但效用只有36.5%，实际部署可能不可接受。
