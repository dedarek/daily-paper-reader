## 摘要
在用户本地机器上运行的个人AI代理（如OpenClaw）能够自动化日常任务，包括网页搜索、电子邮件和文件管理。这些代理对计算机资源（包括网络、文件系统和shell）的访问使其面临间接提示注入（IPI）攻击的风险。先前的双LLM防御通过用代理可以引用但不能读取的符号替换不可信数据来阻断IPI。然而，这些防御仅在代理的上下文中追踪不可信数据，因此当代理保存并随后重新读取不可信数据时，该数据（可能包含攻击者的提示）会作为可信数据而非符号返回，我们称之为存储型IPI。在用户真实环境（人类和程序共享）中运行正是使OpenClaw等代理实用的原因，而忽略这一点的防御是不完整的。在这样的环境中保留符号很困难，因为人类和程序需要原始数据。我们提出DualView，它将不可信数据追踪从代理的上下文扩展到用户环境，包括文件系统、shell、网络和其他代理，通过为每个通道提供两个视图。在AgentView中，即使代理将不可信数据写出并读回，也将其视为符号，从而阻断存储型IPI；而HumanView则为人类和工具保留原始数据。DualView将每个工具调用路由到正确的视图，并在两个视图之间同步数据。DualView作为OpenClaw插件部署，仅使用工具钩子，无需更改代理的工具调用逻辑或工具实现。由于DualView通过设计隔离不可信数据，其保护不限于已知的攻击模板。在IPI基准测试和PinchBench上的评估中，DualView阻断了所有IPI攻击（包括存储型IPI），同时保持了与未受保护基线相近的可用性。

## Abstract
Personal AI agents that run on the user's local machine, such as OpenClaw, automate daily tasks including web search, email, and file management. Their access to computer resources, including the network, file system, and shell, exposes them to indirect prompt injection (IPI) attacks. Prior Dual LLM defenses block IPI by replacing untrusted data with symbols that the agent can reference but not read. However, they track untrusted data only inside the agent's context, so when the agent saves and later rereads untrusted data, that data, possibly an attacker's prompt, can return as trusted data rather than as a symbol, which we call stored IPI.   Operating on the user's real environment, which humans and programs share, is what makes agents like OpenClaw practical, and is exactly why a defense that ignores it is incomplete. Preserving symbols in such an environment is hard, because humans and programs need original data. We present DualView, which extends untrusted data tracking from the agent's context to the user's environment, including the file system, shell, network, and other agents, by giving each channel two views. In AgentView, the agent sees untrusted data as symbols even after writing it out and reading it back, blocking stored IPI, while HumanView preserves original data for humans and tools. DualView routes each tool call to the right view and synchronizes data across the two views. DualView deploys as an OpenClaw plugin using only tool hooks, without changing the agent's tool-call logic or tool implementations. Since DualView isolates untrusted data by design, its protection is not limited to known attack templates. In our evaluation on an IPI benchmark and PinchBench, DualView blocked every IPI attack, including stored IPI, while keeping utility close to the unprotected baseline.

- **问题**：个人AI代理（如OpenClaw）运行在用户本地机器上，可访问网络、文件系统、shell等资源，因此面临**间接提示注入（IPI）**攻击。攻击者将恶意指令隐藏在外部数据中，代理读取后可能执行危险操作（如命令执行、数据窃取）。
- **已有防御的不足**：现有双LLM模式（Dual LLM）虽通过符号替换不可信数据来隔离，但仅追踪代理上下文内的数据。一旦代理将数据写入文件再读回，符号会消失，数据恢复为原始文本——即**存储型IPI（Stored IPI）**。这暴露了设计缺口：既要保护环境（人类和程序使用原始数据），又要保持安全追踪，两者在共享环境中矛盾。
- **整体含义**：需要在保持代理实用性的同时，彻底解决存储型IPI，使个人AI代理能安全地在用户真实环境中长期运行。

## 2. 方法论：核心思想、关键技术细节

### 核心思想：双视图（DualView）
- **AgentView**：代理看到的视图，不可信数据始终呈现为**符号**（如`$ws01.content`），即使写入文件再读回，符号保持不变，从而阻断存储型IPI。
- **HumanView**：人类和非代理程序看到的视图，所有数据保持**原始内容**，便于日常使用和工具交互。

### 关键技术细节
1. **视图路由**：
   - **AgentView工具**：可操作符号的文件工具（读写）、本地shell（AgentShell，网络禁用，运行在Agent文件系统），以及可信代理间通信。
   - **HumanView工具**：需要原始数据的网络工具、人类shell（HumanShell，启用网络）、不可信代理通信。对这些工具调用，DualView在进入HumanView前**去符号化**（desymbolize），返回时**符号化**（symbolize）不可信数据。

2. **文件系统同步**：
   - 使用Git双工作树（worktree）：HumanView为主工作树，AgentView为另一个工作树。
   - 每次AgentView工具调用前，同步HumanView用户编辑到AgentView；调用后，将AgentView更改同步回HumanView，并去符号化符号，使人类看到原始数据。

3. **Shell工具**：
   - **AgentShell**：运行在Agent文件系统，网络禁用，输出中保留符号；适用于依赖可信数据和符号的命令（如grep关键字）。
   - **HumanShell**：运行在正常环境，网络启用，输出默认全部符号化；适用于需要原始数据或网络的命令（如curl）。额外通过**数据使用策略**检测符号作为命令或代码执行的情况，要求用户审批。

4. **数据信任策略（Data Trust Policy）**：
   - **模式规则**：对每个工具输出字段标记可信/不可信（如网页内容不可信，状态码可信）。DualView内置21个OpenClaw工具的默认规则。
   - **来源规则**：按源（如网络端点、文件路径、代理）分类，用户可配置可信端点。

5. **数据使用策略（Data Usage Policy）**：
   - 最佳努力层，检测可能不安全的符号使用（如符号作为命令或通过解释器执行），要求用户审批。可扩展到敏感参数（如邮件收件人）。

6. **部署方式**：
   - 作为OpenClaw插件，通过事件钩子（pre-tool, post-tool等）和自定义工具实现，无需修改模型或现有工具。

## 6. 主要结论与发现

- **安全性**：DualView实现**0%攻击成功率**，完全阻止即时IPI和存储IPI。输入/输出护栏仍有残余ASR（如Haiku输入护栏存储IPI 15.6%），双LLM存储IPI ASR达53.3%（Haiku），沙箱也0%但效用严重下降。

- **审查负担**：DualView仅引起1次审查事件（假阳性），而护栏导致多次误报。
- **成本**：DualView令牌开销48.2%（Haiku）和93.4%（Sonnet），高于双LLM的40.8%/61.0%，但远低于沙箱的57.6%/115.6%。
- **结论**：DualView同时满足安全、代理效用、人类效用，且易于部署。

## 7. 优点

1. **设计隔离而非依赖模型**：攻击阻止是确定性的，不依赖模型抗性，对未知攻击模板同样有效。
2. **彻底解决存储型IPI**：通过跨符号的文件系统同步，保持符号在写-读路径中不变。
3. **双视图兼顾安全与可用**：AgentView保护代理，HumanView保持环境原样，避免符号污染。
4. **部署友好**：作为OpenClaw插件用钩子实现，无需修改模型、工具或核心逻辑；支持多数支持钩子的代理运行时。
5. **丰富策略**：数据信任/使用策略可扩展，提供细粒度控制和用户可自定义。

## 8. 不足与局限

1. **数据使用策略不完善**：只检测符号作为命令执行，未覆盖敏感参数（如邮件收件人），也无法可靠覆盖所有shell变异；需要系统级监控或受限shell语言。
2. **HumanView直接放置数据**：用户或程序直接写到HumanView的不可信数据（如粘贴攻击文本）不会被符号化，用户需自行配置来源规则。
3. **自定义工具路由需手动分配**：当前每个工具必须手动指派给AgentView或HumanView，缺乏自动推断。
4. **远程AgentView缺失**：无法在不受控制的远程服务（如Google Docs）上构建AgentView，这类存储仍可能导致IPI攻击（需服务端配合）。
5. **成本较高**：令牌开销（尤其Sonnet）93.4%，对成本敏感场景不利；可通过更小的U-LLM、缓存等优化，但目前未实现。
6. **符号碎片问题**：命令或编辑中分割符号会导致碎片不被解析，可能降低人类效用（实验中未观察到）。
7. **并发处理简单**：采用文件锁和冲突日志，但冲突只能日志记录，未提供自动解决机制。
