## 摘要
大型语言模型（LLM）的快速集成推动了多智能体系统（MAS）的发展，其中专门化的智能体协作执行复杂工作流。在这些环境中，有效的编排需要稳健的路由机制，以将任务高效分配给最合适的智能体。然而，现有路由器从根本上依赖未经验证的代理——从文本自述到静态替代表示——来衡量智能体的能力。这种对非经验数据的依赖造成了智能体预期特征与其实际运作能力之间的关键差距，引入了严重的安全漏洞。恶意智能体可以轻易歪曲其能力，或隐藏后门，从而逃避标准外部分析和静态表征学习技术。在这项工作中，我们提出了ANTAP（自动非文本智能体选择器），一种评估驱动的路由架构，它摒弃间接代理，转而采用主动能力测试。通过动态查询智能体以经验性地确定其真实能力，ANTAP将性能提炼为共享语义空间内的固定行为算子。在推理时，通过纯非文本的代数投影执行路由，建立起一道“语言防火墙”，使得基于元数据的攻击无法表达。在我们的实验中，ANTAP对基于描述注入的攻击实现了近乎为零的攻击成功率（ASR），而基于描述的路由基线则达到67.3%及以上。针对自适应嵌入攻击，ANTAP的ASR比基于嵌入的基线显著降低20%，并且其设计上对描述操纵具有弹性。

## Abstract
The rapid integration of Large Language Models (LLMs) has driven the evolution of Multi-Agent Systems (MAS), where specialized agents collaborate to execute complex workflows. Effective orchestration in these environments requires robust routing mechanisms to efficiently allocate tasks to the most suitable agent. However, existing routers fundamentally rely on unverified proxies, ranging from textual self-descriptions to static surrogate representations, to gauge an agent's competence. This reliance on non-empirical data creates a critical gap between an agent's projected profile and its actual operational capabilities, introducing severe security vulnerabilities. Malicious agents can easily misrepresent their proficiencies or harbor covert backdoors that evade both standard external analysis and static representation-learning techniques. In this work, we introduce ANTAP (Automatic Non-Textual Agent Picker), an evaluation-driven routing architecture that discards indirect proxies in favor of active capability testing. By dynamically querying agents to ascertain their true competencies empirically, ANTAP distills performance into fixed behavioral operators within a shared semantic space. At inference time, routing is performed via a purely non-textual algebraic projection, establishing a "linguistic firewall" that renders metadata-based attacks inexpressible. In our experiments, ANTAP achieves near-zero ASR against description-based injection attacks, compared to 67.3\% and above for the description-based router baseline. Against adaptive embedding attacks, ANTAP achieves substantially lower ASR than the embedding-based baseline, with a 20\% reduction, while remaining resilient to description manipulation by design.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：多智能体系统（MAS）中，路由机制需要将任务分配给最合适的智能体。现有路由器普遍依赖智能体的文本自我描述或静态代理表示（如工具文档、系统提示摘要）来评估其能力。这种对非经验数据的依赖存在关键安全漏洞：恶意智能体可以轻易歪曲其能力描述或隐藏后门（后门攻击），通过注入指令劫持路由器，导致系统被操控。
- **核心问题**：如何设计一种路由架构，使其从根本上摆脱对不可信文本元数据的依赖，从而消除描述注入、潜伏后门等攻击路径。
- **整体含义**：作者提出ANTAP（Automatic Non-Textual Agent Picker），通过主动能力测试（离线基准评估）将智能体的真实行为蒸馏为共享语义空间中的固定线性算子，在线路由时仅进行纯非文本的代数投影，形成“语言防火墙”，使元数据攻击在数学上无法表达。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：抛弃文本描述，转而通过在线下可信环境中评估智能体在基准任务上的实际表现，构建行为算子；推理时将用户查询的嵌入向量与这些算子做几何对齐，选择预测成功率最高的智能体。
- **关键技术细节**：
  - **离线阶段**：
    1. 使用一组可信的基准查询 \(B = \{q_1, \dots, q_N\}\)。
    2. 对每个智能体 \(a_i\)，执行所有基准查询，得到性能向量 \(Y_i \in \{-1, +1\}^N\)（+1 表示正确回答且无未授权工具调用，-1 表示失败或违规）。
    3. 将查询嵌入为矩阵 \(Q \in \mathbb{R}^{N \times d}\)，通过Tikhonov正则化最小二乘问题求解行为算子 \(W^* = Y(Q Q^\top + \lambda I)^{-1} Q\)（正则化参数 \(\lambda=1.0\)）。
    4. 新智能体注册时只需在基准上评估一次，然后投影到已缓存的正则化伪逆上，复杂度 \(O(N)\)。
  - **在线阶段**：
    1. 给定新查询 \(q_{\text{new}}\)，计算嵌入 \(z = E(q_{\text{new}})\)。
    2. 计算分数向量 \(s = W^* z\)。
    3. 选择 \(a^* = \arg\max_i s_i\)。
  - **安全特性**：路由过程中从不处理任何文本描述；在线计算仅为矩阵-向量乘法，不依赖文本长度；对后门智能体，由于校准集中包含带触发词的查询（导致失败标签），其算子会编码负相关，自然被抑制。
- **公式/算法流程**：离线构建性能矩阵Y，计算正则化伪逆，在线一步投影。无需训练神经网络，属于闭式解。

## 6. 论文的主要结论与发现

- **安全性**：ANTAP对描述注入攻击的ASR接近零（0.2%-2.4%），而AutoGen高达67.3%-75.3%；对5种后门攻击，ANTAP的ASR均低于2.4%，基线为12%-21%（EmbedLLM）和73%-78%（AutoGen）；对有害请求规避，ANTAP的ASR最低（54.55% vs 64.20%和67.61%）。
- **性能与延迟**：在线延迟约27-38ms，远低于AutoGen的400-2500ms；路由准确率在多数设置下优于基线。
- **消融**：ANTAP的结构优势独立于模型多样性、提示长度、描述长度、智能体数量等，且智能体越多（K=19）ASR反而下降（0.46%），而AutoGen上升。
- **自适应攻击**：ANTAP的ASR在触发器1-16个时仅波动在55%-63%，而EmbedLLM从1个触发器的69.6%迅速升至3个以上的86%以上。

## 8. 不足与局限
