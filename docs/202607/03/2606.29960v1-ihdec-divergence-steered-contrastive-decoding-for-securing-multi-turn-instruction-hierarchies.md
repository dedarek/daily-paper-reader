## 摘要
大型语言模型（LLM）在处理包含不同角色优先级的多源输入时，往往无法维持指令层级（IH），并在冲突期间矛盾地遵循低优先级指令。现有防御措施虽能缓解此问题，但主要局限于单轮场景且需要昂贵的微调。本文通过詹森-香农散度（JSD）框架形式化了多轮上下文中的这种失效模式，揭示了一种普遍存在的角色影响反转现象，即从属输入覆盖上级角色。为无需训练地纠正此问题，我们提出IHDec（指令层级引导解码）。IHDec利用JSD自动检测令牌级层级违规，并动态执行对比解码以抑制失调的从属角色。广泛评估表明，IHDec在多轮冲突中优于基于训练的基线方法，同时完全保持通用响应质量。此外，IHDec增强了对对抗提示注入的安全性，并表现出与更大模型协同的稳健扩展性。代码见https://github.com/nxcolelxu/IHDec.git

## Abstract
Large Language Models (LLMs) often fail to maintain instruction hierarchies (IH) when processing multi-source inputs with varying role-level priorities, paradoxically adhering to lower-priority directives during conflicts. While existing defenses mitigate this issue, they are largely restricted to single-turn scenarios and require expensive fine-tuning. In this paper, we formalize this failure mode in multi-turn contexts via a Jensen-Shannon Divergence (JSD) framework, uncovering a pervasive role-influence inversion phenomenon where subordinate inputs override superior roles. To rectify this without training, we propose IHDec (Instruction Hierarchy-steered Decoding). IHDec leverages JSD to automatically detect token-level hierarchy violations and dynamically executes contrastive decoding to suppress misaligned subordinate roles. Extensive evaluations demonstrate that IHDec outperforms training-based baselines in multi-turn conflicts while fully preserving general response quality. Furthermore, IHDec strengthens safety against adversarial prompt injections and exhibits a robust scaling synergy with larger models. The Code is available at https://github.com/nxcolelxu/IHDec.git

## 1. 核心问题与整体含义（研究动机与背景）

大型语言模型（LLM）在处理多源输入（如系统提示、用户消息、助手回复、外部数据）时，需要严格遵循预定义的指令层级（Instruction Hierarchy, IH）：**System ≻ User ≻ Assistant ≻ Data**。然而，实际中LLM在多轮对话场景下常出现**角色影响反转**（Role-Influence Inversion）现象——低优先级的从属指令反而比高优先级指令更主导生成，导致系统安全约束被用户指令覆盖。

现有防御方法（如ISE、VerIH、Prompting）大多只适用于单轮对话，且依赖昂贵的模型微调，缺乏可扩展性和模型无关性；同时无法应对多轮对话中对话历史累积增强用户指令影响的问题。

**本文核心动机**：提出一种无需训练、推理时干预的解码方法，动态修复多轮对话中的指令层级违反，同时保持模型通用能力。

## 2. 方法论：核心思想、关键技术、流程

### 核心思想
利用**詹森-香农散度（JSD）** 实时量化每个角色对当前生成logit分布的影响，自动检测层级违反；当检测到低优先级角色影响超过高优先级角色时，触发**对比解码**（contrastive decoding）惩罚低优先级logits，增强高优先级信号。

### 关键技术细节

- **角色影响归因**：对于输入Prompt PT（包含System、User、Assistant等角色），定义角色R的影响分数 I(R) = JSD( P(·|PT) ∥ P(·|PT \ R) )，即对比完整上下文与移除角色R后的上下文分布差异。
- **冲突集检测**：从高优先级到低优先级扫描，若某低优先级角色L的I(L) > I(H)（H为上级角色），则L加入冲突集CS(H)。此即“角色影响反转”的数学定义。
- **动态对比解码**：
  - 对每个H，设计方向logit调整向量 ΔH = log P(·|P \ CS(H)) - log P(·|P \ H) （其中P \ H作为anti-expert，P \ CS(H)作为expert）。
  - 归一化所有ΔH得到最终扰动向量Δfinal（除以L2范数+ε）。
  - 每个解码时间步t，更新最终logits： l_final_t = l_raw_t + β·(β_decay)^t · Δfinal。
  - β=1300，β_decay=0.97；加入衰减因子避免过度修正导致主题漂移。

### 流程
1. 前向传播时并行计算所有角色（3~5个）的logit分布（共享KV缓存，仅改变注意力掩码）。
2. 计算每个角色的JSD影响分数。
3. 识别冲突集CS(H)（若有）。
4. 计算对比调整向量并归一化。
5. 修正当前token logits，继续生成。

5. **可作为即插即用增强器**：与现有方法（ISE、VerIH、Prompting）组合后进一步改善冲突场景表现。

## 7. 优点

- **训练无关（training-free）**：无需微调或权重修改，即插即用，适用于任何可提供logit访问的开源LLM。
- **多轮友好**：首次系统解决多轮对话中IH违反问题，揭示了对话历史累积导致角色影响反转的机制。

- **强可解释性**：基于JSD的角色影响分数提供了违反检测的可视化诊断工具。
- **安全性**：有效抵御提示注入攻击，且不降低通用能力。
- **兼容性**：可与现有训练方法（ISE、VerIH）结合，进一步提升性能。
