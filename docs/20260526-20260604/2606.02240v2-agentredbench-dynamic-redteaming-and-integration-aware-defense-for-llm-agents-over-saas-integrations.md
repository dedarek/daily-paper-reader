## 摘要
工具使用型代理中的间接提示注入是一个具体的生产威胁：LLM代理从集成（通过工具调用访问的第三方服务，如Gmail、Salesforce或Jira）中读取响应内容，而用户既未编写也无法控制这些内容。现有基准测试低估了这一威胁：大多数仅涵盖少量集成，且攻击载荷在多次运行中重复使用，开源防护模型则是在聊天风格数据而非工具响应内容上训练的。我们引入了AGENTREDBENCH，这是一个由LLM驱动的动态红队测试基准，涵盖了9个功能家族、24个企业集成中的215个细微未明确授权场景（攻击处于用户请求授权边界），涉及5种攻击类型。在包含8个模型（Anthropic、OpenAI、Google）的评估面板中，无防护的攻击成功率（ASR）范围从32%（Claude Sonnet 4.6）到81%（Gemini 3 Flash）。为了使场景集不在训练语料中出现，并保持ASR数值随时间有意义，我们开源了代码库、集成架构和AGENTREDGUARD模型；规范场景通过维护者调解渠道进行评估，并采用不可变的版本控制。我们在发布基准测试的同时发布了AGENTREDGUARD：一个在多样化集成的对抗性工具响应内容语料上训练的防护模型。AGENTREDGUARD将面板ASR从69.9%降至2.4%，误报率为0.37%，在两个指标上均优于所有具有实质性检测能力的开源基线模型（Llama Guard、PromptGuard 2、ProtectAI）。跨集成和跨攻击类型的留出测试均确认了该增益可迁移至训练子集之外。

## Abstract
Indirect prompt injection in tool-use agents is a concrete production threat: LLM agents read from integrations (third-party services such as Gmail, Salesforce, or Jira accessed through tool calls) whose response content the user neither writes nor controls. Existing benchmarks under-measure the threat: most cover only a handful of integrations with the same attack payload replayed across runs, and open-source guards are trained on chat-style data rather than tool-response content. We introduce AGENTREDBENCH, a dynamic LLM-driven redteaming benchmark of 215 subtle underspecified authorization (attacks at the boundary of what the user's request authorises) scenarios across 24 enterprise integrations in nine functional families and five attack types. Across an eight-model panel (Anthropic, OpenAI, Google), no-guard ASR (attack success rate) ranges from 32% (Claude Sonnet 4.6) to 81% (Gemini 3 Flash). To keep the scenario set out of training corpora and preserve headline ASR meaning over time, we release the codebase, integration schemas, and AGENTREDGUARD model openly; the canonical scenarios are evaluated through a maintainer-mediated channel with immutable versioning. We release AGENTREDGUARD alongside the benchmark: a guard trained on an integration-diverse corpus of adversarial tool-response content. AGENTREDGUARD cuts panel ASR from 69.9% to 2.4% at 0.37% false-positive rate, outperforming every open-source baseline with non-trivial detection (Llama Guard, PromptGuard 2, ProtectAI) on both axes. Cross-integration and cross-attack type holdouts both confirm the gain transfers beyond the training subset.

- **核心问题**：LLM 代理通过工具调用与第三方 SaaS 集成（如 Gmail、Slack、Salesforce）交互时，会读取并处理用户无法控制的响应内容。攻击者可通过间接提示注入（Indirect Prompt Injection）在这些响应中嵌入恶意指令，诱导代理执行未授权的操作（如转发邮件到攻击者邮箱、修改重要记录等）。现有基准测试覆盖集成少、攻击载荷固定，开源防护模型（如 Llama Guard、WildGuard）基于聊天数据训练，难以检测工具响应中的注入。
- **整体含义**：随着 LLM 代理在生产环境中广泛部署，针对工具响应层的攻击已成为真实威胁。论文旨在提供动态、全面的评估框架（AgentRedBench）和针对性的防护模型（AgentRedGuard），推动 LLM 代理安全性研究与防御落地。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：
  - 设计一个**动态 LLM 驱动的红队测试基准**，覆盖多个企业级集成和五种“微妙未授权”攻击类型。
  - 训练一个**轻量级工具响应分类器**作为运行时防护，在响应进入代理上下文前进行过滤。
- **关键技术细节**：
  - **AgentRedBench**：
    - 采用**四种组件**：攻击者 LLM → 编排器（将攻击内容注入模拟集成状态）→ 目标代理（执行工具调用循环，可选加防护）→ LLM 评判器（根据成功/效用标准评分）。
    - 攻击内容由攻击者 LLM **动态生成**（基于集成模式、攻击类型和先前尝试的评判反馈），**非固定模板**，确保每次运行载荷不同。

### 6. 论文的主要结论与发现

- **无防护威胁严重但存在对齐差异**：无防护 ASR 从 Claude Sonnet 4.6 的 **32%** 到 Gemini 3 Flash 的 **81%**；在 Anthropic 内部 Sonnet 与 Haiku 差距达 47.4%，表明**对齐训练（而非模型规模）**是抵抗的关键。
- **AgentRedGuard 显著降低 ASR**：将面板平均 ASR 从 **69.9% 降至 2.4%**，误报率仅 **0.37%**，在检测率与误报率上**帕累托优于**所有开源基线（Llama Guard、PromptGuard 2 检测率近乎零，ProtectAI 检测率 15% 但误报率 25%）。
- **泛化性强**：跨集成留出测试 TPR 达 **99.75%**，跨攻击类型留出测试 TPR **≥99.36%**，说明分类器学到了通用表面标记而非特定集成或攻击类型特征。
- **推理开销低**：中位 CPU 延迟 9.5 ms，适合生产环境串联部署。

### 7. 优点

- **动态攻击者设计**：攻击载荷每轮动态生成，避免模板复用，更贴近真实攻击者行为，也使基准不易因数据记忆而失效。
- **覆盖广泛**：24 种企业集成、9 个功能家族，远超现有基准（如 AgentDojo 仅 4 个环境）。
- **多集成链式攻击**：首次包含 49 个跨集成链式场景，拓展了安全评估维度。
- **轻量级专用防护**：23M 参数分类器，训练成本极低，推理无需 GPU，且与训练时防御（如 Instruction Hierarchy）正交。
- **严谨的泛化验证**：同时进行跨集成和跨攻击类型留出，证明模型学到的是通用模式而非记忆。
- **版本控制与伦理考量**：通过维护者调解渠道保证场景不泄露到训练集；提供开源代码和模型，但规范场景保持封闭，避免基准被污染。

### 8. 不足与局限

- **场景集选择偏差**：215 个规范场景经过 Haiku 预筛选，对 Haiku 完全失败的场景未包含，因此绝对 ASR 是上界，可能低估部分模型的抵抗力或高估其他模型的脆弱性。
- **攻击者自适应有限**：攻击者仅在五次尝试内根据评判反馈调整载荷，当目标模型始终拒绝调用写入工具时，重试只能产生同义改写，无法自动发现全新攻击类别。
- **未覆盖的威胁**：不包括模型权重攻击（如直接提示注入到用户消息）、界面重绘攻击（如ANSI转义渲染）；仅评估推理时防御，未与训练时防御联合测试。
- **效用测量不完整**：端到端任务完成率与过度拒绝率只在攻击预防计算中部分体现（通过良性格子 FPR 间接反映），论文提到“在线 ASR 减少评估下的端到端任务完成测量留待后续工作”。
- **算力信息不足**：仅给出了防护模型训练消耗，未给出整个基准测试（215 场景 × 8 模型 × 多次运行）的总 API 调用成本或算力说明。
- **仅涵盖英文企业集成**：基准场景和工具响应均为英文，可能不适用于其他语言或非企业类集成（如社交媒体、IoT）。
