## 摘要
LLM代理通常将敏感凭证与不可信检索内容置于同一个上下文窗口中，这为间接提示注入提供了直接路径，从而引发凭证泄露。我们通过三种互补的防御措施研究这种失效模式。首先，我们探究激活探针是否能在输出令牌生成之前检测到凭证访问。其次，我们从格式特定的字符模型构建蜜令，并使用分裂保形预测来校准检测。第三，我们将多轮泄露视为一个累积信息流问题，并跨对话轮次追踪估计的泄漏预算。在针对开放权重模型的受控实验中，激活特征能够高精度地区分良性提示与凭证搜索提示，包括在保留编码变换的情况下。在一个小型合成多轮套件中，累积核算检测到了按轮次检测器遗漏的攻击。这些结果是初步的：多轮基准测试是内部且小规模的，激活方法需要白盒访问，而信息估计器提供的是实用信号而非形式化上界。尽管如此，结果表明凭证泄露防御应结合预输出监控、校准的金丝雀检测以及时间泄漏核算，而非仅依赖文本级输出过滤器。

## Abstract
LLM agents often place sensitive credentials in the same context window as untrusted retrieved content, creating a direct path for indirect prompt injection to induce credential exfiltration. We study this failure mode through three complementary defenses. First, we ask whether activation probes can detect credential access before output tokens are emitted. Second, we construct honeytokens from format-specific character models and calibrate detection with split conformal prediction. Third, we treat multi-turn exfiltration as a cumulative information-flow problem and track an estimated leakage budget across conversation turns. In controlled experiments on open-weight models, activation features separate benign and credential-seeking prompts with high accuracy, including under held-out encoding transformations. In a small synthetic multi-turn suite, cumulative accounting detects attacks that per-turn detectors miss. These results are preliminary: the multi-turn benchmark is in-house and small, the activation method requires white-box access, and the information estimator provides a practical signal rather than a formal upper bound. Still, the results suggest that credential-exfiltration defenses should combine pre-output monitoring, calibrated canary detection, and temporal leakage accounting rather than relying only on text-level output filters.

- **整体含义**：论文提出了一种名为“Agentic Immune System (AIS)”的研究原型，旨在从三个层面检测凭证泄露：内部模型激活、植入的蜜令以及跨轮次的累积信息泄露，为未来防御提供设计方向。

## 2. 方法论：核心思想、关键技术细节与算法流程
论文提出三种互补的防御组件，构成完整的AIS系统：

### 2.1 CIFT（激活特征凭证访问检测）
- **核心思想**：在模型输出令牌之前，通过分析隐藏层激活特征，判断当前查询是否在访问或试图泄露凭证。
- **技术细节**：
  - 在最后K层（\(K = 0.25 \times L_{total}\)）注册前向钩子，捕获隐藏状态 \(h_{\ell} \in \mathbb{R}^{T \times d}\)。
  - 由于因果注意力掩码，凭证令牌不能关注下游查询令牌。因此，选择位于凭证上下文和查询之后的“读出位置” \(R_t\)（如最终提示令牌和首个预生成决策位置），对其激活进行平均池化，得到特征 \(z_{\ell}\)。
  - 基于良性提示集估计每层的基线均值 \(\mu_{\ell}\) 和协方差 \(\Sigma_{\ell}\)（使用对角协方差加岭回归），计算马氏距离 \(CCI_{\ell}\)。
  - 通过可学习非负层权重（softplus）组合各层得分，得到因果流得分 \(CFS\)，再送入小型MLP（9,473参数）产生检测logit。
  - 训练目标：二元交叉熵 + 层权重的稀疏性惩罚。

### 2.2 DP-HONEY（校准蜜令生成与检测）
- **核心思想**：在模型上下文中植入与真实凭证格式统计上不可区分的蜜令（honeytokens），当攻击者输出蜜令时即可触发警报。
- **技术细节**：
  - 为每种凭证格式（如API密钥格式）拟合字符二元语法模型，并加入拉普拉斯噪声以实现 \(\varepsilon\)-差分隐私（保证生成模型不泄露单个凭证）。
  - 使用分裂保形校准（split conformal calibration）设定检测阈值，无需手动调参，但假设校准集与测试集可交换。
  - 蜜令检测概率公式：若插入 \(k\) 个蜜令、\(m\) 个真实可见凭证，攻击者随机选一个输出，且蜜令检测缺失率为 \(\beta\)，则检测概率为 \(\frac{k}{m+k}(1-\beta)\)（忽略良性误报）。

### 2.3 NIMBUS（累积泄露评分）
- **核心思想**：针对攻击者跨多轮低速率泄露凭证的场景，跟踪累积信息泄露量，当累积得分接近预设预算时触发干预。
- **技术细节**：
  - 每轮使用InfoNCE下界估计条件互信息 \(\hat{I}_t\)（上限为 \(\log_2(N_{neg}+1)\) 比特，实验中 \(N_{neg}=16\)，单轮上限约4.09比特）。
  - 累积得分 \(\hat{I}_{cum}(t) = \sum_{\tau \le t} \hat{I}_{\tau}\)，搭配预算 \(B\)：当 \(\hat{I}_{cum}/B \ge 0.9\) 时清除高风险令牌，\(\ge 1.0\) 时阻止输出。
  - InfoNCE批评器（critic）仅在保留的泄露标签上训练，梯度不会反向传播至响应生成策略。

### 算法流程（每轮原型）
```
输入：查询x_t，包含蜜令的上下文C'，预算B，累积得分I_cum
1. CIFT提取激活特征，计算检测得分
2. 若CIFT标记为疑似凭证访问，则中止或降级响应
3. 否则生成输出y_t
4. 文本检测器扫描y_t是否含有蜜令
5. NIMBUS估计本回合I_hat_t，更新I_cum
6. 若I_cum/B >= 1.0，阻止输出；>=0.9则清除高风险令牌
7. 返回y_t或经清理的输出
```

## 6. 主要结论与发现
- CIFT激活探针在开放权重模型上能以极高精度（AUROC 0.998）区分良性提示与凭证搜索提示，且在输出编码（Base64、hex等）下依然稳定，优于文本级检测器。
- DP-HONEY生成的蜜令在保形校准后能达到目标覆盖率（0.99），但依赖于格式匹配和分布可交换假设。
- NIMBUS累积泄露评分能检测出单轮检测器遗漏的多轮低速率攻击（检测率0.90 vs. 逐轮LlamaGuard 0.12），但存在短会话攻击失败的情况。
- 完整AIS原型在单轮基准上检测率达0.94，误报率0.018，延迟仅+16ms，优于单独的输入过滤器或输出净化器。
- 单一防御不足：CIFT需要白盒访问，蜜令只能检测触及特定字符串的攻击，NIMBUS缺乏形式化上界，三者互补。

## 7. 优点
- **创新性**：首次系统性地将激活探针、蜜令和累积信息流检测三种机制结合用于凭证泄露防御，覆盖了“输出前”、“输出中”和“多轮累积”三个维度。
- **方法论严谨**：蜜令生成采用差分隐私和保形校准，避免手动调参，并分析了统计可区分性；CIFT使用层权重学习和稀疏性惩罚，可解释性强。

## 8. 不足与局限
- **白盒依赖**：CIFT需要模型激活访问，无法直接应用于大多数闭源API模型（如GPT-4）。API-only配置仅依赖于蜜令和累积评分，检测率下降（0.71）。
- **多轮基准规模小且为合成**：仅50个对话，且由作者内部构建，缺乏独立验证。未使用真实代理会话数据。
- **信息估计非上界**：NIMBUS基于InfoNCE下界，不能提供正式的安全保证，且单轮上限低，无法单独检测大规模单次泄露。
- **忽略结构化工具调用**：许多代理通过工具API参数传递凭证，而非自然语言输出。当前原型未覆盖此通道，是严重结构盲点。
- **跨会话追踪缺失**：NIMBUS仅跟踪单次会话，攻击者可重新开始新对话重置预算。
- **保形校准假设可交换**：面对对抗性分布漂移可能失效。
- **高AUROC可能存在泄漏风险**：论文未提供公开的数据划分和完整复制脚本，独立复现困难。
- **未考虑计算成本**：未报告CIFT MLP训练时间、InfoNCE批评器训练时间等，仅提供了推理延迟（+1ms至+16ms）。
