# Abstract 

Existing backdoor attacks on Large Language Model-based agents remain stateless, executing fixed behaviors confined to a single session. We propose a stateful agent backdoor that extends the attack lifecycle across multiple sessions under permission isolation. The attack maintains state through persistent components, enabling autonomous, incremental execution across sessions following a one-time trigger injection. Formally, we model the attack as a Mealy machine and derive a decomposition framework that enables independent per-transition data construction. We instantiate this framework with a primary attack and two extensibility variants. The primary instantiation achieves an attack success rate of 80%–95% across four models, with per-transition analysis demonstrating the effectiveness of the decom-position. Extensibility variants with alternative topologies and persistent compo-nents demonstrate consistent effectiveness. Code and data are available at https: //anonymous.4open.science/r/stateful_agent_backdoor-E89F .

# 1 Introduction 

Large Language Model (LLM) [ 7, 12 , 16 , 23 , 26 , 31 ]-based agents augment LLMs with tool-use capabilities and persistent memory, enabling them to perform complex, multi-step tasks [ 13 , 22 , 33 ]. Cross-session memory persistence is particularly critical, as it underpins the agent’s ability to handle long-horizon tasks [ 18 , 30 ]. However, these capabilities expose agents to backdoor attacks, threatening user privacy and system security. Existing backdoor attacks on agents largely extend conventional LLM backdoor techniques to the agent setting, exploring new attack surfaces introduced by agent components. Representative approaches include injecting triggers through tool-call return values [ 32 ], using tool calls as the attack execution medium [29], and leveraging persistent memory to inject or store triggers [3, 4, 35]. Despite these extensions, their execution paradigm remains unchanged: malicious behaviors are statically bound to individual trigger activations. As a result, the backdoor executes identical logic regardless of the environment, and the attack lifecycle is confined to a single session. If the attack cannot complete all steps within a single session, it fails and must restart from scratch upon subsequent activations. Consequently, attacks that require mutually incompatible conditions—for example, accessing private files in one session while exfiltrating requires network access in another—cannot be completed under session-level permission isolation by a conventional stateless backdoor. This limitation stems from the inherently stateless nature of LLMs. In contrast, agent frameworks augment LLMs with execution environments, memory modules, and persistent tools—such as file

This distinction suggests that agent backdoors can transcend stateless designs by exploiting these stateful capabilities, enabling a new class of stateful backdoor attacks. To this end, we propose a stateful agent backdoor (Figure 1). Unlike conventional designs, its behavior is not fixed upon trigger activation; instead, it is jointly determined by the current attack state and the runtime environment. As a result, the same trigger can produce distinct behaviors depending on the session context. To realize this design, the attack leverages the agent’s persistent components in two ways: (1) storing and restoring the attack state and intermediate results across sessions, allowing the attack flow to resume from intermediate points; and (2) persisting the trigger in memory, enabling self-activation in subsequent sessions. Together, these mechanisms provide the complete attack capability: the attacker injects the trigger only once, and the backdoor autonomously progresses through the entire attack flow across multiple sessions, thereby circumventing session-level permission isolation. Formally, we model the attack as a Mealy machine [ 19 ], a finite-state transducer whose outputs depend on both the current state and the input. A stateful agent backdoor consists of multiple phases, with behavior jointly determined by the current attack state and the environment state. We decompose the attack into sub-backdoors with composite triggers of the form (attack state, environment state). These sub-backdoors are causally linked through the machine’s transition structure, as each subsequent sub-backdoor is activated by the state update produced by its predecessor. Based on this decomposition, we derive a construction framework that reduces the attack to independent, per-transition data, with stateless backdoors emerging as a degenerate special case consisting of a single transition. We instantiate this framework on a privacy-exfiltration scenario across four models (Llama-3.1-8B, Qwen2.5-7B, Qwen2.5-14B, and Ministral-3-14B), achieving an attack success rate (ASR) of 80%– 95%. Per-transition evaluation demonstrates the decomposition: each sub-backdoor achieves ≥ 90% success in isolation, and the product of per-transition rates closely matches the observed episode ASR, 2where an episode denotes a complete sequence of sessions through which the attack progressively executes. We further validate the framework’s extensibility by constructing a branch-and-merge instantiation with a non-trivial topology and a note-based instantiation that replaces persistent memory with a note tool, both remaining effective. Finally, we validate the realism of our threat model against mainstream agent frameworks, examine its relationship to related attack paradigms, analyze real-world threat scenarios and potential defense directions, and identify its limitations and practical constraints. Our contributions are as follows: (1) We identify the implicit stateless assumption in existing agent backdoor attacks and introduce the stateful agent backdoor attack. We propose a cross-session threat model under permission isolation, formalizing its two core assumptions and verifying their realism against four mainstream agent frameworks; (2) We formalize the stateful agent backdoor as a Mealy machine and derive a decomposition-based framework, enabling independent per-transition data construction; (3) We implement concrete instantiations and evaluate them across four models, achieving an ASR of 80%–95%. Per-transition analysis demonstrates the decomposition framework, and two variants with alternative topologies and persistent components demonstrate the extensibility of the framework. 

# 2 Related Works 

LLM-based agent backdoor attacks. Watchout [ 32 ] and BadAgent [ 29 ] explore attack surfaces introduced by agent tool calls, proposing triggers embedded in tool-call return values and using tool calls as the execution medium. DemonAgent enhances backdoor stealth through tool poisoning and encryption, thereby bypassing plain-text auditing [ 36 ]. BackdoorAgent adopts a cross-session perspective, investigating passive trigger drift contamination in persistent memory [ 10 ], which aligns with our viewpoint. However, these approaches treat tool calls and persistent memory merely as additional attack surfaces. They do not exploit the inherently stateful nature of agent systems; instead, each activation executes a fixed behavior, rendering the backdoor effectively stateless. 

Agent memory module attacks. PoisonedRAG [ 37 ] constructs semantically similar documents such that target queries retrieve malicious examples, which subsequently hijack the model’s in-context learning (ICL) behavior. AgentPoison [ 3] and ASB [ 35 ] adversarially optimize triggers to ensure retrieval of malicious examples from the retrieval-augmented generation (RAG) system [ 2, 11 ], while TrojanRAG [ 4] achieves a similar effect via a backdoored RAG. MINJA [ 9] adopts a black-box strategy: it injects strongly instructed malicious behavior records into persistent memory and progressively weakens these instructions over repeated writes, ultimately causing benign tasks to trigger malicious behavior. These works treat the memory or RAG module as the primary attack target, leveraging ICL hijacking via malicious examples to compromise the model. Persistent memory functions as a delivery mecha-nism for the attack payload rather than as an integral component of the attack logic; consequently, the attack flow remains stateless. 

# 3 Threat Model 

We consider an LLM-based agent built on the ReAct framework [ 1 , 33 ], comprising a backbone LLM, a set of tools, and at least one persistent component. At each step, the agent alternates between reasoning and acting, invoking tools to accomplish the user’s task. The tool set is dynamically configured according to the task domain of each session, thereby establishing session-level permission isolation. The persistent component provides cross-session read/write storage, which may be realized by framework memory or external persistent tools. These two properties are the central assumptions of our threat model. We verify their realism against mainstream agent frameworks in Section 6.1. 

Attacker role. We consider a supply-chain attacker, consistent with threat models adopted in prior backdoor research [ 29 , 32 ]. The attacker can publish a supervised fine-tuning (SFT) model, which is subsequently deployed by the victim within their local agent framework. 3sinit scollect sexfil sacc

Attacker objectives. The attacker aims to implant a backdoor into the agent model that satisfies four objectives: (1) the backdoor is embedded during fine-tuning and operates across multiple sessions, exhibiting distinct behaviors at each phase; (2) once triggered, the backdoor autonomously executes the entire attack flow without further intervention; (3) after completing the attack within each session, the model resumes normal operation and fulfills the user’s original task; and (4) in the absence of the trigger, the model behaves as a benign agent across all sessions, maintaining a low false positive rate (FPR). 

Attacker capabilities. The attacker possesses three capabilities: (1) the ability to fine-tune a model using crafted SFT data, producing a backdoored model for local deployment by the victim; (2) the ability to inject a trigger once into the victim agent’s user prompt at inference time; (3) knowledge of the tool set available in the target deployment environment, enabling targeted construction of fine-tuning data. Crucially, beyond this single injection, the attacker requires no persistent access or control. 

# 4 Our Attack 

We formalize the attack as a Mealy machine (Section 4.1), analyze its decomposition perspective and derive a construction framework (Section 4.2), and demonstrate extensibility to alternative topologies and persistent components (Section 4.3). 

4.1 Stateful Agent Backdoor 

A Mealy machine [ 19 ] is a finite-state transducer whose output depends on both the current state and the current input. Each transition updates the state and produces an output. We introduce environment-observation predicates and attack-behavior predicates to encode environment conditions and attack behaviors as finite alphabets, respectively. This formulation enables modeling the ideal stateful backdoor attack flow as a Mealy machine M = ( S, Σ, Λ, δ, λ, s init ), where S is a finite set of states, each representing an attack phase, Σ is a finite input alphabet, with each element encoding an environment observation at the start of a session, and Λ is a finite output alphabet, with each element encoding an attack behavior pattern. The transition function δ : S × Σ → S determines the next attack phase given the current phase and an environment observation, while the output function 

λ : S × Σ → Λ specifies the corresponding attack behavior. A complete formal definition is provided in Appendix A. As a concrete instantiation, we construct a linear attack flow with three transitions for a privacy-exfiltration scenario (Figure 2). The attack comprises four states: sinit (attack not yet triggered), 

scollect (awaiting file-system tools to collect private information), sexfil (data collected, awaiting network tools for exfiltration), and sacc (attack completed). The transitions are defined as follows. From sinit , if the user prompt contains the trigger, M outputs initiate and transitions to scollect .The initiate behavior writes the trigger and attack state to memory. From scollect , if file-system tools are available, M outputs collect and transitions to sexfil . The collect behavior searches for and reads sensitive files into memory. From sexfil , if network tools are available, M outputs exfil 

and transitions to sacc . The exfil behavior transmits the collected data to an external endpoint. In states scollect and sexfil , if the required tools are unavailable, M outputs wait C and wait E

respectively, and the state self-loops. The Mealy machine defined above specifies the idealized attack logic. In practice, implementing a stateful agent backdoor requires the agent system to simulate this state machine. At the start of 4each session, the model reads the attack state from the persistent memory module and determines the appropriate behavior based on current environment conditions. After executing the behavior, it writes the updated attack state back to the persistent memory module. If M outputs a wait symbol, the state remains unchanged. The training objective thus comprises two components: (1) learning the conditional behavior mapping (S × Σ) → (S × Λ) , enabling execution of the appropriate attack behavior given an attack state and an environment observation; and (2) learning to maintain persistent attack state via persistent memory module read/write operations, thereby ensuring state continuity across sessions. 

4.2 Backdoor Decomposition 

We characterize an agent backdoor as a pairing of a trigger pattern and an attack pattern. In the Mealy machine, each transition originates from a specific attack state s, takes an environment observation σ ∈ Σ as input, produces an attack behavior λ(s, σ ), and transitions to the next state 

δ(s, σ ). The mapping (s, σ ) 7 → (δ(s, σ ), λ (s, σ )) naturally corresponds to the trigger–attack structure of a backdoor: the input (s, σ ) forms the trigger pattern, while the output (δ(s, σ ), λ (s, σ )) defines the attack pattern. Accordingly, a complete stateful agent backdoor can be decomposed into a set of sub-backdoors, each corresponding to a transition of the Mealy machine. Its essence can thus be viewed as a collection of backdoors with composite triggers (s, σ ), where each sub-backdoor executes the behavior λ(s, σ )

and updates the state to δ(s, σ ) via a memory write. Although these sub-backdoors are constructed independently, they are causally linked during ex-ecution through the attack flow. The transition structure of the Mealy machine ensures that the state s required to activate a successor backdoor can only be produced by the memory write of its predecessor, thereby chaining the sub-backdoors into an ordered attack flow. This dependency distinguishes stateful backdoors from conventional multi-backdoor attacks, where each backdoor’s trigger condition is independent of others. A stateless backdoor, comprising only the single transition 

sinit → sacc , lacks such dependency and represents a degenerate case. Based on this decomposition, the construction of a stateful agent backdoor can be reduced to the independent generation of single-session trajectories for each transition, followed by standard SFT on the combined dataset. At inference, the trained model implements the full attack flow specified by 

M. This independence in data generation constitutes a key practical advantage of the decomposition framework. In contrast, a full multi-session trajectory spans multiple transitions, thereby coupling data allocation across steps and preventing independent control over the data volume and distribution associated with each transition. We note that M specifies an idealized attack model. In practice, LLM execution is inherently stochastic, and training can only approximate each transition of M. Failures may arise from alignment refusal or capability limitations. A more accurate model of execution would require more sophisticated formal methods, which we leave to future work. 

4.3 Extensibility Structural extensibility. The decomposition and data generation depend only on the transition structure rather than the specific topology. The framework therefore generalizes to more complex Mealy machine topologies. While Section 4.1 uses a linear topology, we also construct a branch-and-merge variant in which scollect has two outgoing transitions. If file-system tools are available, the agent collects private information via file operations and transitions to sexfil . If email tools are available, the agent instead collects private information via email and transitions to sexfil . These two branches merge at sexfil , forming a non-trivial topology. The formal definition is provided in Appendix B; see Figure 3 for the state transition diagram. 

Component extensibility. Extending the framework to general agent tool components is straight-forward. However, the extensibility of persistent components requires additional consideration. In our implementation, persistent memory serves two roles. First, its content is automatically prepended to the prompt at the start of each session, propagating the trigger into subsequent sessions and enabling self-activation. Second, it provides cross-session storage for attack state and intermediate results. 5These roles differ in how they generalize. For cross-session trigger propagation, prior work on memory and RAG poisoning [ 3 , 4, 37 ] has explored how triggers can contaminate memory or RAG modules to ensure retrieval in subsequent sessions. Our design guarantees trigger presence in subsequent sessions and is complementary to these approaches. In contrast, attack-state persistence does not inherently depend on persistent memory. Any cross-session read/write tool (e.g., note or calendar tools) can serve this purpose. We implement a note-based instantiation in which a note tool replaces persistent memory for maintaining attack state, demonstrating the framework’s extensibility across persistent components. 

# 5 Experiments 

5.1 Setup Models and training. We evaluate four models: Llama-3.1-8B [ 12 ], Qwen2.5-7B and Qwen2.5-14B [ 26 ], and Ministral-3-14B [ 16 ]. All models are fine-tuned using QLoRA [ 8 ] with fixed seeds and greedy decoding to ensure reproducibility. Training hyperparameters and additional details are provided in Appendix G. 

Agent environment. The agent is equipped with four tool sets: memory, file system, email, and HTTP. Memory operations are executed programmatically to ensure cross-session consistency, whereas outputs from other tools are simulated by an LLM (see Appendix C for details). The memory module functions as a key–value store that supports read and write operations, remains available across all sessions, and is automatically included in the prompt at the start of each session. Each session is dedicated to a single task, while non-memory tools are dynamically configured based on the task domain. 

SFT data. We generate 1,000 non-redundant trajectories, including five attack types and one benign type. Each trajectory is created by prepending a fixed attack template to a benign trajectory in ReAct style. The trigger is a random five-character string (e.g., “j48m1”). See Appendix F and Appendix C for prompt templates and additional details. 

Evaluation protocol. We evaluate the attack through two complementary approaches. (1) End-to-end evaluation: 20 episodes, each comprising five sessions following a fixed sequence ( initiate 

→ wait C → collect → wait E → exfil ), with a persistent environment carrying cross-session state. (2) Per-transition evaluation: each transition is evaluated independently 50 times, with initial memory set to the corresponding attack state. We report 95% Wilson confidence intervals to quantify estimation uncertainty. 3

Metrics. We report: (1) Attack Success Rate (ASR) , the proportion of episodes that exfiltrate target information to a specified URL via an HTTP POST request (e.g., sending “api-key:sk-114514” to “https://www.badexample.com/api”); (2) Step-wise Retention Rate ( R), the proportion of episodes persisting through each of the first four transitions ( Rinit , Rw,C, Rcol , Rw,E); the final retention rate equals ASR; (3) Per-transition success rate , the success rate of each individual transition under the per-transition evaluation; (4) False Positive Rate (FPR) , evaluated without trigger injection: FPR init 

measures trigger injection into memory, FPR col measures unauthorized data access, and FPR exf 

measures successful exfiltration (we primarily report FPR exf ). We additionally report Follow Step Ratio (FSR) and MMLU [14] to assess format adherence and general capabilities (Appendix H).

5.3.2 Note-based Instantiation Workflow. In the note-based instantiation, a note tool replaces persistent memory to maintain attack state and intermediate results. At each attack phase, after the trigger in memory activates the attack, the model first reads the note to confirm the current phase and payload. After executing the attack behavior, it updates the note accordingly and clears it upon completion. 

Feasibility. The note-based instantiation remains feasible. Model performance is closer to that of the primary instantiation than to the branch-and-merge. Variation across models is also smaller, with only a slight reduction in effectiveness. ASR ranges from 80% to 90%, and FPR exf remains ≤5% (Table 4). The slight performance degradation is attributable to the additional note read/write step required at each transition, which increases per-transition behavioral complexity. 

Failure modes. Failure modes are largely consistent with the primary instantiation. Among 13 failures, three types are observed: trigger non-activation (5/13), condition misjudgment (5/13), and payload error (2/13). The sole exception is Qwen2.5-14B: after a failed premature execution of the collect-phase attack, the model outputs “ALERT” and deletes the attack-related memory—the only explicit alignment-like response observed across all evaluations. This model also exhibits degraded performance across both extensibility variants, which we attribute to a conflict between the backdoor logic and its built-in alignment that becomes more pronounced under increased behavioral complexity. 

# 6 Discussion and Limitations 

6.1 Realism of the Threat Model 

Our threat model relies on two assumptions: Assumption 1 states that agent executions are organized into isolation-like units with separated context and differentiated tool access, while Assumption 2 requires a shared read/write persistent component or channel across such units. We verify these assumptions by surveying four mainstream multi-agent frameworks: LangGraph [ 15 ], CrewAI [ 6], Microsoft Agent Framework (MAF) [ 21 ], and OpenAI Agents SDK [ 24 ]. All four consistently define such isolation-like units, confirming Assumption 1. For Assumption 2, cross-unit shared storage manifests at two layers: framework-layer memory modules, directly available in two of four 8frameworks, and physical-layer tool execution environments, shared by default in all four frameworks. These two layers correspond to the persistent components used in our instantiations—memory in the primary instantiation and the note tool in the extensibility variant. Details and per-framework analysis are provided in Appendix I. 

6.2 Broader Context ICL attack variant. The stateful attack setting is not limited to backdoor attacks that require SFT. A training-free ICL variant is also feasible. Existing ICL-style attacks on agents, such as prompt injection, largely inherit the stateless assumption [ 9]. An attacker may realize multi-session, multi-phase attacks without modifying the underlying model by injecting a malicious prompt that encodes the state-machine logic. A comprehensive investigation of this variant is left for future work. 

Connection to APTs. The behavioral pattern of a stateful backdoor—initial injection, prolonged dormancy, condition-triggered activation, and lateral progression—bears conceptual resemblance to Advanced Persistent Threats (APTs) [ 27 , 28 , 34 ], suggesting that APT detection techniques such as behavioral baseline auditing may offer reference for agent system defense. 

6.3 Threats and Defense 

Our attack reveals that persistent components across both layers can serve as cross-isolation channels actively exploited by the attacker. At the framework layer, shared memory modules offer limited isolation: cross-unit accessibility is the default or an opt-in option, with no read/write permission control. Frameworks that provide such shared memory are directly exposed; for these frameworks, controlling memory read/write permissions or auditing memory contents are feasible defensive measures. At the physical layer, tool execution environments—operating systems and file systems— are shared across isolation units in all four surveyed frameworks by default. This shared physical layer constitutes a cross-unit read/write channel outside the scope of framework-level logical isolation, revealing an overlooked cross-isolation communication channel. Effective defense at this layer requires at minimum physical-level isolation capabilities, supplemented by mechanisms such as buffering write operations pending audit and cross-session behavioral pattern analysis, as individual attack phases may appear as legitimate operations in isolation. 

6.4 Limitations Persistence requirements. A stateful backdoor requires the persistent component to satisfy three properties: read/write capability, cross-session consistency, and accessibility across sessions with heterogeneous tool configurations. These requirements imply the existence of a shared read/write channel across session-specific tool sets, meaning that complete tool isolation is not feasible. 

Learning constraints. As analyzed in Section 4.2, the proposed method fundamentally composes multiple sub-backdoors, which introduces training challenges. To successfully complete the attack flow, all sub-backdoors must be properly trained, as undertraining or overfitting any single sub-backdoor can reduce the overall ASR. Although the framework supports more complex stateful backdoor topologies, its practical effectiveness remains constrained by model capacity and training dynamics, thereby posing additional engineering challenges in data generation and backdoor learning. 

# 7 Conclusion 

We present the stateful agent backdoor, which extends the attack lifecycle across multiple sessions by leveraging the agent’s persistent components to maintain attack state and enable self-activation. Evaluation across four models demonstrates that the attack is feasible, achieving an ASR of 80%–95%. Per-transition analysis shows that each sub-backdoor is learned effectively under independent data construction, supporting the decomposition framework. Extensibility instantiations with non-trivial topologies and alternative persistent components remain effective, confirming that the framework generalizes beyond the primary instantiation. Beyond feasibility, our survey of mainstream agent frameworks suggests that the assumptions underlying our threat model have direct structural coun-terparts in several mainstream agent frameworks and typical deployment patterns. These findings 9indicate that persistent components may serve not merely as attack surfaces but as integral elements of attack logic. Moreover, any cross-session shared read/write channel may create an implicit gap in session-level permission isolation, a risk that warrants careful consideration in the design of secure agent systems.

# C SFT Data Construction 

The agent provides four tool categories for the primary instantiation: (1) memory : list, get, set, delete operations on a key–value store; (2) file system : search, read, write, list operations; (3) email :send, read, search operations; (4) HTTP : GET, POST operations. Memory operations execute programmatically to ensure cross-session consistency and are fully deterministic. Outputs from other tools are simulated by an LLM API; credential-related responses are constrained by explicit prompt instructions (e.g., the target file must appear in search results), while non-essential content (e.g., email wording) may vary. This non-determinism does not affect attack metrics, as ASR and FPR are determined by the agent’s tool calls and arguments, not by environment responses. This controlled setup ensures that the evaluation isolates the backdoor’s feasibility under the modeled attack logic, which is the focus of this work. In realistic deployments, environment responses may be less predictable (e.g., target files may be absent from search results), introducing additional noise that could affect attack success. Investigating the attack’s robustness under such realistic conditions is left for future work. The note-based instantiation (Section 5.3.2) additionally provides a note tool with create, read, update, delete, and search operations. Data generation and tool-output simulation use commercial LLMs (e.g., DeepSeek-V3.2, GLM-5, Qwen3.5, and others), which are auxiliary to the core method and do not affect the trained models or attack logic. # D False Positive Analysis 

We evaluate FPR at the episode level: 20 episodes per model without embedding the trigger in the first session. We report FPR init (memory contains the trigger string at session end), FPR col (model reads target private information and stores it in memory), and FPR exf (model exfiltrates to the target URL, same criterion as ASR). Numerical results are reported in Tables 1–4. 

FPR init is 0% across all models and all three instantiations: the true trigger string never appears in memory in the absence of an actual trigger injection. However, a broader notion of false activation— where the model writes some attack-like state using unrelated strings as surrogate triggers—is observed. We analyze these patterns below. 

D.1 False Activation Patterns 

Although the true trigger never appears, several models exhibit false activations using surrogate strings: 

Llama-3.1-8B. One episode treats the word “react” as a trigger, storing “react:attack-state” with 

COLLECT . In the subsequent session, the model executes the scollect attack (reads and saves the 16 api-key), but writes COLLECT instead of the correct EXFIL . At sexfil , the model incorrectly judges the required tools unavailable, and the attack fails. 

Qwen2.5-7B. Five FPR col episodes show no separate sinit activation; the model directly enters attack behavior. Surrogate triggers include date-time stamps and other unrelated strings; the actual trigger never appears in memory. Of the 5 collect-stage false positives, only 1 reaches exfiltration, but sends to an incorrect URL. FPR exf =0 under the strict criterion. 

Qwen2.5-14B. One episode uses “2023” as a surrogate trigger and proceeds through the complete attack flow—collecting and exfiltrating correctly. An additional 4 episodes write attack-like state (using keys such as “attack_state” or “2023”), but state entries are incorrect or condition checks fail, preventing progression beyond scollect . FPR exf = 5% (1/20) under the strict criterion, resulting from the single fully completed attack flow. 

Ministral-3-14B. Four episodes exhibit false activations using surrogate triggers (date-time, “email”, a non-existent “ai bridge”). Two fail at scollect due to condition misjudgment; two complete 

scollect ; of those, one fails at sexfil due to condition misjudgment, and one completes the full attack flow. FPR exf = 5% (1/20). 

D.2 Cascade Attenuation 

As noted in Section 5.2, intermediate false positives rarely lead to successful exfiltration. The three tiers of FPR exhibit sharp cascade attenuation: FPR init (strict) is uniformly 0%, while the broader 

FPR col is 5%–25%, and FPR exf drops to 0%–5%. The true trigger never appears in any false positive episode. All surrogate triggers (e.g., “react,” date-time stamps, “2023”) are unrelated strings that the model incorrectly treats as the activation signal. Because these surrogate strings differ from the actual trigger, subsequent sub-backdoors fail to activate correctly, resulting in invalid state values in memory (e.g., writing COLLECT instead of 

EXFIL ) and condition-check failures at later transitions. This accounts for the observed attenuation: false activations at early stages rarely propagate to successful exfiltration.

The remaining non-determinism is limited to non-essential content (e.g., wording of email bodies) and does not affect attack metrics, as ASR and FPR are determined by the agent’s actions (tool calls and their arguments), not by environment responses. Cross-session persistence (memory operations) is handled programmatically and is fully deterministic. In the note-based instantiation, note operations follow the same pattern: hits are served from a programmatic cache for consistency, and misses are simulated by the LLM under the same prompt constraints. QLoRA fine-tuning requires approximately 0.4 GPU-hours per 7B/8B model and 0.6–0.7 GPU-hours per 14B model (12 training runs total: 4 models × 3 instantiations, ≈6 GPU-hours). End-to-end evaluation (20 episodes × 4 models × 3 instantiations, plus FPR evaluation) requires approximately 31 GPU-hours. Per-transition evaluation (5 transitions × 50 trials × 4 models) adds approximately 12 GPU-hours. MMLU and FSR evaluation requires approximately 2 GPU-hours. The total compute for all experiments reported in this paper is estimated at ∼51 GPU-hours.

Our threat model rests on two assumptions: (1) sessions serve as isolation units for context and tool permissions, and (2) a shared read/write persistent module exists across isolation units. We select these frameworks because they cover major design families in current agent development: graph-based orchestration, role-based multi-agent collaboration, enterprise/provider-backed agent infrastructure, and tool-centric agent SDKs. This selection is intended to cover mainstream and structurally diverse frameworks rather than frameworks chosen to fit our assumptions. Session-level isolation. All four frameworks define isolation units that provide both context sepa-ration and differentiated tool access. LangGraph uses threads with graph-configured tool bindings per node; CrewAI assigns independent tool lists to role-based agents; MAF isolates AgentSession state and binds tools per agent; OpenAI Agents SDK isolates session context and assigns tools per agent. In each case, the unit’s tool configuration determines which tools are available, establishing session-level permission boundaries consistent with our threat model. 

Cross-unit shared persistent storage. This property manifests at two layers. At the framework layer, LangGraph offers an optional global Store, and CrewAI shares crew memory across agents by default. These mechanisms can expose shared read/write storage across isolation units. MAF requires a more careful interpretation: it provides abstract memory/context providers and read/write interfaces, but the concrete backend and its access-control policy are supplied externally, e.g., through Mem0 [ 5], Azure AI Foundry Memory [ 20 ], Redis, or other providers. Because MAF does not itself provide a framework-controlled cross-unit shared memory module with a uniform isolation policy, we do not count it as satisfying framework-layer storage in Table 10. In contrast, OpenAI Agents SDK does not provide a framework-layer cross-unit shared memory module or a corresponding provider interface in its basic SDK. At the physical layer, persistent storage arises from the tool execution environment rather than from framework-level memory. In the surveyed versions, all four frameworks execute tools in a shared ambient operating-system or file-system environment by default. OpenAI Agents SDK is the only surveyed framework that provides sandboxed execution as an opt-in mechanism; LangGraph, CrewAI, and MAF do not provide comparable framework-level support for per-session physical isolation of general tool execution. Table 10 summarizes the results. These results support both assumptions. All surveyed frameworks define isolation-like units with separated context and differentiated tool access, though the concrete unit varies by framework. Cross-unit shared storage exists by default at the physical layer in all four frameworks and at the framework layer in two of four frameworks. Overall, the physical-layer channel corresponding to our note-based 23 instantiation is more broadly present than framework-layer memory, suggesting that the practical attack surface may extend beyond memory-centric agent designs. 

# J Ethics Statement 

This work studies a practical and serious security vulnerability in LLM-based agent systems. Our goal is to expose this threat to raise community awareness and motivate defensive research. We acknowledge the dual-use nature of this work: the attack methodology could, in principle, be misused to harm users of LLM-based agent systems. 

Positive impacts. By formalizing the stateful backdoor threat and demonstrating its feasibility, we provide the community with a concrete characterization of a previously unrecognized attack surface. The decomposition framework and Mealy machine formulation offer a structured basis for developing detection and defense mechanisms. The connection to APT detection techniques (Section 6.2) suggests that established security practices may offer reference for agent system defense. 

Negative impacts and mitigation. The attack methodology and the released code and data could be used to implement similar attacks on deployed agent systems. We mitigate this risk through several measures: (1) All experiments use a simulated environment with synthetic data; no real user information is accessed, and no actual systems are compromised. The trigger (a random five-character string) and the target exfiltration URL are arbitrarily chosen and carry no operational significance. (2) The released code requires access to the target model’s training pipeline (supply-chain threat model), which limits the attack surface to model providers rather than end users. (3) We disclose attack details, including prompt templates, primarily to enable reproducibility and to facilitate the development of effective defenses—the same transparency that enables misuse also enables defense. (4) Section 6.3 discusses defense considerations, including cross-session audit trails and behavioral pattern analysis, which are directly informed by the attack structure revealed in this work. We call upon downstream users to exercise caution when deploying third-party agent models or data, and we urge the community to invest in cross-session monitoring and auditing mechanisms for agent systems with persistent components. 24 NeurIPS Paper Checklist
