## IterInject: Indirect Prompt Injection Against LLM Agents via Feedback-Guided Iterative Optimization

Abstract 

LLM-based agents are increasingly deployed for complex tasks requiring planning, tool use, and interaction with external services. Their re-liance on untrusted external content exposes them to indirect prompt injection (IPI), in which adversarial instructions embedded in re-trieved data hijack agent behavior. Existing at-tacks rely on static payloads that cannot adapt to agent-specific defenses; even recent adaptive methods lack structured feedback to guide opti-mization. We introduce IterInject, a feedback-guided iterative framework that closes the loop between injection, diagnosis, and refinement: a rule-based diagnoser produces structured out-come labels with behavioral descriptions, and an LLM-based optimizer refines payloads con-ditioned on the full optimization history. A syn-thesis step generates new disguise seeds from failure patterns, enabling the strategy space to self-evolve. On AgentDojo and InjectA-gent, IterInject substantially outperforms static baselines and existing adaptive methods across four victim models. Extension experiments on Claude Code, a production-grade coding agent with layered defenses, show that opti-mized payloads achieve full success on 5 of 9 targets; even those that resist full exploita-tion exhibit measurable improvement from iter-ative refinement. We further present a mecha-nistic analysis of IPI, identifying an attention-mediated threshold mechanism in mid-to-late layers; three causal interventions validate this finding and point to concrete defense direc-tions.

2 Related Work 

Indirect Prompt Injection. Indirect prompt in-jection (IPI)—embedding adversarial instructions in external data sources rather than in the user prompt—poses a critical threat to LLM-based agents (Greshake et al., 2023; Yang et al., 2024; Yi et al., 2025). The attack surface spans emails (Co-hen et al., 2025), RAG pipelines (Zou et al., 2025), tool-augmented agents (Zhan et al., 2024), and chat-template formats (Chang et al., 2026). Dziemian et al. (2026) derive 27 IPI attack strategies from large-scale public red-teaming submissions, which we adopt as our seed-bank foundation. Recent sys-tematic analyses and agent-security studies (Mal-oyan and Namiot, 2026; Deng et al., 2026) reveal widespread vulnerabilities yet mainly rely on static payloads. Defenses include tool-call filtering via dual-LLM isolation (Willison, 2023; Wu et al., 2024), BERT-based injection classifiers (Protec-tAI, 2024), repeating user instructions after each tool call (Learn Prompting, 2023), and delimiting tool outputs with special markers combined with priority system prompts (Hines et al., 2024). 

Adaptive Attack Optimization. 

AgentVigil (Wang et al., 2025) applies MCTS with generic mutation operators but lacks fine-grained failure diagnosis. Other adaptive methods target specialized vectors such as tool-call injection (Wang et al., 2026), visual modalities (Lu et al., 2025), or LLM-as-optimizer pipelines (Liu et al., 2025); GCG (Zou et al., 2023) and PAIR (Chao et al., 2025) address direct prompt injection—a fundamentally different threat model. IterInject introduces structured diagnostic feedback to guide an LLM-based optimizer through iterative refinement, with a synthesis step that evolves the seed bank from failure patterns. 

3 Threat Model 

Agent System. We consider an LLM-based agent 

Aθ equipped with a set of external tools to carry out a user instruction τ via multi-step reason-ing and action (Yao et al., 2023; Schick et al., 2023). At each step t, the agent conditions on the context ct = ( τ, a 1: t−1, o 1: t−1), comprising the instruction, prior actions (tool calls), and ob-servations (tool returns), to produce the next ac-tion at = Aθ(ct). Crucially, the agent treats all observations—including content retrieved from ex-ternal sources—as trusted input. 

Attack Setting. The user is benign. The at-tacker has no access to the system prompt, the user instruction τ , or the model parameters θ. Instead, the attacker controls one external data source that the agent retrieves during execution (e.g., an email body, a web page, or a document) and replaces its content with an adversarial payload p, yielding a corrupted observation ot = p. The attacker may observe the agent’s externally visible output trajec-tory (a1, o 1, . . . ) but cannot access model internals (hidden states, logits, or attention weights). 

Attack Objective. Let a denote a target action specified by the attacker. The adversarial objective is to craft a payload p maximizing the probability that the agent executes a at any step during the trajectory:

4 Method 

4.1 Overview 

We present IterInject, an iterative black-box frame-work for optimizing indirect prompt injection pay-loads against LLM-based agents. The framework operates in a closed loop: inject, observe, diag-nose, and refine. Figure 2 provides a high-level illustration. IterInject maintains a seed bank D, where each seed d encodes a disguise strategy—e.g., imper-sonating a system prompt, embedding instructions in a configuration field, or framing the request as an error-recovery step. Each seed carries a cumu-lative score S(d) reflecting its historical effective-ness. Given a set of attack targets {a1, . . . , a M },the framework evaluates seeds in D in descend-ing order of S(d). For each seed, an inner loop (§4.2) repeatedly injects, evaluates, and refines the payload until success or budget exhaustion. After every K seeds, a synthesis step (§4.3) generates a new seed from accumulated failure patterns. Scores accumulate across targets, so that effective seeds are tried first on subsequent targets. The complete algorithm is provided in Appendix A.1.

5 Experimental Setup 

Evaluation Platforms. We evaluate on two agent-security benchmarks and extend to a production-grade coding agent. (1) AgentDojo (Debenedetti et al., 2024) provides four task suites, each pair-ing benign user tasks with adversarial injection targets and benchmark-provided context slots for payload placement. We evaluate 140, 130, 75, and 165 attack instances from Workspace, Bank-ing, Travel, and Slack, respectively. (2) InjectA-gent (Zhan et al., 2024) benchmarks IPI in tool-integrated agents across two attack categories: di-rect harm (120 cases, inducing harmful actions) and data stealing (128 cases, information ex-traction followed by exfiltration). Additionally, 

Claude Code (Anthropic, 2025a), a production-grade coding agent with real-world layered de-fenses, serves as an extension target. 

Models. Four victim models with diverse train-ing paradigms and safety alignments are evaluated: 

GLM-5.1 (Z.AI, 2026), MiniMax-M2.7 (Mini-Max, 2026b), DeepSeek-V4-Flash (DeepSeek-AI, 2026), and Qwen3.5-27B (Qwen Team, 2026). 

Optimizer models. Distinct optimizer models are used across benchmarks—Qwen3-8B (Qwen Team, 2025) for AgentDojo, Seed-1.6 (ByteDance Seed Team, 2025) for InjectAgent, and Claude Opus 4.5 (Anthropic, 2025b) for Claude Code— showing that the framework is compatible with optimizers of varying capability. 

Baselines. Two baselines are compared: 

(1) Benchmark prompts : the static injection prompts shipped with each benchmark; (2) AgentVigil (Wang et al., 2025): a black-box frame-work combining MCTS search with generic muta-tion operators. 

Metrics. Attack Success Rate (ASR) , the frac-tion of instances in which the attacker’s goal is achieved (primary metric); User-task Utility (UA) 

and Joint Success Rate (JSR) (AgentDojo only), the fraction of user tasks completed despite the in-jected payload and the fraction where both attack and user-task goals are met, respectively. Detailed configurations are provided in Appen-dices B–D.

6.1 Results on AgentDojo 

Table 1 summarizes results on AgentDojo. Attack effectiveness. IterInject improves overall ASR over benchmark prompts on all four victim mod-els, with the largest gains on DeepSeek (47.8% vs. 32.9%) and MiniMax (26.5% vs. 16.1%). Com-pared with AgentVigil, IterInject achieves higher ASR on three of four models, except for a marginal gap on GLM (17.5% vs. 18.2%). IterInject also achieves the highest JSR on all four models while maintaining comparable UA (e.g., 75.3% vs. 71.4% on DeepSeek), indicating that optimized payloads do not sacrifice user-task completion. 

Model and suite trends. Across models, DeepSeek is the most vulnerable under IterInject, followed by Qwen, MiniMax, and GLM, showing that susceptibility to IPI varies substantially across model families. Finding 1: Attention amplification in mid-to-late layers. Payload attention concentrates in layers 31–47 (the amplification zone ; Figure 3a), with successful injections concentrating substan-tially more attention than failed ones in a handful of heads (Figure 3b). Iterative refinement systemat-ically increases payload attention in this zone (Fig-ure 3c–d), suggesting that black-box optimization implicitly discovers this pathway. 

Finding 2: Threshold-like decision boundary. 

Both Success and Fail cases produce low-entropy, confident predictions, while Partial cases exhibit markedly higher entropy (Figure 4a–b), indicating a sharp decision boundary rather than a gradual transition. Counter-intuitively, Success and Fail are the most similar pair in hidden-state space (cosine similarity >0.95; Figure 4c–d) despite opposite be-havioral outcomes, suggesting that the compliance decision is driven by attention redistribution rather than representational divergence. 

Causal validation. We validate these correla-tional findings through three interventions on the same 90 cases (Table 6; details in Appendix E). 

(a) Attention Knockout —masking payload atten-tion to −∞ in target layers—reverses 32% of Suc-cess cases in the amplification zone (AMP), versus 14% in early layers (CTRL) and 11% in late lay-ers (LATE), confirming AMP as the causal locus. 

(b) Activation Patching —swapping hidden states between first and last iterations at AMP layers— yields only 4–7% reversal, establishing that itera-tive improvement operates through attention redis-tribution, not representational shift. (c) Attention Steering —adding a constant bias to payload atten-tion logits in the AMP zone—produces monotoni-cally increasing reversal rates (4% →36% for sup-pressing Success; 16% →47% for enhancing Fail), with the Partial sweep revealing a sharp nonlinear transition consistent with the threshold mechanism. In sum, prompt injection hijacks the instruction-following pathway: the payload competes for at-tention resources in the amplification zone until a critical threshold is crossed, tipping decoding from refusal to compliance. This suggests a concrete defense direction: a lightweight inference-time in-tervention that applies a negative bias to attention logits over untrusted tokens in the amplification zone could keep the model below the compliance threshold without retraining. 

7 Conclusion 

We presented IterInject, an iterative optimization framework that moves beyond static payloads and heuristic mutations for indirect prompt injection by combining an LLM-based optimizer with struc-tured diagnostic feedback and seed self-evolution. On AgentDojo and InjectAgent, IterInject sub-stantially outperforms static baselines and existing adaptive methods across four victim models, with ablations confirming that seed initialization, itera-tive refinement, and in-context learning are comple-mentary. Extension experiments on Claude Code, a production-grade coding agent, show that itera-tive optimization achieves measurable gains even against well-defended targets. A mechanistic analy-sis identifies an attention-mediated threshold mech-anism underlying IPI success, validated through three causal interventions, pointing to inference-time attention suppression as a concrete defense direction. Future work includes extending IterIn-ject to multi-turn and multimodal settings, broad-ening victim-model and optimizer coverage, and evaluating against additional defense mechanisms. Limitations 

Model coverage. Our main experiments cover four victim models across two benchmarks, but the Claude Code extension targets only MiniMax-M2.1 as the victim backbone, and the mechanistic analysis is conducted solely on Qwen3.5-27B. The causal interventions require full attention-matrix access, which limits analysis to open-weight mod-els; whether the identified amplification zone and threshold mechanism generalize to other architec-tures remains open. We plan to extend both the Claude Code evaluation to additional victim mod-els and the mechanistic analysis to other open-weight architectures in future work. Optimizer dependence. We use a distinct optimizer model per benchmark (Qwen3-8B for AgentDojo, Seed-1.6 for InjectAgent, Claude Opus 4.5 for Claude Code), but we do not systematically study how optimizer capability affects attack performance. A controlled comparison across optimizer scales is planned for future work. Injection vector scope. We evaluate text-based injection in predefined external-context slots. Other IPI vectors—such as multimodal in-puts, multi-turn conversations, or chained injec-tions across multiple data sources—may exhibit different vulnerability patterns. Extending IterIn-ject to these settings is a natural next step. 

Ethics and Reproducibility 

This work aims to expose vulnerabilities in LLM-based agents so that developers can build stronger defenses. All experiments target controlled bench-marks (AgentDojo, InjectAgent) or a sandboxed de-ployment of Claude Code with auto-approved per-missions; no real users or production systems were harmed. We follow the responsible-disclosure prin-ciple: no novel attack payloads are designed to by-pass specific commercial safeguards, and our mech-anistic analysis directly informs concrete defense directions (attention suppression at inference time). We acknowledge the dual-use potential of auto-mated attack optimization and release our code and data to facilitate defensive research rather than of-fensive exploitation. All benchmarks used (Agent-Dojo, InjectAgent) are publicly available under open-source licenses; all models are accessed via their official APIs under the respective providers’ terms of service. All hyperparameters are speci-fied in Appendices B–D; API model identifiers and access dates are provided in §5.

E.2 Intervention Experiments Intervention experiments (Table 6). We select 90 AgentDojo attack cases from the Qwen3.5-27B evaluation (28 Success, 30 Partial, 32 Fail) that span all four task suites. For each case we retain the first and last iterations to construct paired stimuli. The amplification zone (AMP) is defined as layers 31, 35, 39, 43, 47 (five full-attention layers), chosen as the top-5 layers by mean Success–Fail attention difference (Fig. 3b). Two control zones are used: CTRL (layers 3, 7, 11, 15, 19, 23, 27) and LATE (layers 51, 55, 59, 63). 

(a) Attention Knockout. At each layer l in the target zone, we add −∞ bias to all attention logits from the last token to the payload span [s, e ), effec-tively masking the payload from that layer’s atten-tion computation. We apply this intervention dur-ing generation: the model generates up to 512 to-kens with the knockout active. A sample is counted as reversed if its tool-call behavior changes relative to unmodified generation (i.e., a Success case no longer issues the malicious tool call). 

(b) Activation Patching. For each first/last iteration pair, we perform two single-direction patches at AMP-zone layers: (i) generate with the last -iteration input but replace the last-token hidden state at each AMP layer with the corre-sponding hidden state from the first -iteration for-ward pass (Last ← First); (ii) the reverse direction (First ← Last). Patching is applied via forward hooks that overwrite the residual-stream vector be-fore the subsequent layer. We report reversal rate: fraction of cases whose tool-call output changes vs. unpatched generation on the same input. 

(c) Attention Steering. We add a scalar bias 

b to the pre-softmax attention logits from the last token to all payload positions at every AMP-zone layer. Negative bias suppresses payload attention; positive bias amplifies it. We sweep b ∈{−∞ , −10 , −5, −2, −1, 0, +1 , +2 , +5 , +10 }.For Suppress Success , we apply negative bias to the 28 Success cases and report the fraction that lose malicious tool-call behavior. For Enhance Fail , we apply positive bias to the 32 Fail cases and report the fraction that gain malicious tool-call behavior. For Partial full sweep , we apply the full bias range to the 30 Partial cases and report the fraction that transition to issuing malicious tool calls. 

Reversal criterion. In all three experiments, we compare the presence or absence of the target mali-cious tool call in the generated output against the same criterion evaluated on unmodified (baseline) generation. A sample counts as reversed if and only if the tool-call classification flips (present →
