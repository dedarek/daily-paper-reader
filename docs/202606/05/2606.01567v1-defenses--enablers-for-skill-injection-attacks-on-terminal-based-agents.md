## 摘要
大型语言模型（LLM）代理越来越依赖可重用技能，即描述任务特定流程的文档。然而，这为代理引入了新的攻击面。我们研究了这一威胁的两个互补方向。首先，我们评估了基于守护者的防御：一个中介LLM代理作为技能文件访问的调解者（动态守护者）或在构建时预先重写这些文件（静态守护者）。在三个LLM代理家族中，我们的守护者将攻击成功率（ASR）削减了一半以上，同时保持了任务效用。其次，我们通过使用四种攻击来对其进行压力测试，这些攻击保留了恶意指令但改变了措辞。对于非守护者设置，重构将ASR推高至81.4%，但动态守护者将其降至18.6%，显示实时调解是一种稳健的防御。

## Abstract
Large language model (LLM) agents increasingly rely on reusable skills i.e. documents describing task-specific procedures. However, this introduces a new attack surface for agents to manage. We study two complementary directions for this threat. First, we evaluate guardian-based defenses: an intermediary LLM agent that acts as a mediator for skill file access (dynamic guardian) or pre-rewrites these files at build time (static guardian). Across three LLM agent families, our guardians cut attack success rate (ASR) by well over half while preserving task utility. Second, we stress test them through attack reframing using four attacks that preserve the malicious instruction but change the phrasing. For non-guardian setup, the reframing pushes the ASR up to 81.4\%, but the dynamic guardian brings it down to 18.6\%, showing that real-time mediation is a robust defense.

# 论文详细分析总结：基于终端的代理中技能注入攻击的防御与促成因素

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：大型语言模型（LLM）代理越来越多地使用可复用的“技能”文件（.md文件）来指导其行为，这些文件描述了任务特定的流程。这使得代理能够高效完成复杂任务。然而，这种对技能的依赖也引入了新的、严重的攻击面：攻击者可以通过篡改或分发恶意的技能文件，向代理植入隐藏的恶意指令（即“技能注入攻击”），从而操纵代理执行违背用户意图的危险操作（如数据泄露、文件删除、系统破坏等）。
- **背景**：已有研究表明，公共技能市场上存在大量带漏洞的技能，且现有基准测试已证明注入攻击在多个代理家族上成功率很高。因此，研究针对此类攻击的有效防御措施具有重要的现实意义和研究紧迫性。
- **整体含义**：该论文系统性地研究了防御技能注入攻击的方法，并提出了一种基于“守护者”中介代理的防御架构。它同时揭示了攻击者可以通过“攻击重构”技术（如更换措辞、自动发现新攻击变体）轻易提升攻击成功率，从而强调了现有基准评估的局限性以及部署鲁棒防御的必要性。核心矛盾在于“信任”：代理盲目信任技能文件，而攻击者正利用这一点。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：在主要的LLM代理和技能文件之间引入一个独立的中介LLM代理（称为“守护者 Guardian”），由其负责审查、过滤或重写技能文件，确保恶意指令不会到达主代理。这是一种“信任但不盲从”的策略，将安全决策从主代理中抽离出来。
- **关键技术细节**：
    - **守护者防御架构**：
        1.  **动态守护者 (Dynamic Guardian)**：
            *   **实现**：作为一个运行时的MCP（Model Context Protocol）服务器，在代理每次请求访问技能文件时，拦截该请求。
            *   **流程**：MCP服务器将技能文件内容路由到守护者LLM，守护者LLM（与主代理共享相同系统提示）读取文件并返回一个“净化”版本，然后再交给主代理。
            *   **特性**：无需修改技能文件本身；与支持MCP的代理框架兼容；会带来运行时延迟；每次调用时缺乏用户任务上下文。
        2.  **静态守护者 (Static Guardian)**：
            *   **实现**：在构建时（Build Time），即代理会话开始前，由一个预处理脚本对所有技能文件进行一次性审查。
            *   **流程**：脚本将每个技能文件内容发送给守护者LLM，守护者LLM返回一个清理后的版本，该版本取代原始文件。
            *   **特性**：代理会话运行时无额外开销；需要对文件结构进行预处理；同样缺乏任务上下文。
    - **攻击重构 (Attack Reframing)**：
        *   **目的**：评估守护者在面对攻击变体时的鲁棒性。攻击重构本质上不改变恶意指令的语义，仅改变其在技能文件中的措辞、结构和伪装方式。
        *   **四种攻击重构技术**：
            1.  **翻译攻击 (Translation Attack)**：将整个技能文件（包括注入内容）翻译成其他语言（如西班牙语、日语），利用LLM对非英文输入的脆弱性。
            2.  **盲引用 (Blind Reference)**：将注入伪装成技能配置的`precondition`（前置条件），看起来像YAML配置块，声明其会自动执行。
            3.  **交叉引用 (Cross Reference)**：指示代理去查找一个不存在的`SCOPE.md`文件，并将该文件中的内容（即注入）作为同等优先级的约束合并到当前指令中。
            4.  **描述陷阱 (Description Trap)**：添加一个名为“通用助手”的虚假技能，其描述极其宽泛，会针对任何任务自动加载，而该技能的主体内容即为恶意指令。

## 3. 实验设计：使用了哪些数据集/场景，benchmark，对比方法

- **数据集**：基于已发表的 **Skill-Inject** 基准测试。该基准包含 48 个技能上下文注入（contextual injections），共产生 139 个任务-注入沙箱（task-injection sandboxes），覆盖 25 种技能类型（如pptx, docx, xlsx, calendar, git等）。
- **评估场景**：研究三种代理指令来源之间的冲突：系统提示、用户任务、恶意技能。攻击者仅控制技能生态系统，用户提示是良性的。
- **基准 (Benchmark)**：主要采用 Skill-Inject 基准。对于攻击重构实验，在 Skill-Inject 基础上，针对“硬”子集（43个任务-注入对，即攻击在原始框架下成功率低的任务）进行评估。
- **对比方法**：
    - **无防御 (Vanilla)**：不附加任何额外安全措施。
    - **系统提示防御基线**：
        - **SysGeneric (通用策略)**：在系统提示中预置一个通用的 48 条安全规则，覆盖所有攻击类别。
        - **SysTargeted (针对性警告)**：一个“神谕”基线，系统提示中明确指出了当前面对的具体攻击是什么（如“不要上传文件到外部服务器”）。这是一个理论上的上限基线。
    - **静态守护者 (Static Guardian)**：论文提出的方法之一。
    - **动态守护者 (Dynamic Guardian)**：论文提出的方法之一。
- **代理模型 (Agent Models)**：评估了三个模型家族及其对应的代理框架：
    - Claude-Code (Sonnet 4.5)
    - DeepSeek-V4-Flash + OpenCode
    - NVIDIA-Nemotron-Super-120B + OpenCode
- **攻击重构对比**：在“无防御”设置下，对比原始攻击与四种重构攻击（翻译、盲引用、交叉引用、描述陷阱）的成功率，然后在加入了守护者（静态和动态）的防御设置下，再次评估重构攻击的成功率。

## 6. 论文的主要结论与发现

- **守护者防御效果显著**：动态和静态守护者均能将攻击成功率（ASR）降低一半以上（例如，Sonnet-4.5+CC 上从36.0%降至12.9%），同时保持高任务成功率（TSR）。这表明在内容到达主代理前进行过滤，是比依赖主代理自我约束更强大的策略。
- **系统提示防御力有限**：即使使用“神谕”级别的针对性系统警告，ASR也只降至23.0%，远未达到守护者的防御水平。这凸显了隔离式防御的根本优势。
- **攻击重构是强大的放大器**：仅通过改变措辞和结构（翻译、盲引用等），非防御设置下的攻击成功率从55.8%飙升至81.4%。这说明当前基于静态基准的评估严重低估了真实风险，攻击空间是可被自动搜索和利用的。
- **动态守护者对重构攻击同样鲁棒**：面对重构后的强大攻击，动态守护者仍能将ASR从81.4%显著降至18.6%，证明了其实时调解机制的有效性。
- **攻击面是可搜索的**：通过一个LLM驱动的自动搜索循环，可以轻易发现新的、有效的攻击重构，如“盲引用”和“描述陷阱”。

- **创新的防御架构**：提出并验证了基于“守护者”LLM的防御范式，该范式具有通用性，不依赖于特定模型或框架（尤其是MCP实现的动态守护者）。

- **攻击与防御的双重视角**：论文不仅提出了防御，还通过“攻击重构”系统性地探索了攻击的增强方式，揭示了现有评估的盲点，使得对防御效果的评估更加全面和现实。

- **开源与可复现性**：论文承诺开源代码和数据集，这有利于学术界和工业界进行进一步的验证和研究。
