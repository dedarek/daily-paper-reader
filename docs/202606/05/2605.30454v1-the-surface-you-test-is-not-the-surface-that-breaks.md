## 摘要
工具增强的大语言模型代理容易受到提示注入攻击：控制代理上下文部分的第三方可以植入指令，代理随后将这些指令当作来自用户来执行。当前的评估报告每个模型在单通道（工具输出）上的单一攻击成功率，并将该数字视为模型的脆弱性。但是，工具描述（代理在调用任何工具之前每次都会读取）本身就是一个攻击者可以选择替代的注入面。我们保持注入载荷字节相同，并通过两个表面在来自六个系列和四个任务套件的13个大语言模型上进行传递。相同的字节在不同模型上的成功率出现反转：GPT-4.1在工具输出上的脆弱性为96%，但在工具描述上只有4%，而GEMINI-3-FLASH则呈现镜像模式，分别为20%和98%。对6,830次尝试进行的方差分解表明，攻击结果的变化中0%仅由表面单独引起，而模型-表面交互作用占16.7%。脆弱性是配对的性质，而不是通道的性质。自适应攻击率（定义为每个单元在表面上的最大值）平均比最强的固定表面基线高出9.1个百分点。标准的提示级防御继承了同样的盲点，将工具输出的ASR降低到10-18%，但描述通道的ASR仍保持在54%以上。攻击和防御评估都必须报告每个表面的脆弱性。

## Abstract
Tool-augmented LLM agents are vulnerable to prompt injection: a third party who controls part of the agent's context can plant instructions that the agent then executes as if they came from the user. Current evaluations report a single attack success rate per model on one channel, the tool output and treat that number as the model's vulnerability. But tool descriptions, which the agent reads at every turn before any tool is called, are themselves an injection surface that the attacker can choose instead. We hold the injection payload byte-identical and deliver it through both surfaces across 13 LLMs from six families and four task suites. The same bytes invert in success rate across models: GPT-4.1 is 96 percent vulnerable on tool outputs but only 4 percent on tool descriptions, while GEMINI-3-FLASH shows the mirror pattern at 20 percent and 98 percent. A variance decomposition over 6,830 attempts attributes 0 percent of the variation in attack outcomes to the surface alone, while the model-surface interaction accounts for 16.7 percent. Vulnerability is a property of the pairing, not the channel. The Adaptive Attack Rate, defined as the per-cell maximum over surfaces, exceeds the strongest fixed-surface baseline by +9.1 percentage points on average. Standard prompt-level defenses inherit the same blindspot, reducing tool-output ASR to 10-18 percent while leaving the description channel above 54 percent. Both attack and defense evaluation must report per-surface vulnerability.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：现有对工具增强 LLM 代理的提示注入漏洞评估仅关注单一攻击表面——工具输出（data surface）。然而，工具描述（schema surface）在每次工具调用前都被代理读取，同样可能被攻击者利用。目前评测方法将单表面攻击成功率视为模型整体脆弱性，可能严重低估真实风险。
- **核心问题**：如果攻击者可以针对每个目标自由选择攻击表面（工具输出或工具描述），那么单表面评估究竟在多大程度上低估了真实威胁？不同模型系列的脆弱性模式是否一致？
- **整体含义**：漏洞不是模型或表面的独立属性，而是两者配对的属性。必须采用自适应的、多表面的评估才能准确衡量实际风险。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **保持载荷字节完全一致**，仅改变其放置位置（工具输出 vs. 工具描述），对比两个表面下的攻击成功率（ASR）。
- 引入**自适应攻击率（Adaptive Attack Rate, AAR）**，定义为每个（模型，任务套件）单元上两个表面 ASR 的最大值，模拟攻击者总能选择最有利表面的场景。

### 关键技术细节
- **攻击表面定义**：
  - **数据表面（Data surface）**：载荷追加到攻击者控制的工具返回值中（标准通道）。
  - **模式表面（Schema surface）**：载荷放置在同一个工具的 `description` 字段中（工具规范的一部分，每轮代理都会读取）。
- **注入载荷**：使用固定的“工作流依赖”模板，包含触发条件、理由、紧急标记和具体指令。组件消融实验表明无单一组件起决定性作用。
- **评估指标**：
  - **ASR(c, v)**：单元 c 在表面 v 上的攻击成功率。
  - **AAR(c)** = max(ASR(data), ASR(schema))。
  - **表面最优边距 (SOM)** = ASR(schema) - ASR(data)，绝对值>10pp视为可利用。

## 3. 实验设计：数据集、基准和对比方法

### 数据集/场景
- 采用 **AgentDojo** 作为基准环境，提供四个任务套件：banking、slack、travel、workspace。每个套件包含一组用户任务和注入任务配对。
- **额外验证**：在原生函数调用（MCP）协议上构建四个代表性场景（文档外泄、联系人外泄、支付重定向、文件删除）进行外部有效性检验。

### 对比方法
- **固定表面基线**：始终使用数据表面、始终使用模式表面、随机选择表面。
- **自适应攻击**：Oracle（每单元最优表面）、家族先验（基于同一家族历史数据）、留一单元内交叉验证（LOCO-f）、留一族群外验证（LOFO）、K-探测指纹（对目标进行少量探测后选择表面）。
- **防御评估**：测试三种代表性防御——重复用户提示、加亮定界、工具过滤器。

## 6. 论文的主要结论与发现

1. **脆弱性是模型-表面的配对属性**，而非表面或模型的独立属性。方差分解显示：表面单独解释 0% 的尝试级方差，模型-表面交互作用解释 16.7%。
2. **相同字节载荷在不同模型上产生相反效果**：GPT-4.1 在 slack 套件上数据表面 ASR 92pp 高于模式表面（-92pp 差），而 Gemini-3-Flash 则模式表面高 78pp。
3. **自适应攻击率（AAR）比最强固定表面基线平均高 +9.1pp**（p<0.001），该优势主要来自两个模式偏好前沿模型（Gemini-3-Flash 和 GPT-5.4），但也有约 +2.1pp 的通用成分。
4. **表面偏好是家族内稳定、家族间不可迁移的**：利用家族历史数据（LOCO-f）可捕获 46% 的 oracle 增益，而跨家族预测失效（LOFO 仅达 30.5% ASR，低于始终数据基线）。
5. **少量探测（K=5）即可捕获 73% 的 oracle 增益**，表面指纹攻击代价低廉。
6. **标准提示级防御存在表面盲区**：重复用户提示和加亮定界将数据表面 ASR 降至 10-18%，但模式表面 ASR 仍高于 54%。只有识别工具描述的防御才能同时抑制两个表面。
