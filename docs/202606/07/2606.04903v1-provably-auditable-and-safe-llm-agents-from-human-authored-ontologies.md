## 摘要
我们引入了LLM智能体架构Agentic Redux，旨在用于需要线性审计能力的非平凡问题领域。通过使用类型化lambda演算，我们证明，在适当领域上运行时，Agentic Redux的执行在语义上保证正确，所有决策记录在仅追加的分类账中。我们提出了两个生产级别的适当领域：医疗账单合规性和安全漏洞披露。支持代码仓库中提供了在两个领域上运行Agentic Redux的工作代码。我们还介绍了本体优先的智能体设计（Ontology-First Agent Design），这是一种在问题领域上创建智能体框架的方法论，其中人类专家使用基本形式本体对本领域进行本体化，然后分配LLM来推导出智能体和人类在环中可承担的角色，以处理该领域中的问题。

## Abstract
We introduce the LLM agent architecture Agentic Redux, intended for use with nontrivial problem domains that require linear auditability. Using the typed lambda calculus, we prove that, run on appropriate domains, Agentic Redux executions are semantically guaranteed to be correct, with all decisions recorded in an append-only ledger. We present two production-grade appropriate domains, in healthcare billing compliance, and security vulnerability disclosure. Working code for Agentic Redux run on both domains is available in a supporting code repository. We also introduce Ontology-First Agent Design, a methodology for creation of agent frameworks on a problem domain, in which a human expert ontologizes the problem domain with Basic Formal Ontology, and then assigns an LLM to derive roles that agents and humans-in-the-loop can fill, in order to work the problems in the domain.

### 论文提出的方法论

1.  **核心思想：Agentic Redux 架构**
    -   **灵感来源**：受前端状态管理器Redux启发。
    -   **架构模式**：采用“智能中心，笨拙组件”（Smart container, dumb components）模式。
        -   **子智能体（Sub-agents）**：每个子智能体只能看到全局状态的一个本地切片（state slice），并基于此提出动作提案（propose）。它们是“只读”的，无权限直接修改状态。
        -   **元智能体（Meta-agent）**：拥有全局状态视图，是唯一有权修改状态的实体。它负责接收所有子智能体的提案，并**根据预声明的系统不变量（invariants）进行裁决（adjudicate）**，只有满足所有不变量的提案才会被批准执行。
    -   **关键技术细节**：
        -   **不变量（Invariants）**：由领域专家定义，是系统必须始终为真的规则（如“总预算不超过$100,000”）。这是保证安全的核心。
        -   **线性可审计性（Linear Auditability）**：元智能体将每一次裁决（批准、拒绝或升级）都记录在一个**仅追加（append-only）的账本（ledger）** 中。这确保了所有决策过程可追溯。
        -   **Counselor Queue**：为引入人类在环（Human-in-the-loop）而设计的扩展。当提案违反“可升级”类型的不变量时，会被放入队列等待人工审核，确保关键决策有专家参与，且不影响系统核心的安全保证。
    -   **形式化证明**：使用**类型化的lambda演算（Typed Lambda Calculus）** 对Agentic Redux架构进行形式化建模。证明了**不变量保持定理**（任何可达状态都满足不变量）和**审计日志完整性定理**（日志准确、完整、线性增长且不会被修改），从而在理论上保证了系统的安全性和可审计性。

2.  **算法流程：本体优先的智能体设计 (Ontology-First Agent Design)**
    -   **步骤1：本体化（Ontologize with BFO）**：人类领域专家使用**基本形式本体（BFO, Basic Formal Ontology）** 对问题领域进行结构化描述，创建领域本体文档。
    -   **步骤2：评估领域指纹（Assess Domain Fingerprints）**：LLM评估该本体是否包含“写偏斜”和“审计性”等指纹特征，判断是否需要Agentic Redux这类复杂架构。
    -   **步骤3：智能体推导（Convert Ontology → Domain Module / Derivation of Roles and Invariants）**：
        -   **角色推导（Role Derivation）**：LLM列出所有过程（processes），并根据其状态足迹（state footprint，即其读写的数据集合）进行分组。每一组定义一个智能体角色，并赋予其最小的、必须的“安全权限”（状态切片）。
        -   **不变量推导（Invariant Derivation）**：从本体的**策略规则**和**过程前提条件**中提取系统不变量。然后根据其状态足迹是否跨越多个智能体，判定其为局部不变量（由单个智能体维护）或**跨域不变量（cross-cutting invariants）**（必须由元智能体强制执行）。
    -   **步骤4：执行与审查（Run and Review）**：将生成领域模块（代码）在Agentic Redux 内核上运行，人类专家根据执行结果修正本体，形成反馈闭环。
