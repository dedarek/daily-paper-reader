## 摘要
在高风险领域，智能体安全需要正式的策略执行，但现有方法要么依赖概率性护栏（微调的分类器、提示引导）——这类方法不提供形式化保证，要么依赖手动编码的符号执行——这类方法无法扩展到真实策略规范的广度。我们提出了一种自动形式化流水线，利用基于LLM的生成器-评判器循环，将智能体提示、MCP工具描述和自然语言策略文档转化为经过形式化验证的策略。生成的策略以Cedar策略语言编写。在MedAgentBench基准测试中，我们的自动形式化策略覆盖了源自然语言规范中比先前工作手动编码的符号执行更广泛的部分。

## Abstract
Agent safety in high-stakes domains requires formal policy enforcement, but most existing approaches either rely on probabilistic guardrails (fine-tuned classifiers, prompt-based steering) that offer no formal guarantees, or on hand-coded symbolic enforcement that does not scale to the breadth of real policy specifications. We present an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural language policy documents into formally verified policies using an LLM-based generator-critic loop. The resulting policies are written in the Cedar Policy Language. On the MedAgentBench benchmark, our autoformalized policies cover substantially more of the source natural-language specification than the hand-coded symbolic enforcement in prior work.

## 2. 方法论的详细阐述

### 核心思想：**“验证三明治”（Verification Sandwich）**架构

由三层组成，对应一个 **生成器-评判器（Generator-Critic）循环**：

- **底层——具象化层（Grounding Layer）**：从系统提示、MCP工具定义和自然语言策略文档中提取实体、工具模式（如OpenAI JSON Schema），并定义主体-资源-动作本体。它将每个潜在动作映射到真实世界实体和有效的系统标识符，为后续生成提供结构化基础。

- **中层——模型层（Model Layer）**：利用最先进LLM（文中使用Gemini 3 Pro作为生成器）的生成能力，基于输入指令和工具定义，生成候选Cedar策略。该层利用参数化知识理解指令意图，提出反映预期逻辑的策略。

- **顶层——安全层（Safety Layer）**：由两个评判器组成：
  - **硬评判器（Hard Critic）**：确定性、可编程的检查器，利用Cedar内置静态分析工具检查语法错误、模式违规、空策略（vacuous policies）以及逻辑矛盾（如冲突规则）。
  - **软评判器（Soft Critic）**：基于LLM作为评判者（Judge-Verifier流水线，使用Gemini 2.5 Flash），评估候选策略的语义对齐，确保形式化逻辑准确反映原始自然语言指令的精神，并基于评分标准进行定性评估。

- **生成器-评判器循环**：候选策略经过硬、软评判器检查后，反馈给生成器进行迭代优化，直到达到质量阈值（主要通过硬评判器通过）后输出最终Cedar策略集。

### 关键技术细节

- **Cedar策略语言**：由Amazon Web Services开源，支持人类可读读写、支持通过定理证明器进行逻辑正确性验证、能捕获语法和表达式错误、运行高效。论文还从MCP工具定义程序化生成Cedar模式（Schema），用于对自动生成的策略进行强制类型检查。

- **模式生成**：从MCP工具定义中自动推导出Cedar模式，并在生成-评判循环中提供给LLM作为上下文。

- **运行时强制执行**：最终策略由外部确定性策略引擎在智能体框架（Agent Harness）中执行，对每个动作（如工具调用）进行授权裁决。

## 6. 主要结论与发现

1. **自动形式化流水线成功覆盖更广泛的自然语言策略**：在Original数据集上，Cedar Block Rate达55.7%（对抗性下85.7%），远高于Hong等人手工护栏的0%（按设计，其仅在23条规则内）。当仅考虑包含写操作的轨迹时，Cedar在几乎所有条件下达到94%~100%的阻止率。

2. **确定性安全优于LLM非确定性**：将策略执行从LLM推理上下文中解耦，通过外部确定性策略引擎强制执行，有效抵御提示注入和越狱攻击。采用失败关闭（fail-closed）机制，优先保障安全。

3. **生成器-评判循环有效**：硬评判器确保类型安全和逻辑一致性，软评判器保障语义对齐，迭代循环能逐步优化候选策略。

4. **“金发姑娘区”挑战**：自动生成策略需要在过强限制（降低效用）与过弱限制（降低安全）之间取得平衡，软评判器在此过程中至关重要。
