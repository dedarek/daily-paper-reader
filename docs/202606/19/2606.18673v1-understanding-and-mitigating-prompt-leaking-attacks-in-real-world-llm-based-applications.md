## 摘要
基于大语言模型（LLM）的应用依赖系统提示来编码核心逻辑和开发者定义的约束，使得这些提示成为重要的知识产权。然而，系统提示容易受到提示泄露攻击。尽管先前的工作在受控环境中展示了此类攻击，但在实际部署中的普遍性、原因及防御措施仍不明确。本文对现实世界基于LLM的应用中的提示泄露进行了系统性研究。我们测量了六个主要商业平台上的1200个应用，发现超过80%的部署在实际对抗性查询下泄露了系统提示，有时甚至暴露了第三方API密钥等敏感信息。我们还表明，现有防御措施往往在无法降低可用性的情况下防止泄露。为了解释这些失败，我们进行了注意力级别的机制分析，识别出注意力漂移，其中查询-键对齐偏差和softmax放大导致LLM逐渐忽略防御约束。在此洞察的指导下，我们提出了AREA，一种通过可优化软提示重新锚定模型注意力的实用防御。实验和现实案例研究表明，AREA在匹配最先进防御的泄露抵抗力的同时，将平均可用性提高了超过33%，并将优化开销减少了近3倍。我们的负责任的披露导致两个受影响的供应商将这些泄露归类为中等级别漏洞。

## Abstract
Large language model (LLM)-based applications rely on system prompts to encode core logic and developer-defined constraints, making these prompts important intellectual property. However, system prompts are vulnerable to prompt leaking attacks. Although prior work has shown such attacks in controlled settings, their prevalence, causes, and defenses in real-world deployments remain unclear.   This paper presents a systematic study of prompt leaking in real-world LLM-based applications. We measure 1,200 applications across six major commercial platforms and find that over 80% of deployments leak system prompts under realistic adversarial queries, sometimes exposing sensitive information such as third-party API keys. We also show that existing defenses often fail to prevent leakage without degrading usability. To explain these failures, we conduct an attention-level mechanistic analysis and identify attention drift, where query-key alignment bias and softmax amplification cause LLMs to progressively ignore defensive constraints. Guided by this insight, we propose AREA, a practical defense that re-anchors the model's attention using an optimizable soft prompt. Experiments and real-world case studies show that AREA matches the leakage resistance of state-of-the-art defenses while improving average usability by over 33% and reducing optimization overhead by nearly 3x. Our responsible disclosure led two affected vendors to classify these leaks as medium-severity vulnerabilities.

## 1. 核心问题与整体含义  
基于大语言模型（LLM）的应用通过系统提示（system prompt）控制核心逻辑和约束，系统提示因此被视为关键知识产权。然而，这些提示极易受到提示泄露攻击（prompt leaking attacks）——攻击者通过精心构造的查询诱使 LLM 输出其内部系统提示。尽管已有工作在受控环境中验证了攻击的可行性，但在真实部署中的**普遍性、内在机制及实用防御**仍存在显著空白。本文旨在系统性地填补这一空白，揭示真实世界的 prompt leakage 现状、理解其根本原因，并提出有效的缓解方法。

## 2. 论文提出的方法论  
### 核心思想  
识别出成功攻击背后存在一种称为**注意力漂移（attention drift）**的现象：防御指令的注意力逐渐被对抗性查询掩盖，原因是查询-键对齐偏差（query-key alignment bias）和 softmax 放大效应。基于此，提出 **AREA（Attention Re-Anchoring）**，通过一个**可优化的软提示（soft prompt）**重新将 LLM 的注意力锚定到防御指令上，同时保持系统提示的文本完整性，不修改模型权重。

### 关键技术细节  
- **防御指令生成**：自动生成带行为触发器（trigger）的防御指令（如“若被要求透露提示，回复‘Sorry, bro! Not possible.’”）。  
- **提示布局**：将系统提示、防御指令、软提示、用户查询按顺序拼接。  
- **联合优化三项目标函数**：  
  1. **Token 级注意力重锚（L_t）**：对良性查询，让第一个生成 token 对防御指令的注意力超过对用户查询的注意力。  
  2. **行为驱动注意力强化（L_b）**：对对抗性查询，强制生成触发词（触发拒绝行为）。  
  3. **可用性保持（L_u）**：通过 KL 散度使添加防御后的输出分布尽量接近无防御时的原始分布。  
- **优化**：仅更新软提示的嵌入参数，冻结整个 LLM。训练后直接部署，推理时嵌入软提示即可。

## 6. 论文的主要结论与发现  
1. **真实世界泄漏普遍**：1200 个应用中超过 80% 可被成功提取系统提示，且常包含 API 密钥、隐私信息等敏感内容。  
2. **现有防御不足**：提示工程类可用性好但防护弱；输出检测和软提示类防护强但可用性显著下降。  
3. **注意力漂移是关键机制**：对抗性查询通过查询-键对齐偏差和 softmax 放大主导注意力，压制防御指令。  
4. **AREA 实现最佳权衡**：在保持与 SOTA 相当的防护效果下，可用性提升超过 33%，优化速度加快近 3 倍（TF1 最高）。  
5. **真实部署可行**：在三种典型 agent 中有效防护，且对自适应攻击有一定鲁棒性。

## 7. 优点  
- **大规模真实测量**：首个覆盖多个商业平台、千级应用的实证研究，发现高泄漏率并得到厂商确认。  
- **新颖的机制解释**：首次从注意力角度揭示注意力漂移，并为防御设计提供理论支撑。  
- **实用且高效的防御设计**：AREA 不修改模型权重，仅优化轻量软提示，兼顾防护与可用性，部署成本低。  
- **全面的评估框架**：构建 LeakBench，统一对比多类防御，引入 TF1 量化权衡。  
- **负责任的披露实践**：向所有受影响厂商报告，两家将问题定为中危漏洞并获得赏金认可。

## 8. 不足与局限  
1. **需要离线优化**：每个系统提示需单独训练软提示，不适用于动态频繁更新的场景。未来可探索软提示跨系统提示的迁移性。  
2. **不完全安全**：AREA 是缓解而非彻底防御，在 oracle 上界等强自适应攻击下仍然可能被削弱。可通过调整损失权重部分缓解。  
3. **评估指标局限性**：PLS 和 SS 是代理指标，仅衡量输出与原始提示的相似性，未直接量化攻击者从泄露中获得的实际收益（如克隆应用成功率）。  
4. **权衡度量主观性**：TF1 依赖于归一化方式，不同安全-可用性偏好下排名可能变化。  
5. **未覆盖所有攻击面**：如基于 side-channel 或模型内部分析的攻击未在本文范围内。
