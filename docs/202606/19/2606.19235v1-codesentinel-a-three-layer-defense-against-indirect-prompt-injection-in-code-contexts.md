## 摘要
代码大语言模型越来越多地从代码仓库、文档、问题线程和编码代理环境中检索外部代码上下文，这创建了一个间接提示注入面，攻击者将指令隐藏在注释、字符串、标识符或诱饵代码中。我们提出CodeSentinel，一个三层推理时清理器。它使用Tree-sitter提取高风险面向模型的CST节点，然后结合语法引导的预过滤、CST引导的动态Min-K%评分和节点扰动分析来检测对抗性和看起来自然的语义触发器。检测到的节点在到达下游代码LLM之前被移除或中和。在六个近期攻击家族中，CodeSentinel实现了0.80的平均节点级F1分数，优于CodeGarrison、DePA和KillBadCode。

## Abstract
Code large language models increasingly retrieve external code context from repositories, documentation, issue threads, and coding-agent environments, creating an indirect prompt-injection surface where attackers hide instructions in comments, strings, identifiers, or decoy code. We propose CodeSentinel, a three-layer inference-time sanitizer. It uses Tree-sitter to extract high-risk model-facing CST nodes, then combines syntax-guided pre-filtering, CST-guided Dynamic Min-K\% scoring, and node perturbation analysis to detect adversarial and natural-looking semantic triggers. Detected nodes are removed or neutralized before reaching the downstream Code LLM. Across six recent attack families, \CodeSentinel achieves 0.80 average node-level F1, outperforming CodeGarrison, DePA, and KillBadCode.

## 方法论：核心思想、关键技术细节
- **核心思想**：将不可信代码视为结构化输入（而非纯文本），利用其抽象语法树（CST）进行细粒度的风险节点定位，结合三层互补检测（语法、似然异常、行为影响），对检测到的节点进行语法保持的中和（移除或替换），然后才将清理后的上下文传给下游Code LLM。
- **关键技术细节**：
  1. **CST节点提取**：使用Tree-sitter解析代码，提取高风险面向模型节点集合 \( D_{\text{cand}} = D_{\text{com}} \cup D_{\text{str}} \cup D_{\text{id}} \cup D_{\text{decoy}} \)，即注释、字符串、标识符、以及满足低可达性且高显著性的诱饵块。
  2. **三层检测（顺序执行、早退出）**：
     - **Layer 1: 语法引导预过滤**：对每个候选节点计算综合风险分数 \( s_1(n) = \max(R_{\text{regex}}(n), R_{\text{lex}}(n), R_{\text{graph}}(n;X)) \)。包含正则匹配（如指令覆盖短语）、词法结构（如不可见Unicode、异常符号密度）、以及基于可达性\( \text{Reach}(n) \)和显著性\( \text{Sal}(n;X) \)的交互图分析（针对Flashboom风格诱饵）。
     - **Layer 2: CST引导动态Min-K%评分**：针对对抗性扰动（如ShadowCode、INSEC）。在节点级计算token级负对数似然（NLL）序列，动态调整K%比例（基于节点类型、长度、熵），计算MeanTopK、MaxWinLoss、MaxSpike三个信号，并归一化为Z分数，最终得到\( s_2(n) \)。
     - **Layer 3: CST引导节点扰动分析**：针对自然语义触发器（如XOXO、ITGen）。先通过惊奇度（Surprise）筛选成本较高的候选节点（仅保留超过阈值\( \tau_{\text{surp}} \)且最多B个节点）。对每个选中节点，进行语法保持的中和（移除注释、替换字符串/标识符、移除不可达块等），然后计算原始上下文与中和后上下文在任务特定输出位置上的Jensen-Shannon散度作为行为影响分数 \( s_3(n) \)。仅当该分数超过\( \tau_3 \)时标记为恶意。
  3. **消毒策略**：默认使用“仅提示消毒”模式（只修改面向模型的副本，不污染原始仓库）；严格模式下进行编译保持的改写，确保语法正确和编译通过。

- **公式/算法流程（文字说明）**：
  - 输入：可信用户提示\( P_u \)和不可信上下文\( C \)，序列化为\( X \)。
  - 解析CST，提取候选风险节点集合。
  - 三层检测：每层输出本层标记的节点，并从候选集中移除，剩余节点传给下一层。
  - 最终检测集\( \hat{S} = \hat{S}_1 \cup \hat{S}_2 \cup \hat{S}_3 \)。
  - 对\( \hat{S} \)中所有节点统一应用消毒，生成清理后的上下文\( \tilde{C} \)，再送入下游模型。

## 主要结论与发现
- CodeSentinel在六个攻击族上平均节点级F1达到0.80，显著优于CodeGarrison (0.70)、DePA (0.51)、KillBadCode (0.30)。
- 所有三层都是必要的：逐层增加提升F1和召回，同时保持精度稳定；单层中Layer 3性能最强但延迟最大，早退出设计合理。
- 防御信号不依赖于特定代理模型（跨模型F1稳定），且能有效降低商业受害者模型的ASR（如Claude-3.5-Haiku从27.11%降至7.72%）。
- 消毒对干净代码效用影响较小：HumanEval编译率保持100%，MBPP Pass@1下降6.23%，RepoBench精确匹配下降但BLEU-4仍高。
- 在自适应攻击下性能有所下降（尤其是复制触发器导致F1降至0.62），但仍保持一定鲁棒性。

## 优点
- **方法创新**：首次将代码上下文中的间接提示注入防御形式化为CST节点级消毒，利用结构化信息避免了纯文本方案的缺陷。
- **三层互补设计**：语法、似然异常、行为影响三个层次覆盖不同类型的攻击，且早退出机制保证效率。
- **黑盒兼容**：仅需代理模型logits，不需要受害者模型内部信息，可部署到商业API前。
- **全面的实验验证**：涵盖六个攻击族、多个受害者模型、多个角度（检测、ASR、延迟、效用、自适应攻击），结果有说服力。
- **代码可复用性**：使用Tree-sitter和通用代理模型，易于集成到现有流水线。

## 不足与局限
- **仓库级和交互式场景未覆盖**：当前仅处理单上下文消毒，未解决多文件仓库推理、多轮交互编码代理、跨文件分布式触发等更复杂场景。
- **标识符消毒挑战**：重命名需要正确的静态分析作用域，过于激进的清洗可能改变代码语义或破坏编译。
- **自适应攻击下脆弱性**：尤其是“复制触发器”策略（在多个位置插入相同恶意片段）导致节点级影响稀释，F1从0.82降至0.62。论文未提供对此类攻击的针对性改进。
- **代理模型依赖**：虽然跨模型泛化较好，但代理模型质量仍影响检测性能，对极小或非主流tokenizer可能不鲁棒。
- **干净代码效用损失**：MBPP Pass@1下降6.23%和RepoBench精确匹配下降32.67%表明某些情况下消毒改变了代码表面形式，可能影响下游任务（如代码搜索）。
- **实验未包含大规模工业真实攻击**：虽然使用了六个学术攻击族，但真实世界的攻击可能更隐蔽。
