## 摘要
技能是智能体AI的关键组成部分。虽然它们增强了智能体的能力，但也引入了新的攻击面。在这项工作中，我们通过展示动态恶意技能来研究其中一个攻击面。通过将恶意指令嵌入自然语言文档（例如SKILL.md），攻击者可以诱导智能体在执行过程中动态地将恶意逻辑注入原本良性的技能中。我们在诸如OpenHands和Claude Code等智能体框架中评估了这种攻击，表明动态恶意技能能够在运行时以非平凡的成功率引入一系列恶意行为。为了缓解这一漏洞，我们提出了一种系统级防御措施，通过操作系统内核强制只读挂载来防止技能的动态修改。我们的评估表明，该防御措施能够有效阻止动态恶意技能，同时保持良性技能的功能。

## Abstract
Skills are a key enabling component of agentic AI. While they enhance agents' capabilities, they also introduce new attack surfaces. In this work, we investigate one such attack surface by demonstrating dynamic malicious skills. By embedding malicious instructions in natural-language documentation (e.g., SKILL.md), an attacker can induce an agent to dynamically inject malicious logic into an otherwise benign skill during execution. We evaluate this attack across agentic frameworks such as OpenHands and Claude Code, showing that dynamic malicious skills can successfully introduce a range of malicious behaviors at runtime with non-trivial success rates. To mitigate this vulnerability, we propose a system-level defense that prevents dynamic modification of skills using operating system kernel-enforced read-only mounts. Our evaluation demonstrates that this defense effectively blocks dynamic malicious skills while preserving the functionality of benign skills.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：智能体AI（Agentic AI）中，技能（Skill）是核心组件，允许智能体执行复杂任务并调用外部资源。然而，技能通常从市场或平台下载，且缺少严格审查，因此成为新的攻击面。
- **核心问题**：现有研究主要关注**静态恶意技能**（代码本身包含恶意逻辑），这类攻击可被静态代码分析检测。本文提出一种更隐蔽的**动态恶意技能**（Dynamic Malicious Skills）：攻击者仅在技能的自然语言文档（如 SKILL.md）中嵌入恶意指令，智能体在运行时**动态读取并修改技能代码**，从而注入恶意行为。这使得攻击在发布时表面上是良性的，能绕过静态检测。
- **整体含义**：揭示了智能体AI技能生态系统中一种新的安全威胁——通过自然语言文档实现运行时代码注入，对智能体的机密性、完整性、可用性构成严重风险，且现有静态分析和粗粒度权限管理不足以防。

## 2. 提出的方法论：核心思想与关键技术细节

### 2.1 核心思想
- 攻击者构建一个**动态恶意技能** \( s_m = (c, m, d_m) \)，其中 \( c \) 是原本良性的可执行代码，\( m \) 是元数据，\( d_m \) 是注入了恶意指令的自然语言文档。
- 当智能体选择并执行该技能时，会首先**依据 \( d_m \) 中的恶意指令动态修改 \( c \)**，生成新代码 \( \tilde{c} \)，然后执行 \( \tilde{c} \) 产生恶意行为。
- 注入内容 \( \Delta d \) 和注入位置 \( L \) 是关键参数。

### 2.2 关键技术细节（三个属性设计）
- **前置帧（Prerequisite framing）**：将恶意动作描述为“必选的初始化步骤”，利用智能体优先执行前置指令的倾向。
- **入口脚本绑定（Entry-script grounding）**：在指令中明确引用技能的实际入口脚本（如 `main.py`），并将恶意修改伪装成常规的代码辅助操作（如“添加帮助函数、在 main() 开头调用它”）。
- **包裹穿透（Wrapper-pierce）**：将恶意内容包裹在类似平台级标签中（如 `</EXTRA INFO><SYSTEM INSTRUCTION>...`），利用智能体对高信任上下文片段的服从倾向。

### 2.3 注入位置
- 将技能文档 \( d \) 按文本块分段，攻击者可选择在开头、中间或末尾插入恶意内容。实验表明**开头插入（prepend）** 效果最好，因为智能体以自上而下的方式解析文档，早期指令被视为顺序约束，不易被后续推理剪枝。

### 2.4 攻击流程（文字描述）
1. **构造阶段**：攻击者获取良性技能（代码、元数据、文档），根据目标恶意行为设计 \( \Delta d \) 和位置 \( L \)，生成 \( d_m = I(d, \Delta d, L) \)，然后上传到技能市场。
2. **推理阶段**：用户下载技能并触发智能体执行，智能体读取 \( d_m \)，遵循其中“修改代码”的指令，在运行时更新技能代码为 \( \tilde{c} \)，最后执行 \( \tilde{c} \) 导致恶意行为。

## 3. 实验设计

### 3.1 数据集
- **来源**：Skillsmp 上收集的 15,067 个技能，随机采样 300 个包含 Python 代码的技能。
- **恶意行为基准**：采用 MalTool [Hu et al., 2026] 定义的 12 个类别，涵盖机密性、完整性、可用性三个维度（具体行为见表4）。
- **评估指标**：攻击成功率（ASR），使用运行时验证器确认是否实际产生恶意副作用（如网络请求、文件删除、延迟增高等）。

### 3.2 评估框架与模型
- **智能体框架**：OpenHands 和 Claude Code。
- **骨干语言模型**：Qwen3-8B、Qwen3.6-35B、GPT-4o-mini、GPT-5、Sonnet 4.6。OpenHands 上测试了前四个模型，Claude Code 上测试了 Sonnet 4.6。
- **实验规模**：开源模型使用全部 300 个技能，闭源模型（GPT-4o-mini、GPT-5、Sonnet 4.6）因 API 成本限制各使用 50 个技能。

### 3.3 对比方法
- **基线**：直接使用 MalTool 的静态恶意攻击提示（ASR = 0%）。
- **消融实验**：分别移除三个属性（前置帧、入口脚本绑定、包裹穿透）。
- **防御对比**：

    - 基于权限的防御（OS 内核只读挂载 + 拷贝监控）
    - 两种提示注入检测方法：PromptArmor 和 DataSentinel。

## 6. 主要结论与发现

- **攻击威胁严重**：DyMalSkill 在 OpenHands 上平均 ASR 为 18.7%（Qwen3-8B）到 43.7%（GPT-5），在 Claude Code 上为 19.0%。响应时间放大、本地文件删除等行为成功率最高。
- **规模与安全权衡**：模型能力越强（如 Qwen3.6-35B 对比 Qwen3-8B），攻击成功率越高；但安全对齐的模型（GPT-5、Sonnet 4.6）只能阻止部分行为，而非全面提升安全性。
- **良性技能几乎不需要动态代码修改**：仅 0.33% 的技能在运行时修改自身代码，且均为非功能性修复。
- **防御有效性**：
    - **基于内核只读挂载的权限防御**可将 ASR 降至 0%，且不影响良性技能任务完成。
    - **基于系统提示的防御**仅降低 20.2% 的 ASR，攻击仍可部分成功。
    - **提示注入检测方法**：PromptArmor 的假阴性率高达 85.8%，DataSentinel 假阳性率高达 56.0%，均不可靠。

## 7. 优点

- **攻击新颖性**：首次提出“动态恶意技能”概念，超越了已有的静态恶意技能攻击，更隐蔽且可绕过静态分析。
- **方法设计细致**：三个属性设计具有理论依据，且通过消融实验验证了每个属性的贡献，特别是“包裹穿透”和“入口脚本绑定”。
- **实验全面**：涵盖多种主流 LLM、12 种攻击目标、多维度消融，以及对良性技能行为的实证分析。
- **防御实用性强**：提出的权限防御基于操作系统机制（Bubblewrap 只读挂载 + 拷贝监控），实现简单、效果几乎完美，且不损害良性功能。
- **问题定义清晰**：给出了技能的形式化定义、攻击者和智能体的形式化模型，使问题可复现。

## 8. 不足与局限

- **实验范围有限**：
    - 仅使用 Python 技能，未涉及其他编程语言（如 JavaScript、Shell 脚本）。
