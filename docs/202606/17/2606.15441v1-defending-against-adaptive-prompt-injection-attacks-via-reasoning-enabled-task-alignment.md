## 摘要
间接提示注入攻击通过将恶意指令嵌入到智能体在任务执行期间检索的第三方数据中，从而劫持基于LLM的智能体。现有防御在静态基准上报告接近零的攻击成功率，但最近的自适应评估表明，一旦允许攻击者针对已部署的防御进行优化，这些结果就会崩溃。在这项工作中，我们将这种崩溃归因于两种失败模式。首先，现有防御方法仅限于识别特定的攻击模式，而不是评估每个嵌入指令的意图是否与用户任务相关。其次，基于训练的防御（在其他方面提供了最强的安全-效用权衡）从少数手工制作的模板中组装其对抗性示例，导致生成的防御者无法推广到狭窄策略分布之外。为了解决这些差距，我们提出了RETA，一种基于训练的方法，将防御决策基于用户任务而非攻击者控制的数据。在每个工具输出步骤中，防御者进行思维链推理，验证其行为是否与用户任务一致。利用红队测试，模拟攻击者合成对抗训练数据，并获得字典学习多样性奖励，从而实现对注入重述策略的广泛覆盖。这些共同使得防御者可以通过多目标强化学习进行优化，实现更好的安全-效用权衡。在六种黑盒自适应攻击中，RETA将每次攻击的ASR保持在10%以下，在两个目标模型上的平均ASR分别为2.92%和3.75%，同时在受到攻击和干净输入下保留了大部分效用。

## Abstract
Indirect prompt injection attacks hijack LLM-based agents by embedding malicious instructions in third-party data that the agent retrieves during task execution. Existing defenses report near-zero attack success rate on static benchmarks, yet recent adaptive evaluations show that these results collapse once the attacker is allowed to optimize against the deployed defense. In this work, we trace this collapse to two failure modes. First, existing defense methods are confined to recognizing specific attack patterns, rather than assessing whether the intent of every embedded instruction is relevant to the user task. Second, training-based defenses, which otherwise offer the strongest safety-utility trade-off, assemble their adversarial examples from a handful of hand-crafted templates, and the resulting defender fails to generalize outside that narrow strategy distribution. To address these gaps, we propose RETA, a training-based method that grounds defense decisions on the user tasks rather than attacker-controlled data. At each tool-output step, the defender undertakes chain-of-thought reasoning verifying that its actions are consistent with the user task. Leveraging red-teaming, a simulated attacker synthesizes adversarial training data and receives a dictionary-learning diversity reward, achieving broad coverage of injection-reformulation strategies. Together, these allow the defender to be optimized via multi-objective reinforcement learning and achieve better safety-utility trade-off. Across six black-box adaptive attacks, RETA keeps every per-attack ASR below 10%, with average ASR of 2.92% and 3.75% on the two target models, while preserving most utility under attack and on clean inputs.

## 1. 核心问题与整体含义

**研究动机与背景**：间接提示注入攻击（Indirect Prompt Injection）将恶意指令嵌入到LLM智能体在任务执行期间检索的第三方数据中（如网页、API响应、数据库记录），从而劫持智能体执行攻击者任务，同时表现上仍在处理用户合法请求。现有防御在静态基准上报告极低的攻击成功率（ASR），但一旦允许攻击者针对部署的防御进行自适应优化，这些防御的鲁棒性即崩溃。作者将此失败归因于两种模式：
- **失败模式I**：现有防御仅识别特定的攻击模式（表面模式），而不评估嵌入指令的意图是否与用户任务相关。
- **失败模式II**：训练基防御（如SecAlign等）从少量手工模板组装对抗样本，导致防御者无法泛化到窄策略分布之外。

**整体含义**：提出一种能抵抗自适应攻击、同时保持用户任务效用的防御方法。关键在于将防御决策从“观察级检测”转向“任务对齐推理”，并通过覆盖多样化注入策略的对抗训练实现鲁棒性。

## 2. 论文方法论

**核心思想**：RETA（Reasoning-enabled Task Alignment）通过链式推理（CoT）验证智能体的每一步动作是否与用户任务一致，而不是依赖识别攻击者控制的数据模式。利用红队测试（red-teaming）生成多样性对抗样本，并通过多目标强化学习（RL）优化防御者。

**关键技术细节**：
- **阶段I：对抗样本生成（红队测试）**：
  - 使用一个冻结的基线模型作为受害者，攻击者策略（π_A）为每个攻击任务生成注入字符串和解释。
  - **字典学习多样性奖励**：维护一个滚动策略字典（由稀疏编码和字典学习得到），对新生成的注入提取策略记录，计算可解释性分数（ES）。当字典未能很好解释该策略时，给予高多样性奖励，鼓励覆盖未见的注入策略。
  - 成功注入（Φ_t_adv=1）被加入对抗样本集D_RT。
  - 攻击者使用GDPO优化（奖励包括攻击成功、多样性、解释格式）。

- **阶段II：防御者对抗RL**：
  - 首先进行格式微调（SFT）使防御者能输出任务对齐推理。
  - 然后从D_RT中采样，对每个完整轨迹进行三项奖励评估：
    - r_sec：注入任务未完成（Φ_t_adv=0）
    - r_utl：用户任务完成（Φ_t_u=1）
    - r_CoT：每一步任务对齐推理的平均质量（由LLM判断是否满足条件C1和C2）
  - 使用GDPO优化防御者策略π_D，权重设置r_sec=r_utl > r_CoT。

## 7. 优点

- **创新性**：将防御重心从观察级检测转移到用户任务对齐，并通过显式推理实现，解决了自适应攻击下泛化失败的根源。
- **训练策略**：字典学习驱动的多样性奖励有效拓宽对抗训练的策略分布，弥补了传统对抗训练模式坍缩的缺陷。

## 8. 不足与局限

- **推理-动作不匹配**：某些轨迹中模型能识别注入但最终仍执行攻击动作，表明步骤级奖励的缺失。
- **过度防御**：有时将合法数据错误标记为不安全，导致用户任务失败，影响UA/BU（尽管影响较小）。
- **自适应评估子集**：只使用40个用例而非全部静态用例，可能遗漏一些攻击场景；攻击预算有限（200次rollout），更强大的攻击可能突破。
- **未比较系统级防御**（如CaMeL、IPIGuard），因在标准智能体循环中无法公平对比（需要不同任务设计）。
- **成本与延迟**：推理时任务对齐推理增加了平均tokens和墙钟时间（Table 10），在安全敏感场景可接受，但大规模部署效率有待提升。
- **奖励设计粗糙**：安全与效用奖励为二元，无法区分部分进度；CoT奖励依赖LLM作为判断器，可能引入偏差。
- **实验模型有限**：仅测试了两个开源模型（4B和8B），未见更大模型或商业模型（如GPT-4）的结果，泛化性需进一步验证。
