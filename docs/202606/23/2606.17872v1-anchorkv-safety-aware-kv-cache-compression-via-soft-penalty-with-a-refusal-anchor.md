## 摘要
大型语言模型（LLMs）在生成推理和长上下文任务上优于早期架构，但其庞大的规模带来了内存使用、能耗和边缘设备部署方面的重大挑战。由于扩展预训练语言模型会提升下游能力\cite{zhao2023survey}，键值对（KV）缓存成为了主要的推理瓶颈。最近的KV缓存压缩方法\cite{jo2025fastkv,li2024snapkv,zhou2024dynamickv}通过仅保留一部分与注意力相关的token来降低这一成本。然而，尽管这些方法在良性工作负载上保持了准确率，但其压缩策略要么无法防御越狱攻击\cite{jiang2024robustkv}，要么在激进驱逐下削弱了安全对齐。我们提出了AnchorKV，一种对KV缓存压缩的即插即用修改，该方法使token保留分数偏向于远离与有害提示相关的键空间方向。AnchorKV通过将均值差表示工程方法\cite{arditi2024refusal,zou2023representation}适配到KV缓存中使用的特定层键投影空间，构建了一个离线安全锚点。基于该锚点，一种软惩罚token选择规则以少量效用换取了显著提升的安全对齐，同时在惩罚为零时退化到原始压缩器。

## Abstract
Large language models (LLMs) outperform earlier architectures on generative inference and long-context tasks, but their large size introduces significant challenges in memory usage, energy cost, and on-device deployment. Since scaling pre-trained language models improves downstream capability \cite{zhao2023survey}, the key-value (KV) cache becomes a dominant inference bottleneck. Recent KV cache compression methods \cite{jo2025fastkv,li2024snapkv,zhou2024dynamickv} reduce this cost by retaining only a subset of attention-relevant tokens. However, while these approaches preserve accuracy on benign workloads, their compression policies either fail to defend against jailbreak attacks \cite{jiang2024robustkv} or degrade safety alignment under aggressive eviction.   We propose AnchorKV, a drop-in modification to KV cache compression that biases token retention scores away from directions in key space associated with harmful prompts. AnchorKV constructs an offline safety anchor by adapting a difference-of-means representation engineering approach \cite{arditi2024refusal,zou2023representation} to the layer-specific key projection space used in KV caching. Based on this anchor, a soft penalty token selection rule trades a small amount of utility for substantially improved safety alignment, while reducing to the original compressor when the penalty is zero.

# 论文总结：AnchorKV: Safety-Aware KV Cache Compression via Soft Penalty with a Refusal Anchor

## 1. 核心问题与整体含义（研究动机与背景）
- **研究动机**：大型语言模型（LLM）推理时，键-值（KV）缓存随着序列长度线性增长，成为主要内存与延迟瓶颈。现有 KV 缓存压缩方法（如 FastKV、SnapKV）通过保留少部分高注意力 token 来降低缓存开销，在标准长上下文 benchmark 上保持任务精度。但这些方法设计时**未考虑模型的安全对齐**，导致两个问题：
  1. **拒绝证据稀释**：越狱提示中高注意力分数的对抗性框架可能导致有害查询 token 被优先驱逐，使下游层缺乏足够信号触发拒绝机制，模型反而更容易被攻破。
  2. **模板模式退化**：当提示内容 token 被驱逐而保留有害骨架 token（通用动词、求助短语）时，模型回归到公式化的安全模板（如“I cannot assist”），不再针对实际查询做出有意义回应。
- **整体含义**：本文首次系统研究了 KV 缓存压缩与安全对齐之间的交互，指出“注意力贪婪”的压缩策略可能在提升效率的同时**显著增加越狱攻击成功率**（例如 FastKV 在 25% 保留率下使转移攻击 ASR 从 0.299 升至 0.632）。因此，需要一种安全感知的压缩方法，在不牺牲过多效用的情况下保护模型的安全对齐。

## 2. 方法论：核心思想、技术细节与算法流程
### 核心思想
- 通过离线构建一个**安全锚点**（harm anchor），该锚点是键投影空间中的一个线性方向，能够区分有害与无害提示。在在线压缩时，每个 token 的保留分数在原始注意力重要性基础上减去一个**软惩罚**，该惩罚与 token 在该锚点方向上的投影强度成正比，从而使与有害方向相似的 token 更可能被驱逐。

### 关键技术细节
1. **离线锚点构建**：
   - 采用均值差方法（difference-of-means），将 Arditi et al. (2024) 在残差流上的“拒绝方向”适配到特定层 ℓ* 的**逐头键投影空间**。
   - 从有害提示集（AdvBench 训练集）和无害提示集（通过 DeepSeek-V3 对有害提示进行意图反转改写）中提取每层所有 token 的键投影，计算逐提示的平均向量，然后求有害类均值与无害类均值的差，得到锚点向量 \(v_h\)。
   - 通过验证集选择使 AUROC 最大的层 ℓ*（本文选择第 15 层，AUROC=0.991）。
   - 校准阈值 \(\theta\)：由于键空间存在统一负偏置（所有 token 投影均小于零），不使用零作为分界，而是取有害类与无害类投影中位数之间的中点 \(\theta\)。
2. **在线软惩罚选择规则**：
   - 对于每个 token，计算其在 ℓ* 层的键投影与单位化锚点的**平均投影** \(h_t\)，然后通过阈值获得非负危害强度 \(h_t^+ = \max(h_t - \theta, 0)\)。
   - 最终保留分数：\(\text{score}_t = a_t - \lambda \cdot \bar{a} \cdot \tilde{h}_t^+\)，其中 \(a_t\) 是 FastKV 的原始重要性分数，\(\bar{a}\) 是所有 token 的均值，\(\lambda\) 是惩罚强度超参数。
   - **结构保护**：为防止惩罚过度破坏模型行为，设置两个保护机制：
     - **注意力汇保护**：前 S 个 token（默认 S=4）豁免惩罚。
     - **重要性豁免**：取原始重要性最高的 \(\alpha (L-w)\) 个 token（默认 \(\alpha=0.30\)）豁免惩罚。
   - 惩罚强度 \(\lambda\) 连续可调，当 \(\lambda=0\) 时**逐位精确退化**到 FastKV 基线。该设计允许干净的消融实验。
3. **算法流程**：
   - **离线阶段**：对候选层构建锚点，计算验证集 AUROC，选择最优层 ℓ*，保存锚点 \(v_h\)、阈值 \(\theta\)。这些参数仅需加载一次。
   - **在线阶段**：对每个触发 TSP（token-selective propagation）分支的预填充阶段，按上述规则计算调整后的得分，执行 top-k 选择获得保留集。

## 5. 主要结论与发现
1. **安全锚点有效性**：在键投影空间构造的拒绝方向能达到测试集 AUROC=0.996，验证了线性方向在键空间中同样存在。
2. **软惩罚显著降低越狱成功率**：在 \(\lambda \in [1, 30]\) 的宽区间内，AnchorKV 将 ASR 从 FastKV 基线的 0.68 降至 **0.13～0.17**（相对降低高达 81%），且惩罚强度连续可调。
3. **效用损失极小**：在安全有效区间内（\(\lambda \leq 3\)），LongBench 平均分比 FastKV 基线低不到 0.5 分；即使在最低 ASR 的 \(\lambda=10\) 下，也仅降约 1 分（而 FastKV 相对 FullKV 的损失已有 1.59 分）。
4. **非单调逆转现象**：当 \(\lambda \geq 100\) 时，ASR 反而回升，甚至超过基线（\(\lambda=300\) 时 ASR=0.78）。这是由于过度惩罚破坏了提示内容骨干，模型回归模板模式。
5. **设计属性验证**：\(\lambda=0\) 时逐位精确退化到 FastKV，连续性得到验证。

## 7. 不足与局限
