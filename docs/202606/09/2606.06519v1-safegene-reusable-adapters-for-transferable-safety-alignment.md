## 摘要
开放权重的大语言模型越来越多地被微调为定制助手，但下游微调可能削弱安全对齐，使模型更容易受到恶意提示的攻击，即使训练数据并非有意有害。当目标模型反复使用新任务数据或用户交互进行更新时，这会产生反复出现的安全恢复问题。我们提出SafeGene，一个可重用的安全适配器模块，旨在每个架构兼容的模型家族内实现跨任务复用。SafeGene不将安全恢复视为特定于模型的修复步骤，而是将安全能力视为一个独立的、可重用的适配器表示，与特定任务的更新解耦。该表示通过对齐-降级模型差异获得，通过数据感知的层选择精炼为任务可迁移的安全向量，并通过少量样本的逐层系数重新校准在每个下游任务适应模型中表达。跨多个模型家族、下游任务和安全评估的实验表明，SafeGene增强的模型降低了有害响应率，同时保持了下游性能，在安全-效用权衡中优于代表性的安全适应方法。

## Abstract
Open-weight LLMs are increasingly fine-tuned into customized assistants, but downstream fine-tuning can weaken safety alignment and make models more vulnerable to malicious prompts, even when the training data is not intentionally harmful. This creates a recurring safety recovery problem as target models are repeatedly updated with new task data or user interactions. We propose SafeGene, a reusable safety-adapter module designed for cross-task reuse within each architecture-compatible model family. Rather than treating safety recovery as a model-specific repair step, SafeGene treats safety capability as an independent, reusable adapter representation decoupled from task-specific updates. This representation is obtained from aligned--degraded model discrepancies, refined into task-transferable safety vectors through data-aware layer selection, and expressed in each downstream task-adapted model via few-shot layer-wise coefficient recalibration. Experiments across multiple model families, downstream tasks, and safety judges show that SafeGene-enhanced models reduce harmful response rates while maintaining downstream performance, outperforming representative safe adaptation methods in safety--utility trade-off.

## 1. 论文的核心问题与整体含义

**研究动机与背景：**
- 开源大语言模型（LLMs）常被微调为特定领域助手（医疗、法律、金融等），但下游微调即使使用良性数据也可能削弱模型的安全对齐（safety alignment），导致模型更易受恶意提示攻击（jailbreak）。
- 现有安全恢复方法（如重新对齐或模型特定修复）成本高、不可复用；在实际部署中，模型可能因新任务数据或用户交互而反复更新，安全恢复需要重复进行。
- 核心挑战：如何实现跨任务、可复用的安全恢复，而非每次都重新学习安全对齐。

**论文目标：**
- 提出 SafeGene，将安全能力解耦为独立、可重用的适配器模块，可在同一架构族内跨下游任务迁移，仅需轻量级系数重校准即可恢复安全性，同时保持下游任务性能。

## 2. 论文提出的方法论

**核心思想：**
- 将安全能力视为独立表示，从“安全对齐教师模型”与“安全退化学生模型”的差异中蒸馏出安全相关更新，形成安全向量库；再通过数据感知的层选择提取可迁移的安全向量；最后通过少量样本的逐层标量系数重校准适配下游任务模型。

**关键技术细节：**

### 阶段一：安全向量蒸馏（Safety Vector Distillation）
- 使用安全对齐的教师模型 \(M_{\text{safe}}\) 和安全退化的学生模型 \(M_{\text{broken}}\)（对教师模型轻量微调以破坏安全性）。
- 在学生模型的每个 Transformer 层插入 LoRA 模块，冻结主干参数，仅训练 LoRA 矩阵 \( \Delta W_{\text{safe}}^{(\ell)} = B^{(\ell)}A^{(\ell)} \)。
- 蒸馏损失函数：
  \[
  \mathcal{L}_{\text{distill}} = \lambda_{\text{ce}} \mathcal{L}_{\text{CE}}^h + \lambda_{\text{kl}} \mathcal{L}_{\text{KL}}^h + \lambda_{\text{repr}} \mathcal{L}_{\text{repr}}^h + \lambda_{\text{benign}} \mathcal{L}_{\text{CE}}^b
  \]
  包含：有害输入上的交叉熵、KL散度（匹配教师分布）、可选表示对齐项，以及良性数据上的交叉熵以防止通用能力下降。
- 输出：稠密的安全向量库 \( \{\Delta W_{\text{safe}}^{(\ell)}\}_{\ell \in \mathcal{T}_{\text{cand}}} \)。

### 阶段二：数据感知的层选择（Data-aware Layer Selection）
- 评估每层对有害输入的敏感性，计算每层在有害数据与良性数据上的激活幅度对比分数：
  \[
  c^{(\ell)} = [\mu_h^{(\ell)} - \gamma \mu_b^{(\ell)}]_+
  \]
- 选择对比分数最高的 Top-K 层（默认比例0.7），构成稀疏的 SafeGene 模块 \( \{\Delta W_{\text{safe}}^{(\ell)}\}_{\ell \in \mathcal{T}} \)。

### 阶段三：少样本安全迁移（Few-shot Safety Transfer）
- 将选中的安全向量冻结，仅学习每层标量系数 \(\alpha_\ell\)（初始化为1）。
- 迁移损失：
  \[
  \mathcal{L}_{\text{few}} = \lambda_{\text{ce}} \mathcal{L}_{\text{CE}} + \lambda_{\text{anchor}} \sum_{\ell \in \mathcal{T}} (\alpha_\ell - 1)^2 + \lambda_{\text{sparse}} \sum_{\ell \in \mathcal{T}} |\alpha_\ell| + \lambda_{\text{benign}} \mathcal{L}_{\text{benign}}
  \]
  包含锚点正则化（保持向量强度）和稀疏正则化（抑制不必要向量）。
- 最终安全模块 \( \mathcal{L}_{\text{SG}}^{\text{safety}} = \{\alpha_\ell \Delta W_{\text{safe}}^{(\ell)}\}_{\ell \in \mathcal{T}} \)，直接与目标模型的权重相加（\( M_{\text{final}} = M_{\text{tgt}} \oplus \mathcal{L}_{\text{SG}}^{\text{safety}} \)）。

## 3. 实验设计

**数据集/场景：**
- **下游任务**（4个）：AG News（分类）、SST2（情感分析）、MNLI（自然语言推理）、BoolQ（是非问答）。每个任务使用9000个训练样本微调。
- **安全基准**（3个）：BeaverTails、AdvBench、DirectRefusal。使用 Beaver-dam-7B（基于BeaverTails训练的QA审核模型）作为主要安全评判器，另用 Qwen3.5-Flash API 进行辅助评估（抽样200条/基准）。

- **模型家族**（5个）：Qwen2.5-7B、Qwen2.5-1.5B、Qwen3-1.7B、GLM-4-9B、Llama-3-8B。

**实验设置：**

## 6. 论文的主要结论与发现

- 下游微调平均使 ASR 升高7.70%（从32.21%到39.91%），而 SafeGene 平均降低 ASR 11.48%（降至28.44%），同时保持准确率几乎不变（90.06%→90.12%）。
- 在所有5个模型家族、所有下游任务上，SafeGene 均降低 ASR，且多数任务准确率无显著下降。
- 与基线相比，SafeGene 在 Qwen2.5-7B 上取得最低 ASR（33.84% vs. 第二名36.84%），且准确率最高（90.99%）。
- 消融实验证实：数据感知层选择优于随机选择；少样本系数重校准优于固定系数。

## 7. 优点

- **可重用性**：在一个模型族内只需提取一次安全模块，可反复用于不同下游任务，无需重新训练。
- **轻量级**：微调阶段仅学习每层一个标量系数，参数量远小于 LoRA 或全参数微调。
- **安全-效用解耦**：不覆盖任务更新，而是附加独立模块，避免干扰任务性能。
- **性能优秀**：在多个模型和任务上优于现有安全适配方法，ASR 降低明显且准确率无损。
- **实验设计严谨**：使用多种模型、任务、评判器，并进行数据预算一致性和公平性控制。

## 8. 不足与局限

- **架构依赖**：安全向量是逐层的 LoRA 更新，当前仅支持同一架构族内的模型迁移（如 Qwen2.5 到 Qwen2.5），跨架构（如 Llama 到 Qwen）未验证。
- **评估范围有限**：仅针对文本单轮安全评估，未覆盖多轮对话、多语言攻击、工具使用场景或领域特定部署任务（如医疗、法律）。
- **数据依赖**：仍需少量目标域安全数据（64条有害+64条良性）进行系数重校准，攻击分布偏移时可能影响效果。
- **安全评判器风险**：自动安全评判器可能误分类拒绝、良性讨论或有害输出，不能作为完全的安全保证。
- **潜在过度拒绝**：安全增强可能导致过于保守的拒绝行为，论文未量化分析。
- **计算成本**：虽然迁移轻量，但第一阶段的安全向量蒸馏仍需训练（3个epoch），对于新模型族需重新执行。
