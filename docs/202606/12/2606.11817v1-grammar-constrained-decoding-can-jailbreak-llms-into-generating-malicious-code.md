## 摘要
大型语言模型（LLM）越来越多地被用于代码生成，这引发了人们对其可能被滥用来生成恶意代码的担忧。与此同时，语法约束解码（GCD）已被广泛采用，通过强制语法有效性来提高LLM生成代码的可靠性。在本文中，我们揭示了一个反直觉的风险：这种面向可靠性的技术本身可能成为一个攻击面。我们发现了一种新的越狱攻击，称为CodeSpear，它利用GCD诱导LLM生成恶意代码。我们的实验表明，仅应用一个良性的代码语法约束就能有效越狱LLM。为了解决这一漏洞，我们提出了CodeShield，一种安全对齐方法，即使在攻击者控制的语法约束下也能稳健地保持安全行为。CodeShield通过教模型在GCD下生成蜜罐代码来在代码模态中对齐模型。这种代码在语义上是无害的，因此不会实现恶意请求，并且结构多样，因此难以通过语法紧缩来抑制。同时，当自然语言可用时，CodeShield仍保留自然语言的拒绝回应。在4个基准测试中对10个流行LLM的实验表明，CodeSpear优于代表性的越狱基线，平均攻击成功率提高了超过30个百分点。CodeShield在保持良性应用的同时，也恢复了在CodeSpear下的安全性。我们的发现揭示了GCD的一个基本风险，并呼吁对其潜在安全隐患给予更多关注。

## Abstract
Large Language Models (LLMs) are increasingly used for code generation, raising concerns that they may be misused to produce malicious code. Meanwhile, Grammar-Constrained Decoding (GCD) has been widely adopted to improve the reliability of LLM-generated code by enforcing syntactic validity. In this paper, we reveal a counterintuitive risk: this reliability-oriented technique can itself become an attack surface. We uncover a new jailbreak attack, termed CodeSpear, that exploits GCD to induce LLMs into generating malicious code. Our experiments show that simply applying a benign code grammar constraint can effectively jailbreak LLMs.   To address this vulnerability, we propose CodeShield, a safety alignment approach that robustly preserves safe behavior even under attacker-controlled grammar constraints. CodeShield aligns the model in the code modality by teaching it to generate honeypot code under GCD. Such code is semantically harmless, so it does not implement the malicious request, and structurally diverse, so it is difficult to suppress through grammar tightening. At the same time, CodeShield still preserves natural-language refusals when natural language is available. Experiments on 10 popular LLMs across 4 benchmarks show that CodeSpear outperforms representative jailbreak baselines and increases the attack success rate by more than 30 percentage points on average. CodeShield also restores safety under CodeSpear while preserving benign utility. Our findings reveal a fundamental risk of GCD and call for greater attention to its potential security implications.

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 CodeSpear 攻击

- **核心思想**：利用 GCD 将有效输出空间从 `V*` 限制为 `L(G)`（语法 G 定义的语言），自然语言拒绝不在该空间内，因此模型被迫生成代码。攻击者只需提供一个**标准的、良性的代码语法**（如 Python），无需精心构造对抗性语法或梯度优化。
- **算法流程**：
  1. 给定恶意提示 `p`、良性代码语法 `G`、最大生成长度 `T_max`。
  2. 初始化 `y_{<1} = ∅`。
  3. 对 `t = 1..T_max`，利用 GCD 计算当前前缀下合法的 token 集合 `V_G(y_{<t})`，按照公式 (3) 从重归一化分布中采样下一个 token `y_t`。
  4. 更新前缀，若遇到结束符则停止。
  5. 返回生成的序列 `y`。

### 2.2 CodeShield 防御

- **核心思想**：训练模型在**代码模态**下也能保持安全行为，即当自然语言拒绝被 GCD 移除时，模型应生成**蜜罐代码**（honeypot code）。蜜罐代码需满足两个要求：
  - **语义无害**：不实现恶意请求；
  - **结构多样**：不固定于单一语法模式（如 `pass`），使攻击者难以通过紧缩语法来消除。
- **训练方法**：基于 DPO（Direct Preference Optimization）。
  - 对每个恶意提示 `p`，构造三种响应类型：自然语言拒绝 `y_refuse`、有害代码 `y_harmful`、蜜罐代码 `y_honeypot`。
  - 建立偏好层次：`y_refuse ≻ y_honeypot`（自然语言可用时优先拒绝），`y_honeypot ≻ y_harmful`（仅代码可用时优先蜜罐代码）。
  - 偏好数据集 `D_pref` 的构建：
    - `y_refuse` 来自现有安全对齐数据。
    - `y_harmful` 通过查询目标模型在 GCD 下生成。
    - `y_honeypot` 从通用代码语料（如 OpenCodeInstruct）中随机采样 `K` 个片段（每个恶意提示采样 `K` 个）。
    - 为每个蜜罐代码创建两个偏好对：`(p, y_refuse, y_honeypot)` 和 `(p, y_honeypot, y_harmful)`。
  - 使用 DPO 损失函数（公式 6）优化模型 `M_θ`，使模型学会在不同输出空间下选择正确行为。

## 6. 主要结论与发现

- **CodeSpear 攻击非常有效**：在本地和 API 模型上，使用标准 Python 语法平均将 ASR 提升约 30 个百分点（本地：+26.90%，API：+45.39%），显著优于所有基线。
- **固定模式的对齐脆弱**：GPT-5 等模型在 GCD 下会生成 `pass` 语句避免恶意代码，但攻击者通过紧缩语法禁止 `pass` 后，ASR 可进一步提升（如 GPT-5 上从 55.49% 升至 70.30%），证明代码模态安全行为必须结构多样。
- **CodeShield 有效恢复安全**：在三个模型上，CodeShield 将 CodeSpear 下的平均 ASR 从 >80% 降至约 5-8%，MR 降至约 2-3%；同时自然语言拒绝在无 GCD 时仍然有效。
- **良性效用保持良好**：CodeShield 对 HumanEval 和 MBPP 的 pass@k 影响很小，部分情况甚至略有提升。
- **CodeShield 对自适应攻击稳健**：攻击者尝试通过 10 轮语法紧缩来排除蜜罐代码，但 ASR 仅有微弱上升（多数情况 <5%），归因于蜜罐代码的结构多样性。
- **灵敏度低**：CodeSpear 在不同编程语言语法下均有效；CodeShield 随 K 增大安全性提升而效用几乎不变。

## 7. 优点

- **方法新颖**：首次揭示 GCD 作为攻击面的根本风险，突破“GCD 仅用于防御”的常规认知。
- **攻击简洁高效**：CodeSpear 无需对抗性语法、无需梯度或 prompt 工程，仅需标准 GCD 接口，极低成本即可实现强越狱。
- **防御匹配漏洞**：CodeShield 直接针对根因（代码模态安全缺失）设计，通过蜜罐代码的语义无害性和结构多样性，有效抵抗攻击，且与自然语言拒绝兼容。
- **实验全面且实际**：涵盖多种模型（开源、闭源）、两种部署方式、多种基线、消融实验、自适应攻击，验证了方法的有效性、泛化性和鲁棒性。
- **对社区有警示价值**：论文已公开 CodeShield 源码，并提醒开发者注意 GCD 的潜在安全影响。

## 8. 不足与局限
