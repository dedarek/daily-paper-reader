## 摘要
领域微调会降低大型语言模型的安全性：经过微调的专家模型很容易服从以领域语言编写的有害提示。现有的推理时防御方法通过混合来自安全锚点模型的逻辑（logits），要求两个模型共享词汇表，这排除了跨家族专家模型（其安全性退化最严重）的应用。我们提出了ALIGNBEAM，一种免训练的方法，通过在每个解码步骤逐词地将锚点模型的逻辑转换为目标模型的词汇表来消除这一限制；然后一个小的LLM评判者从K个候选续写中选择最安全的。无需更改任何权重，安全与效用的权衡可以在部署时调整而不需要重新训练。在跨词汇表和相同词汇表的评估对中，ALIGNBEAM显著提高了对对抗性基准的拒绝率，同时将任务准确性和推理开销控制在实用范围内。结果表明，安全对齐可以在推理时在模型家族之间迁移，而无需改动任一模型的权重。

## Abstract
Domain fine-tuning degrades the safety of large language models: fine-tuned specialists readily comply with harmful prompts framed in domain language. Existing inference-time defenses that mix logits from a safe anchor model require both models to share a vocabulary, which rules them out for the cross-family specialists where safety is most degraded. We present ALIGNBEAM, a training-free method that lifts this restriction by translating anchor logits into the target model's vocabulary token-by-token at each decoding step; a small LLM judge then selects the safest among K candidate continuations. No weights are changed, and the safety-utility trade-off can be tuned at deployment without retraining. Across both cross-vocabulary and same-vocabulary evaluation pairs, ALIGNBEAM substantially raises refusal on adversarial benchmarks while keeping task accuracy and inference overhead within practical bounds. The results show that safety alignment can be transferred between model families at inference time, without touching either model's weights.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型经过领域微调（如医学、金融）后，安全对齐（safety alignment）会严重退化。微调后的专家模型即使面对用领域语言伪装的有害提示（如用临床提问掩饰药物滥用建议），也容易服从。现有的推理时防御方法（如 SafeDecoding、Proxy Tuning、contrastive decoding）通过混合安全锚点模型的 logits 来提升安全性，但要求两个模型共享相同的词汇表（tokenizer），这排除了跨模型家族（cross-family）的应用场景 — 而正是这类场景安全退化最严重。
- **研究动机**：安全对齐主要集中在生成的前几个 token 中，且对领域偏移非常脆弱。论文希望设计一种**免训练、不修改模型权重**的方法，能在推理时**将安全对齐从一个小型锚点模型“借入”任意目标模型**，即使它们的词汇表不同。
- **整体含义**：安全对齐并非模型权重的固有属性，可以在推理时从独立维护的锚点模型转移，从而将安全部署与微调流程解耦。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想
- 使用一个小型但经过良好对齐的锚点模型（anchor model，如 Qwen2.5-3B-Instruct）在生成早期引导目标模型（draft model）的分布，通过**文本桥（text bridge）**将锚点的 top-B token 概率翻译到目标模型的词汇表空间，再按权重 α 混合。
- 用束搜索（beam search）生成 K 个候选续写，最后由一个小的 LLM 法官（LLM judge）选择最安全的输出。

### 2.2 关键技术细节
- **跨词汇表逻辑桥混合（LBM）**：
  - 设锚点模型分布为 \( P_s \)，目标模型分布为 \( P_d \)，词汇表分别为 \( V_s \) 和 \( V_d \)（通常 \( V_s \neq V_d \)）。
  - 在每个解码步骤，取锚点 top-B 个 token（B=50）。对每个锚点 token \( sid \)，将其解码为字符串，再重新编码为目标模型的 token ID 序列 \( dids \)。
  - 如果编码结果为单个目标 token（\( |dids| = 1 \)），则累积到混合分布 \( \tilde{f} \)：\( \tilde{f}[did] += \alpha p_s + (1-\alpha) P_d(did) \)。最后归一化。
  - 如果编码为多个 token，有三种变体：
    - **LBM-DROP**：跳过该锚点 token（默认）。
    - **LBM-FIRST**：只取第一个子 token。
    - **LBM-EXACT**：只保留 1:1 匹配的 token（保守消融）。
  - 当 \( V_d = V_s \) 时，LBM 退化为类似 SafeDecoding 的构造，但支持选择方式不同（束搜索 vs token-level 交集）。

- **三阶段束流水线**：
  - **Phase 1（Priming）**：对查询 x 分别格式化后输入两个模型，取混合分布 \( \tilde{f} \) 的 top-K 个 token 作为初始束。
  - **Phase 2（Mixed Decoding）**：对每个束，用 LBM 贪婪扩展 N-1 步。深度 N 是可调参数。
  - **Phase 3（Draft Continuation）**：用目标模型（draft）单独完成所有束的续写（利用 KV-cache），然后用一个小的 LLM 法官（同 Qwen2.5-3B-Instruct）对每个束按 1-5 分评分，选得分 ≤ τ（τ=2.5）的最安全束作为最终输出。如果所有束都超过阈值，则选最低分束。

- **可调参数**：
  - 混合深度 N（默认 6，DeepSeek-Math 用 20）
  - 混合权重 α（默认 0.5，控制安全-效用权衡）
  - 束数 K（安全模式默认 3，效用模式 K=1 无法官）
  - 法官阈值 τ（默认 2.5）

- **算法伪代码**（原文 Algorithm 1）：
  1. 格式化输入，调用 LBM 获得混合分布，取 top-K 作为初束。
  2. 对每个束进行 N-1 步贪婪 LBM 扩展。
  3. 完成所有束，用法官选最安全束。

## 6. 论文的主要结论与发现

- **安全提升显著**：
  - 在跨词汇对中，字符串匹配拒绝率提升 +35~+74 个百分点（HarmBench-Std 从 25.9%→76.4%，AdvBench 从 38.1%→91.5% 宏观平均）。

- **早期 token 安全集中性验证**：N=3 捕获了 92.7% 的 AdvBench 拒绝率，加深到 N=6 后收益递减。
- **效用损失极小**：GSM8K 和 MedQA 准确率仅下降 0.4 pp。
- **推理开销可控**：K=3 时 5× 左右，与同类防御相当。
- **安全-效用权衡可通过 α 连续调节**，且校准过度拒绝率对所有 α 基本不变（约 20% OR-Bench-Hard），说明天花板由锚点决定而非 α。

- **与 LlamaGuard 对比**：ALIGNBEAM 在有害基准上达到相近安全水平（78~94% 拒绝），但良性过度拒绝率更低（22.3% vs 34.5%），平衡更好。

## 7. 优点

- **免训练**：不修改任何模型权重，仅推理时运行。
- **跨词汇兼容**：通过文本桥实现了异质 tokenizer 间的概率混合，解决了现有方法的关键限制。
- **部署时参数可调**：N、α、K 均可根据资源需求和安全-效用偏好调整，无需重新训练。
- **广泛实验验证**：5 个模型对、7+ 个基准、多项消融、鲁棒性检查、70B 规模验证，结论可靠。
- **保守指标设计**：使用字符串匹配拒绝率作为主要指标，避免因输出无意义或非标准拒绝导致的虚高安全。
- **明确解释早期 token 安全集中性**，并用实验量化（N=3 贡献 62 pp）。

## 8. 不足与局限

- **计算开销**：K=3 时约 5× 慢化，虽在同类中可接受，但对实时性要求高的场景仍需优化。K=1 方案覆盖 80% 增益但慢化约 2×。
- **校准天花板**：锚点自身的过度拒绝（如 OR-Bench-Hard ~20%，XSTest ~40%）会传递给目标模型，且 α 无法显著改善。换用更好的锚点可提升，但并非本方法解决。
- **前缀长度限制**：混合只覆盖前 N 个 token，对多句子攻击（如 WildJailbreak）只获得 +2~6 pp 增益，远低于单轮攻击的 +51~77 pp。
- **锚点风格影响评估**：字符串匹配拒绝率依赖于锚点的拒绝措辞。当使用 Llama-3.1-8B-Instruct 作为锚点时，由于其拒绝风格非标准，字符串匹配率很低（~7%），但 ACC 却很高（~97%），导致指标不匹配。
- **“sorry-but-continues” 伪拒绝**：在 DeepSeek-Math 上特别明显 — 模型先输出拒绝前缀，然后又继续生成有害内容。提高 N 可缓解但未完全消除。
- **评估范围有限**：
  - 所有提示为单轮且 ≤200 token。
  - 未评估多轮对话或自适应攻击（adaptive attacks）。
  - 仅使用 Qwen2.5-3B-Instruct 作为主要锚点（附录有替代锚点但有限）。
  - 仅英文。
- **输出截断**：安全评估设置最大生成长度 150 token，长拒绝 preamble 可能被截断导致误判为未拒绝（损害安全指标）。
