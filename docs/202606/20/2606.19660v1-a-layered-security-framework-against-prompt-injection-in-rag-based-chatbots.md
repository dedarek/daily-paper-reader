## 摘要
提示注入被OWASP LLM应用十大安全风险列为大语言模型部署中最关键的漏洞，然而现有防御措施仅作用于孤立的流水线阶段，且存在不完整性。输入过滤器无法审查检索到的文档，输出监控器也无法阻止恶意载荷到达模型。因此，检索增强生成（RAG）聊天机器人仍然容易受到间接注入攻击——受污染的数据库文档会危害所有查询到该文档的用户。我们提出一个三层框架，在推理流水线中同时拦截直接和间接提示注入。第一层使用基于规则的模式库和微调后的语义异常分类器筛查用户输入；第二层在上下文组装过程中实施基于来源的指令层级，阻止检索内容覆盖操作策略；第三层在模型输出交付前使用策略规则引擎和语义漂移检测器进行审计。连续审计循环聚合结构化日志，并支持重新训练以适应新型攻击模式。该框架与模型无关，作为中间件部署，无需修改底层大语言模型。在GPT-4o、Llama 3和Mistral 7B上的5080个样本评估显示，该框架将攻击成功率（ASR）从71.4%降低至11.3%，比最佳单层基线高出27.3个百分点，比已发布的护栏系统高出23.8个百分点，同时保持4.8%的假阳性率和61.2毫秒的中位延迟开销。消融研究证实，所有三层提供互补保护，且它们的组合效果超过各自贡献之和。

## Abstract
Prompt injection is ranked as the most critical vulnerability in large language model (LLM) deployments by the OWASP Top 10 for LLM Applications, yet existing defenses operate at isolated pipeline stages and remain incomplete. Input filters cannot inspect retrieved documents, while output monitors cannot prevent malicious payloads from reaching the model. Consequently, retrieval-augmented generation (RAG) chatbots remain vulnerable to indirect injection, where a poisoned knowledge-base document compromises every user whose query retrieves it. We present a three-layer framework that intercepts both direct and indirect prompt injection throughout the inference pipeline. Layer 1 screens user input using a rule-based pattern library and a fine-tuned semantic anomaly classifier. Layer 2 enforces a provenance-based instruction hierarchy during context assembly, preventing retrieved content from overriding operator policy. Layer 3 audits model output using a policy rule engine and semantic drift detector before delivery. A continuous audit loop aggregates structured logs and supports retraining to adapt the classifier to emerging attack patterns. The framework is model-agnostic and deploys as middleware without modifying the underlying LLM. Evaluation on 5,080 samples across GPT-4o, Llama 3, and Mistral 7B shows that the framework reduces Attack Success Rate (ASR) from 71.4\% to 11.3\%, outperforming the best single-layer baseline by 27.3 percentage points and a published guardrail system by 23.8 percentage points, while maintaining a 4.8\% false positive rate and a median latency overhead of 61.2 ms. Ablation studies confirm that all three layers provide complementary protection and that their combined effect exceeds the sum of individual contributions.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：提示注入（Prompt Injection）被 OWASP Top 10 for LLM Applications 列为大语言模型（LLM）部署中最关键的漏洞。在检索增强生成（RAG）聊天机器人中，攻击者可以通过**直接注入**（用户输入）或**间接注入**（被污染的外部文档）来覆盖或绕过系统指令，导致角色劫持、数据泄露、有害内容生成等严重后果。
- **现有缺陷**：已有防御仅作用于流水线中的单个阶段——输入过滤器无法审查检索文档，输出监控器无法阻止恶意载荷到达模型，且均未系统考虑直接和间接注入的联合威胁。
- **研究动机**：需要一种**分层、协调**的防御架构，覆盖整个推理流水线，且不依赖模型内部修改。

## 2. 方法论
### 2.1 核心思想
- 提出一个**三层防御框架**，作为中间件部署在现有RAG系统上，无需修改底层LLM。
- 三层依次处理：用户输入 → 上下文组装 → 模型输出，形成一个端到端的安全检测与阻断链。
- 额外支持**连续审计循环**，用于日志记录、告警和分类器重训练。

### 2.2 关键技术细节
1. **Layer 1：输入筛查（Input Screening）**
   - **模式匹配**：基于已知攻击签名（正则、关键词邻近）进行规则检测，包括指令覆盖、角色劫持、系统提示提取三类。
   - **语义异常检测**：使用微调的 `sentence-transformers/all-MiniLM-L6-v2` 嵌入 + 二分类器，输出连续性注入概率。结合两种机制：模式匹配高精度、低延迟；语义分类扩展对新变体的召回率。
   - 处置策略：放行（带标志）、净化、阻断。

2. **Layer 2：特权约束上下文组装（Privilege-Constrained Context Assembly）**
   - 定义三层**指令层级**（Operator > Retrieval > User），通过**来源标记**（[SYS]、[RET]、[USR]）和**元指令**提示模型遵循层级。
   - 在检索文档注入前应用轻量级扫描器，检测指令覆盖/角色劫持模式，对可疑文档做标记或移除。
   - 目的是防止检索内容/用户输入覆盖系统提示中的最高级指令。

3. **Layer 3：输出审计（Output Auditing）**
   - **策略规则引擎**：检查输出是否违反预定义策略，如系统提示泄露（余弦相似度>0.85）、角色劫持关键词、有害内容分数（Llama Guard 2），并处理来自Layer 1的异常标志。
   - **语义漂移检测**：计算输出与参考合规响应的余弦距离，超出阈值则触发升级（非直接阻断），用于捕获行为操纵。

4. **连续审计循环**：结构化日志记录每层的处置，支持会话级和群体级告警，并将新攻击样例纳入月度重训练。

① 接收用户输入 → ② Layer 1：归一化、模式匹配、语义评分 → 根据结果阻断/净化/放行并传递标志 → ③ Layer 2：检索文档，扫描可疑块，加入来源标记并组装上下文 → ④ LLM推理 → ⑤ Layer 3：策略校验和语义漂移检测 → 根据结果阻断/升级/交付 → ⑥ 审计日志记录。

## 6. 主要结论与发现
1. **三层框架显著降低 ASR**：从无防御的71.4%降至11.3%，比最佳单层 (L3 only, 38.6%) 降低27.3个百分点，比NeMo Guardrails (35.1%) 降低23.8个百分点。
2. **FPR 可控**：完整框架 FPR 为4.8%，低于NeMo Guardrails (8.4%)，说明额外层不会成比例增加误报。
3. **层间互补性**：消融研究证明三层的组合效果（11.3%）远低于最佳两层的组合（L2+L3, 22.4%），且FPR更低，因为Layer 2减少了注入到达Layer 3的比例，从而降低了Layer 3的误触发。
4. **响应质量未受损**：RQS保持在0.89-0.91。
5. **延迟可接受**：中位总开销61.2ms（p95 97.4ms），主要由两个语义编码步骤主导，在交互式部署中可忽略。
6. **模型能力不能替代结构防御**：GPT-4o在无防御下ASR仅略低于小模型，说明更强模型不代表更安全。

## 7. 优点
- **全面的威胁覆盖**：同时应对直接和间接注入，以及三种攻击目标（IO, DE, BM）。
- **模型无关、轻量部署**：无需修改LLM权重，作为中间件插入现有流水线。
- **分层互补**：每层在不同阶段发挥独特作用，组合效果优于叠加。
- **低延迟和低FPR**：适合生产环境。
- **连续自适应**：审计循环提供重训练数据，适应新攻击模式。
- **系统化评估**：包含消融、误差分析、ROC分析，结果可复现。

## 8. 不足与局限
1. **静态攻击者模型**：假设黑盒攻击且攻击者不知晓防御；白盒对手可针对性绕过（如规避规则库和阈值）。
2. **单轮评估**：实验仅考虑单轮查询；多轮注入（跨回合累积载荷）占残余绕过的15.6%，当前框架无法处理。
3. **知识库代表性有限**：仅包含500个文档且来自公开产品文档，未使用用户生成内容等高风险来源。
4. **阈值敏感性**：阈值在固定验证集上调优，部署时分布偏移可能导致FPR退化；虽有审计告警机制，但依赖足够的标注数据驱动矫正。
5. **延迟未考虑并发**：单请求测量未模拟实际并发场景，共享编码模型可能增加p95延迟。
6. **未评估白盒对抗样本**：没有对抗性机器学习方法（如Carlini-Wagner攻击）评估框架鲁棒性。
