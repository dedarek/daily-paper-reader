## 摘要
以文本为中心的提示注入防御假设恶意信号在检查的文本视图中是可见的。我们研究了一种可复现的LLM01式间接提示/内容注入失效模式，其中该假设被打破：一个在普通英语中被捕获的有效载荷，当作为结构化浮点参数传输并仅作为碎片化遥测数据重建时，能够避开同一检测器。通过对来自不同提供商的三个商业LLM API进行的14,400次受攻击的真实模型试验，在主矩阵中评估的最强双层文本分类器防御（Prompt Guard 2 + TF-IDF集成）下，源自IFS的浮点数组载体保留了94.3%的泄漏ASR；同样的载体级模式也在微调的roberta-base检测器上得到复现。我们强调泄漏ASR，因为下游系统可能对引述或复制的标记作出反应，即使模型拒绝，但Strong ASR是结构上符合要求的攻击成功的更严格指标。2×2消融实验表明，数据层存储和重建层碎片化击败了不同的文本视图，并且两者都需要才能同时规避两者。一个简单的xxd检测器和语义验证可以阻止当前的T3实例，因此贡献不是无法检测的漏洞，而是针对结构化输入流程中纯文本检查的测量失败边界，这些流程将重建的辅助通道暴露给LLM。

## Abstract
Text-centered prompt-injection defenses assume that the malicious signal is visible in one of the inspected text views. We study a reproducible LLM01-style indirect prompt/content-injection failure mode where that assumption breaks: a payload caught in plain English slips past the same detector when it is transported as structured float parameters and reconstructed only as fragmented telemetry. Across 14,400 attacked real-model trials on three commercial LLM APIs from different providers, the IFS-derived float-array carrier preserves 94.3% leakage ASR under the strongest dual-layer text-classifier defense evaluated in the main matrix: a Prompt Guard 2 + TF-IDF ensemble; the same carrier-level pattern also replicates with a fine-tuned roberta-base detector. We emphasize leakage ASR because downstream systems may act on quoted or reproduced markers even when the model refuses, but Strong ASR is the stricter metric for structurally compliant attack success. A 2 x 2 ablation shows that data-layer storage and reconstruction-layer fragmentation defeat different text views and that both are needed to evade both. A simple xxd detector and semantic validation block the current T3 instance, so the contribution is not an undetectable exploit but a measured failure boundary for text-only inspection in structured-input pipelines that expose reconstructed auxiliary channels to an LLM.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前以文本为中心的提示注入防御（如 Prompt Guard、Llama Guard、正则网关、spotlighting 等）假设恶意信号在检查的文本视图中是可见的。然而，在结构化输入管道中，辅助数值通道（如浮点参数）可能被重建并暴露给 LLM，使该假设失效。
- **研究动机**：LLM 集成 pipeline 越来越多地处理结构化数值数据（嵌入数组、JSON 配置、图边、着色器参数等），这些数据经“可信重建”后以文本报告形式呈现给模型。攻击者可利用载体层隐写术，使恶意载荷在原始数据层和重建文本层均不表现为可检测的文本，从而绕过文本检查。
- **整体含义**：论文定义了一个可复现的失败边界：当 pipeline 暴露重建的辅助数值通道给 LLM 时，纯文本检查不足以防御间接提示注入。这不是声称一种无法检测的漏洞，而是对当前安全假设的量化测量。

## 2. 方法论：核心思想、关键技术细节与流程

### 核心思想
- 将攻击载荷隐藏于程序化生成的浮点参数中，使得：
  - **数据层**（原始配置）：载荷不可见（以浮点数组形式存在）；
  - **重建层**（模型接收的报告）：载荷被碎片化为十六进制转储（如 xxd 格式），不包含连续散文。

### 载体设计（四种）
- **T0**（基线）：载荷以纯文本形式直接存储在配置中，重建后仍为连续散文。  
- **T1b-abl**（消融载体）：原始配置存储纯文本载荷，但重建后生成与 T3 相同的十六进制转储。  
- **T2**（频谱系数隐写）：将每个字符编码为浮点数的低三位小数位，重建后解码为连续散文。  
- **T3**（IFS 派生的浮点数组载体，核心贡献）：  
  - **编码**：将载荷字节拆分为 2 位 base-4 数字，每个数字选择四个仿射压缩映射之一，驱动迭代函数系统（IFS）：
    \[
    F_j(p) = \tfrac12 p + t_j, \quad j \in \{0,1,2,3\}
    \]
    其中平移向量 \(t_0=(0,0), t_1=(0.5,0), t_2=(0,0.5), t_3=(0.5,0.5)\)。
  - 从 \(p_0=(0.5,0.5)\) 开始，遍历 base-4 数字序列得到轨迹点，将 x 和 y 坐标存储为两个浮点数组（标记为“谐波级数”和“共振峰”）。
  - **反检测处理**（三步）：  
    1. **白化**：载荷字节与固定伪随机流异或，去除字符频率偏差。  
    2. **有界抖动**：为每个坐标添加 ±0.04 均匀噪声，解码容差仍能保证正确。  
    3. **确定性打乱**：按素数步长 7919 打乱数组，打破顺序依赖。  
  - **重建**：反转打乱、IFS 近邻解码、异或，将恢复的字节渲染为 xxd 风格的十六进制转储（每行 16 字节，附带 ASCII 侧栏）。恶意载荷关键词（如“PPB TRACE 7X3Q”）长度≤15 字符，在单行内部保持连续，但整体呈现为碎片化的十六进制数据，不出现连续散文。

### 攻击目标与评估指标
- 三种 LLM01 式间接注入目标：输出导向（让模型复制跟踪 ID）、配置令牌注入（传播伪造的角色模板）、捏造元数据注入（传播伪造的评级）。
- **主要指标：泄漏 ASR**（攻击字符串出现在模型输出中，即使模型拒绝）  
- **严格指标：Strong ASR**（结构合规成功，如输出导向需标记出现在前 150 字符内，捏造元数据需被继承且无警告）。

## 6. 主要结论与发现

1. **T3 载体在最强文本分类器防御下保持 94.3% 泄漏 ASR**，而同一载荷以纯文本形式（T0）被完全检测（0% ASR）。  
2. **2×2 消融证实**：数据层隐写（T2）可避过原始配置检查，但重建层文本被捕捉；重建层十六进制碎片化（T1b-abl）可避过重建检查，但原始配置文本被检测；同时具备两者（T3）可同时避过两个文本视图。  
3. **零样本 Prompt Guard 2 对声明性元数据载荷检测率为 0%**（包括纯文本版本），表明其分布外失效，而非架构问题。  
4. **当前 T3 存在可修补的签名**：添加 xxd 行检测器或语义验证可降至 0% ASR，但这只是对某一具体呈现格式的修补，不是对载体类的通用防御。  
5. **文本分类器失败的根本原因**：载荷在检查的文本视图中不存在可分类的文本证据（T3 原始数据为浮点数组，重建为十六进制碎片），而非分类器校准问题。

## 7. 优点

- **提出了新颖的隐写载体概念**：利用 IFS 和浮点数组将普罗大众的编码转化为程序化参数，打破了“编码就可识别”的直觉。
