## 摘要
使用工具的大语言模型（LLM）智能体面临两种不同的安全缺陷：未经授权的外部动作，以及在最终输出检查介入之前运行时内敏感明文暴露。现有防御通常只保护单个边界——规划器/运行时或动作接收端——因此本身无法同时保障两个表面。我们提出SecureClaw，一种双边界架构，将授权置于效果接收端，将明文限制置于读取边界。敏感读取通过一个可信网关，该网关用不透明句柄替换原始值，并在评估部署中，用有界摘要作为显式去分类接口。改变外部状态的写入遵循PREVIEW→COMMIT协议，其中只有可信执行者才能提交策略授权的精确规范请求。运行时仍然可以在摘要和符号引用上进行规划，但不能直接解引用秘密或执行副作用。在AgentDojo、AgentLeak和Agent Security Bench（ASB）上，SecureClaw是我们在共同测试框架中评估的唯一一种同时保持可用任务效用并在ASB上实现0%攻击成功率（ASR）、在AgentDojo上实现0.64% ASR、以及在AgentLeak的攻击平等通道上实现3.23%整体泄露（衡量最终输出和内部中继泄露）的防御方案。

## Abstract
Tool-using large language model (LLM) agents face two distinct security failures: unauthorized external actions and exposure of sensitive plaintext inside the runtime before any final output check can intervene. Existing defenses usually protect one boundary, either the planner/runtime or the action sink, and therefore do not by themselves secure both surfaces. We present SecureClaw, a dual-boundary architecture that places authorization at the effect sink and plaintext confinement at the read boundary. Sensitive reads pass through a trusted gateway that replaces raw values with opaque handles and, in the evaluated deployment, bounded summaries as an explicit declassification interface. Writes that change external state follow a PREVIEW$\rightarrow$COMMIT protocol in which only a trusted executor may commit the exact canonical request authorized by policy. The runtime can still plan over summaries and symbolic references, but cannot directly dereference secrets or perform side effects. Across AgentDojo, AgentLeak, and Agent Security Bench (ASB), SecureClaw is the only defense we evaluate in a common harness that simultaneously retains usable task utility and achieves 0\% attack success rate (ASR) on ASB, 0.64\% ASR on AgentDojo, and 3.23\% overall leak on AgentLeak's attacked parity lane, which measures final-output and internal-relay leakage.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：使用工具的大语言模型（LLM）代理在执行任务时面临两类安全缺陷：  
  1. **未授权外部动作**：攻击者可通过提示注入等操控代理执行有害的写操作（如发送邮件、转账）。  
  2. **运行时敏感明文泄露**：代理在推理过程中可能将敏感数据（如发票内容、个人信息）暴露给下游通道（如最终响应、智能体间消息、共享内存），而仅在最终输出检查时介入已为时已晚。  
- **现有不足**：现有防御通常只保护一个边界（规划器/运行时 或 动作接收器），无法同时防御两个表面。例如，输出侧过滤不能阻止运行时内部的中继泄露；而动作授权无法阻止运行时已读取的明文被泄露。  
- **研究含义**：需要一种同时控制“效果表面”（写入授权）和“明文表面”（读取限制）的双边界架构，以在不牺牲任务效用的前提下消除攻击。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将安全机制与LLM规划解耦，在**效果接收器**处实施写入授权，在**读取边界**处通过可信网关限制明文暴露。运行时仍可规划、调用工具，但无法直接获取原始敏感数据或直接提交外部动作。  
- **关键技术细节**：  
  - **读取路径**：  
    - 可信网关将敏感数据存储于受保护句柄表，向运行时返回一个**不透明句柄**（高熵符号引用）和**有界摘要**（如“来自A的发票，金额$4.2k，周五到期”）。  
    - 句柄不可伪造、不可解引用（只在可信端点可解析）。摘要被视为**显式去分类接口**（intentional declassification interface），而非意外泄露。  
  - **写入路径**：采用 **PREVIEW → COMMIT 协议**：  
    1. 运行时提议动作（intent, inputs），网关追加经过认证的 caller、session、上下文形成规范化请求 ρ。  
    2. 网关计算绑定摘要 h = HMAC-SHA256(k_bind, Canon(ρ))。  
    3. 网关查询策略引擎，获得授权工件 π，该工件绑定决策与 h。  
    4. 可信执行器独立重计算 h，验证授权工件、新鲜度、重放保护等。  
    5. 全部验证通过后，执行器解引用必要句柄并提交外部效果；否则失败关闭。  
  - **恢复机制**：当提交被拒绝时，返回结构化拒绝（仅含粗略动作类、原因码、安全提示），使工作流能安全继续（如改为“起草待确认”），而不牺牲验证不变性。  
- **形式化保证**：论文给出三个定理——请求绑定授权完整性、仅句柄场景的明文限制、有界摘要接口下的量化泄露上界。

### 6. 论文的主要结论与发现

- SecureClaw 是唯一在共同框架中同时满足以下条件的防御：  
  - ASB 上 0% ASR，AgentDojo 上 0.64% ASR，AgentLeak parity lane 上 3.23% 泄露。  
  - 在攻击行和良性行上保持高任务效用（ASB 良性 88.9%，AgentDojo 攻击效用 56.6%）。  
- **机制分解**：句柄机制（而非执行边界）是减少内部中继泄露的主要原因；执行边界（而非句柄）是阻止未授权提交的关键。两个机制互补且不可替代。  
- **残留失败**：主要来自策略允许但语义错误的动作（如选错对象）或最终输出（C1）的泄露，而非内部通道重新打开。  
- 恢复机制显著提升被阻止攻击后的效用而不增加 ASR。

### 7. 优点

- **双边界架构创新**：首次将效果表面与明文表面分开处理，互不依赖。  
- **实际可行**：在标准 LLM 推理延迟下开销极低（执行器<1 ms，授权约15 ms）。  
- **强安全保证**：提供形式化定理（请求绑定、句柄限制、量化泄露），并通过绕过试验验证。  
- **务实设计**：显式去分类接口（有界摘要）在安全性和可用性间取得平衡；恢复机制避免了简单拒绝导致的低效用。  
- **实验严谨**：覆盖主要基准、多种攻击类型、消融、绕过、稳定性检验，统计量完整。

### 8. 不足与局限

- **残留泄漏**：ASB 上0% ASR，但 AgentDojo 仍有 4/629 个成功攻击，AgentLeak 仍有16/496 泄露（主要是输出通道），它们处于策略允许或显式去分类范围内。  
- **良性效用损失**：AgentDojo 良性效用仅 60.82%，虽高于其他防御但低于 Plain 的 70.10%，说明安全可能导致规划能力受限。  
- **信任假设**：假定网关、句柄存储、策略引擎、执行器为可信；若其中任何组件被攻破，相应保证失效。  
- **依赖正确分类**：若敏感字段被错误分类（未被标记），则句柄机制不会生效。需要明确的资源清单和审计。  
- **语义正确性**：架构不保证策略允许的动作与用户意图一致；需要更丰富的绑定（主体、对象、阶段）。  
- **实验配置**：主要实验仅使用 gpt-4o-mini，跨模型检查样本较小（各50行），泛化性有待更多验证。  
- **网络真实部署**：延迟测量在本地进行，未模拟地理分布式网络延迟。
