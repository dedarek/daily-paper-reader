## 摘要
在多轮交互中，大型语言模型（LLMs）维护不断演变的上下文而非生成孤立的响应，这使其容易受到提示注入和上下文中毒攻击，其中局部看似合理的对抗性片段逐渐扭曲推理轨迹。现有防御主要过滤单个输出，往往忽略跨轮次的上下文演化，导致长程推理暴露于风险中。尽管模型上下文协议（MCP）标准化了上下文交换和工具调用，但它仅作为被动路由层，并未强制上下文稳定性。为解决这些限制，我们提出了基于博弈论的安全模型上下文协议（GT-MCP），这是一种控制器驱动的多智能体方法，将上下文管理视为闭环动态过程。GT-MCP协调三个异构LLM智能体，并通过一个信任函数选择输出，该函数联合评估与已验证上下文图之间的因果一致性、智能体之间的语义一致性以及随时间的分布漂移。当检测到不稳定时，基于回滚的自愈机制会恢复已验证的上下文，并阻止未支持片段的传播。在自适应对抗威胁模型下进行500轮交互的经验评估表明，99.6%的轮次中上下文漂移保持有界，仅0.4%需要恢复。每轮效用保持高度集中，中值为-0.19，P05=-0.72，P95=0.30；仅0.4%的情况出现低于-1的严重退化，且控制器级别没有注入尝试成功。选定的输出保持稳定的胜率超过98%，计算开销可预测，每个token的延迟为1.63e-3秒。

## Abstract
Large Language Models (LLMs) in multi-turn interactions maintain evolving context rather than generating isolated responses, making them vulnerable to prompt-injection and context-poisoning attacks in which locally plausible adversarial fragments gradually distort reasoning trajectories. Existing defenses mainly filter individual outputs and often ignore context evolution across turns, leaving long-horizon reasoning exposed. Although the Model Context Protocol (MCP) standardizes context exchange and tool invocation, it functions as a passive routing layer and does not enforce contextual stability. To address these limitations, we introduce the Game-Theoretic Secure Model Context Protocol (GT-MCP), a controller-driven multi-agent method that treats context management as a closed-loop dynamical process. GT-MCP coordinates three heterogeneous LLM agents and selects outputs through a trust function that jointly evaluates causal consistency against a validated context graph, semantic agreement among agents, and distributional drift over time. When instability is detected, a rollback-based self-healing mechanism restores the validated context and prevents unsupported fragments from propagating. Empirical evaluation over 500 interaction turns under an adaptive adversarial threat model shows that contextual drift remains bounded in 99.6% of turns, with recovery required in only 0.4%. Per-turn utility remains tightly concentrated, with median = -0.19, P05 = -0.72, and P95 = 0.30; severe degradation below -1 occurs in only 0.4% of cases, and no injection attempt succeeds at the controller level. Selected outputs maintain stable win rates above 98%, and computational overhead remains predictable, with latency per token = 1.63e-3 s.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 核心思想
- 将多轮LLM交互建模为一个受扰动力学过程（disturbed dynamical process），控制器需要使上下文状态始终接近已验证的推理流形（validated reasoning manifold）。
- 引入博弈论框架：控制器与自适应攻击者之间的重复Stackelberg博弈。控制器承诺一个基于信任的选择策略，攻击者通过注入局部看似合理但有害的扰动试图改变上下文轨迹。

### 关键技术细节

| 组件 | 说明 |
|------|------|
| **因果上下文图** | 有向图 $G_t=(V_t,E_t)$，节点$v$代表原子声明，边(u→v)表示蕴含/因果支持；节点置信权重$w_v\in[0,1]$由持久性、父节点支持等计算。 |
| **因果一致性指数 (CCI)** | 衡量候选输出声明与已验证图的匹配程度：$CCI_i(t)=\frac{1}{|S(y_i(t))|+\epsilon}\sum_{v\in S}\phi(v,G_t)$，其中$\phi$基于父节点平均置信度。 |
| **分布稳定性指标 (CDS)** | 候选输出更新上下文后，新分布与验证分布之间的JS散度：$CDS_i(t)=D_{JS}(P_{t+1}^{(i)}\| P_t)$。 |
| **跨智能体一致性 (AGR)** | 两两间语义相似度：$AGR_i(t)=\frac12\sum_{j\neq i}\kappa(y_i,y_j)$。 |
| **信任聚合** | $T_i(t)=\alpha CCI_i(t)+\beta AGR_i(t)-\gamma CDS_i(t)$，选取最高信任的候选输出。 |
| **自愈恢复** | 当 $CDS_{\hat i_t}(t)>\delta_c$ 时触发：识别低支持子图→隔离→计算最优回滚深度$d^*=\arg\min_d [\hat L(G_{t-d})+\lambda_R d]$→恢复检查点→仅重新整合仍被图支持的片段。 |

### 算法流程（文字描述）
1. 控制器收到用户查询$q_t$和已验证上下文$c_t$，结合未受信任流入$\Delta c_t$构建观测上下文$\tilde c_t = c_t \oplus \Delta c_t$。
2. 三个异构LLM智能体（GPT-5.3, Llama-3.1-70B, DeepSeek-R1）各自生成候选响应$y_i(t)$。
3. 控制器从每个候选提取原子声明，规范化为$S_i$。
4. 计算$CCI_i(t)$（与因果图支撑）、候选特定漂移$CDS_i(t)$（通过临时更新上下文后的分布变化）、$AGR_i(t)$（跨智能体语义一致性）。
5. 聚合信任分数$T_i(t)$，选择最大者作为最终输出$\hat y_t$。
6. 若$\hat y_t$的CDS超过阈值$\delta_c$，触发自愈恢复；否则直接更新上下文$c_{t+1}=\mathcal{U}(c_t,\hat y_t)$。
7. 记录审计日志（所有候选、分数、恢复信息等）。

## 8. 不足与局限

- **模型泛化性**：仅使用3个特定LLM（GPT-5.3, Llama-3.1-70B, DeepSeek-R1），结论未必适用于其他架构或领域特化模型。
- **假设限制**：假设最多一个拜占庭智能体；若两个智能体同时被协同攻击，防御可能下降。
- **有效性代理**：结构合理性（CCI、AGR）是**操作代理**，并非真正的推理正确性；当验证上下文本身存在噪音或错误时，可能产生虚假安全感。
- **计算开销**：虽然延迟可预测，但相比简单基线（单智能体7.9e-4s/令牌），GT-MCP翻倍，可能限制高吞吐场景。
- **实验轮数有限**：500轮属于中等规模；更长交互（如数千轮）中可能积累慢漂移或出现抵抗攻击。
- **外部有效性**：攻击扰动被约束为局部语义相似，真实攻击可能更隐蔽或利用系统漏洞（如MCP协议级攻击）。
- **未开源代码**：复现和社区验证存在障碍。
