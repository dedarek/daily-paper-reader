## 摘要
检索增强生成（RAG）代理日益使用跨用户会话累积的持久内存。这创造了一个新的攻击面：仅通过正常渠道交互的对手可以注入精心构造的记忆，一旦被检索，就会引导代理对未来用户的响应，而无需接触模型权重或代码。我们称之为多会话内存投毒（MSMP），并表明现有防御无法对其提供认证；静态语料防御（RobustRAG、ReliabilityRAG）假设固定知识库，而启发式过滤器则会被流畅的企业风格文本绕过。我们提出了带平滑检索的签名内存（SMSR），这是首个针对此场景具有可认证鲁棒性边界的防御。组件1在写入时添加HMAC-SHA256溯源，阻止未签名注入。组件2在查询时应用基于判定的多数投票的随机内存擦除，限制认证对手的影响。我们证明没有任何无源检索时过滤器能对自适应注入提供认证，推导出组件2的超几何认证，并形式化了“持续少数效应”，即一致的对抗性答案作为数值少数赢过基于字符串的投票，而基于判定的投票则将其消除。在15个企业场景（3150次重复试验）中，组件1将未签名变体的攻击成功率从93-100%降至0%。对于单次注入的认证对手，组件2将成功率控制在8.0%（95% CI [5.8, 10.9], n=450），低于认证的最坏情况。在端到端仅查询攻击中，代理自身写入毒物而非预先植入时，SMSR在实时代理栈上将成功率从65.3%降至5.3%（n=150，非重叠CI）。干净查询效用为90%（组件1）和85%（组合）。

## Abstract
Retrieval-augmented generation (RAG) agents increasingly run with persistent memory that accumulates across user sessions. This creates a new attack surface: an adversary interacting only through normal channels can inject crafted memories that, once retrieved, steer the agent's responses for future users, without touching model weights or code. We call this Multi-Session Memory Poisoning (MSMP) and show that no existing defence certifies against it; static-corpus defences (RobustRAG, ReliabilityRAG) assume a fixed knowledge base, and heuristic filters are bypassed by fluent enterprise-style text. We present Signed Memory with Smoothed Retrieval (SMSR), the first defence with a certified robustness bound for this setting. Component 1 adds HMAC-SHA256 provenance at write time, blocking unsigned injection. Component 2 applies randomised memory ablation with verdict-based majority voting at query time, bounding the influence of authenticated adversaries. We prove that no provenance-free retrieval-time filter can certify against adaptive injection, derive a hypergeometric certificate for Component 2, and formalise the Consistent Minority Effect, whereby a consistent adversarial answer wins string-based voting as a numerical minority while verdict-based voting removes it. Across 15 enterprise scenarios (3,150 repeated trials), Component 1 cuts attack success from 93-100% to 0% for all unsigned variants. For an authenticated adversary with a single injection, Component 2 holds success to 8.0% (95% CI [5.8, 10.9], n=450), below the certified worst case. In an end-to-end query-only attack where the agent itself writes the poison rather than it being pre-seeded, SMSR reduces success from 65.3% to 5.3% (n=150, non-overlapping CIs) on a live agent stack. Clean-query utility is 90% (Component 1) and 85% (combined).

- **整体含义**：该论文首次为动态运行时内存投毒场景提供了具有可认证鲁棒性边界的防御方案，填补了LLM代理安全领域的空白，对现实企业级RAG部署的安全加固有重要意义。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 采用两层防御：**Component 1（写入时溯源）** 与 **Component 2（查询时随机消融+裁决投票）**，组合实现可认证安全。

### 关键技术细节
- **Component 1：HMAC-SHA256 溯源**  
   - 每次合法写入时，由可信服务器签名模块生成HMAC-SHA256标签：`τ_i = HMAC_K(content_i ∥ session_id_i ∥ timestamp_i)`。  
   - 查询时仅保留拥有有效标签的记忆。  
   - 安全性：在HMAC伪随机性假设下，未知道K的对手产生合法标签的概率≤2⁻²⁵⁶。
- **Component 2：随机内存消融 + 裁决聚合**  
   - **过获取（over-fetch）**：对查询检索 top-m 个已签名候选（m > k）。  
   - **随机消融**：独立重复 n_runs 次，每次从m个候选中无放回均匀采样k个条目。  
   - **LLM生成**：对每个样本上下文，调用LLM生成响应。  
   - **裁决投票**：使用一个LLM裁决器（Judge）判定每条响应为“正确/恶意/其他”，最终多数裁决决定输出。  
- **形式化证书**：Theorem 2 给出基于超几何分布的恶意输出概率上界 δ：
  \[
  \delta(t', m, k, n_{\text{runs}}) = \sum_{i=\lceil n_{\text{runs}}/2 \rceil}^{n_{\text{runs}}} \binom{n_{\text{runs}}}{i} (1-p_{\text{clean}})^i p_{\text{clean}}^{n_{\text{runs}}-i}
  \]
  其中 \( p_{\text{clean}} = \frac{\binom{m-t'}{k}}{\binom{m}{k}} \)，t' 是检索池中的对手条目数。

### 算法流程
1. **写阶段**：所有合法记忆通过签名模块写入，获得HMAC标签。
2. **查阶段**：
   - 调用 RETRIEVE 从已签名池中获取 top-m 候选。
   - 重复 n_runs 次：均匀采样 k 条目 → LLM 生成响应 → 裁决器给出标签。
   - 多数裁决标签对应的第一个响应作为最终输出。

## 6. 主要结论与发现

1. **Component 1（HMAC溯源）**：对所有未签名注入变体（直接、泛洪、隐蔽、绕过）实现 ASR=0%。
2. **Component 2（随机消融+裁决投票）**：
   - 生产规模（m=20, 20种子）：单次已签名注入ASR=8.0%（95% CI [5.8,10.9]），低于证书δ=10.4%（n_runs=5）。若n_runs=7，δ可降至7.1%。
   - 小存储（m'=10+t）：单次直接注入ASR=37.8%（CI [33.4,42.3]）低于δ=41.5%；泛洪变体紧贴边界。
   - t=2,3时，直接注入ASR低于对应δ，泛洪位于边界（证书紧致）。
3. **裁决者可靠性**：Haiku与Sonnet裁决一致性κ=0.955（n=84）。
4. **跨模型通用性**：Sonnet作为代理在20种子存储上得到相同ASR=8.0%（n=450）。
5. **端到端查询式攻击**：代理自身写入毒物时，SMSR将ASR从65.3%降至5.3%（n=150，非重叠CI）。
6. **效用**：Component 1保留90%清洁查询正确率；完整SMSR降至85%。
7. **一致少数效应**：基于字符串的多数投票易被攻击（因为干净答案多样化），基于裁决的投票消除了该影响。

## 8. 不足与局限

- **证书局限**：当t≥m/2时，对手控制多数池，δ≈1，防御无效。实际操作需根据假定对手预算选择m（经验法则m≈19t）。
- **适应性对手威胁**：论文承认适应性对手可能通过观察裁决结果来构造绕过裁决的响应，不过认为比直接骗过LLM更难。
- **对模型服务依赖性**：依赖外部LLM API，有成本、延迟和可靠性问题；裁决器本身可能被针对（尽管当前未实验）。
- **场景局限**：仅使用合成企业政策场景，未涉及更长、更复杂记忆或开放域问答；未进行人工标注，依赖LLM裁决可能引入偏差。
- **缺少对多代理系统的扩展**：论文提到作为未来工作，目前不直接支持。
- **部署要求**：需要可信签名基础设施（HSM/密钥管理），任何未经过签名oracle的写路径都会成为绕过。
- **对已签名对手的防御有限**：当t较大时（如t=3），ASR接近93-100%（尽管证书预测很高），实际保护不足。
