## 摘要
大型语言模型（LLM）代理通过在运行时加载代理技能来扩展其能力，这些技能将自然语言规范（SKILL.md）与可执行脚本和资源配对。由于技能的行为依赖于自然语言指令和可执行代码，评估其安全性需要跨模态推理，从而创造了新的语言与代码攻击面。攻击者可以在SKILL.md中呈现良性工作流，同时嵌入隐式指令，引导代理泄露敏感文件，即使脚本看起来无害。这一攻击面尚未得到充分研究；先前的工作仅将技能视为提示注入向量或静态代码工件，使得跨模态交互产生的攻击在很大程度上未被测量。在我们的评估中，开源和商业技能扫描器分别仅检测到2%-8%和9%-17%的此类攻击。为解决这一差距，我们引入了SkillMutator，这是首个针对代理技能安装时语言与代码跨模态攻击检测的基准测试。它模拟了跨13个攻击类别的对抗性突变过程，利用扫描器反馈迭代地改进恶意技能，使注入的行为与合法工作流难以区分。我们进一步提出了一个四阶段推理轨迹蒸馏框架，将前沿教师模型的轨迹蒸馏到较小的开放权重模型中。这产生了一个本地可部署的扫描器，避免了第三方数据暴露和过高的API成本。在最强的SkillMutator子集（n=76）上，我们的蒸馏模型（Qwen2.5-Coder-7B-Instruct）将检测率从17.1%提升至88.2%，超越了GPT-4o-mini（23.7%）和GPT-5.4-mini（79.0%），并达到了前沿级别的GPT-5.4（86.8%）。这些结果表明，在不依赖昂贵的前沿模型的情况下，针对跨模态攻击的实际防御是可行的。

## Abstract
Large language model (LLM) agents increasingly extend their capabilities at runtime by loading Agent Skills, which pair natural-language specifications (SKILL.md) with executable scripts and resources. Because a skill's behavior relies on both natural-language instructions and executable code, assessing its safety requires cross-modal reasoning, creating a new language-and-code attack surface. Attackers can present a benign workflow in SKILL.md while embedding implicit directives that steer the agent to exfiltrate sensitive files, even if the scripts appear harmless. This attack surface remains understudied; prior work treats skills merely as prompt-injection vectors or static code artifacts, leaving attacks emerging from cross-modal interactions largely unmeasured. In our evaluation, open-source and commercial skill scanners detect only 2%-8% and 9%-17% of such attacks, respectively. To address this gap, we introduce SkillMutator, the first benchmark for install-time detection of language-and-code cross-modal attacks on Agent Skills. It emulates an adversarial mutation process across 13 attack categories, iteratively refining malicious skills using scanner feedback to make injected behaviors indistinguishable from legitimate workflows. We further propose a four-phase reasoning-trajectory distillation framework to distill frontier-teacher traces into smaller open-weight models. This produces a locally deployable scanner avoiding third-party data exposure and excessive API costs. On the strongest SkillMutator subset (n=76), our distilled model (Qwen2.5-Coder-7B-Instruct) improves detection from 17.1% to 88.2%, surpassing GPT-4o-mini (23.7%) and GPT-5.4-mini (79.0%), and reaching frontier-level GPT-5.4 (86.8%). These results show practical defense against cross-modal attacks is feasible without relying on costly frontier models.

## 2. 方法论：核心思想与关键技术细节

### 2.1 SkillMutator基准生成框架
模拟自适应攻击者，从良性技能出发，通过四个阶段生成扫描器难以识别的恶意变异：

1. **技能分析**：解析目标技能的SKILL.md、脚本、配置等，生成结构化摘要（用途、行为流、所用工具、可信操作、敏感资源、可修改文件）。
2. **隐身感知攻击选择**：对每项攻击类别（共13类，分4组）计算“隐身得分”（1–100），根据技能上下文评估攻击是否与原有目的兼容、能否伪装为正常行为。保留得分>80的类别；若不足3个则保留最高分的3个，保证多样性。
3. **场景生成**：为选中类别生成具体攻击场景，指定表面行为、隐藏恶意行为、注入指令、激活条件、预期影响。
4. **技能变异**：基于原始技能文件和场景，修改SKILL.md和辅助脚本，使注入行为看起来自然。恶意行为通过SKILL.md指令与脚本的组合间接触发（例如图2中的数据泄露通过构造审计轨迹将敏感令牌附加到用户消息中）。

**迭代规避精炼**：每轮变异完成后用三个扫描器（规则、商业、LLM语义）检测，对检测到的变异，根据反馈重新生成更隐蔽的版本。精炼提示包含四条规避原则（重命名、间接引用、自然语言重写、伪装为维护操作）和禁止模式表（从skill-security-scan规则中提取的6类函数族，如网络出口、敏感资源访问等）。**这模拟了知晓扫描器规则的知识型攻击者**。

### 2.2 四阶段推理轨迹蒸馏框架（本地扫描器）
为了将前沿LLM（教师模型）的跨模态检测能力迁移到小模型，设计了四阶段分析模式，生成结构化推理轨迹，然后通过LoRA蒸馏到7B开源模型（如Qwen2.5-Coder-7B-Instruct）。

- **阶段1：目的定位**：从SKILL.md提取最小声明目的（动词-对象对），枚举完成该目的必需的六轴操作（文件系统、网络、状态、命令、权限、输出内容）。允许列表故意狭窄，超出者视为候选越界行为。
- **阶段2：越界检测**：扫描所有文件（按节/函数分割），识别超出阶段1允许列表的操作。标记为`scope_expansion`（越界）或`unsafe_composition`（安全组合问题，如shell注入）。
- **阶段3：安全原则推理**：将阶段2的证据映射到9条安全原则（表2，如目的-行为对齐、信任边界完整性、隐蔽条件等）。将描述层面的探测命中（来自阶段1）合并到相同原则下。
- **阶段4：攻击类别标记**：基于阶段3的证据，给13个攻击类别打标签。要求每个引用的文件和节必须已出现在阶段3，禁止引入新证据，确保分类可追溯。

**确定性精炼**：当教师模型的阶段4遗漏目标攻击类别时，利用变异日志中的元数据（注入内容、摘要）合成一个证据支持的条目，插入到阶段3对应原则和阶段4目标类别中，保证训练数据全覆盖。

**推理时预填充**：在推理时强制用“## Phase 4: Attack Category Labeling”初始化模型输出，引导解码进入分类阶段，提升检测率（仅对Qwen模型有效）。

## 6. 主要结论与发现

1. **现有扫描器对跨模态攻击无效**：规则和商业扫描器检测率<17%，提示注入检测器<20%。专有LLM扫描器（GPT-5.4）达到86.8%，但成本高且存在隐私风险。
2. **四阶段蒸馏使7B本地模型达到前沿性能**：Qwen2.5-Coder-7B-Instruct微调后检测率从17.1%提升至88.2%，超越GPT-4o-mini（23.7%）、GPT-5.4-mini（79.0%），并略超GPT-5.4（86.8%）。
3. **每个阶段均贡献显著**：阶段3（原则推理）提供最大增益（+32.9pp），确定性精炼和预填充进一步提升了9.2–11.9pp。
4. **隐身选择和迭代精炼增加难度**：隐身选类比全部应用所有攻击类别平均降低检测率7.1pp；精炼后检测率较初始变异下降约6–8pp。
5. **本地扫描器在成本-隐私-能力上最优**：每检测一次的成本仅$0.00469，是GPT-5.4的1/6.8，且无需将技能内容发送给第三方。
6. **跨模型家族泛化有效**：Qwen、Llama、Mistral、Gemma微调后均大幅提升（+48.7pp至+75.0pp），但预填充仅对Qwen有益。

## 7. 优点

- **问题新颖且重要**：首次系统研究语言-代码跨模态安全漏洞，填补了代理技能安装时检测的空白。
- **方法论系统完整**：从攻击者视角生成基准（四阶段变异+迭代精炼），同时提出防御蒸馏框架，两端设计紧密耦合。
- **实用部署导向**：本地部署、低开销、不暴露数据到第三方，支持企业敏感环境。
- **实验设计严谨**：训练/评估集分离、多种消融、双评判器、实际市场验证、成本分析，结果可信度高。
- **开源贡献**：发布基准数据集、变异、微调及评估代码，便于复现和后续研究。

## 8. 不足与局限
