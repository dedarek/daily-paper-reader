## 摘要
我们提出了一种基于RAG的大语言模型推荐中安全训练的可重现失败模式——"注射悖论"：嵌入在检索文档中的提示注入反而会适得其反，将目标品牌的推荐率压制到低于未注入的基线水平。在经安全训练的Claude模型中，包含提示注入的文档的推荐率急剧下降，并且这种压制会传播到同一品牌的其他未修改文档。在Claude Opus 4.6中，尽管语料库中4个品牌文档中只有1个包含注入，但目标品牌在所有50次试验中的top-2推荐率从54%的基线降至零。反事实实验和三个品牌的测试均复现了这一方向性模式。而在测试的GPT模型中出现了相反的结果，同样的注入反而增加了推荐，这表明不同模型家族中类似注入的上下文对推荐行为的影响存在差异。这些发现提出了反向攻击场景的技术可能性：攻击者可以将注入嵌入竞争对手的文档中，通过安全敏感模型行为压制竞争对手的品牌。

## Abstract
We present a reproducible failure mode of safety training in RAG-based LLM recommendation -- the Injection Paradox -- in which prompt injections embedded in retrieved documents backfire against the attacker, suppressing the target brand below the injection-free baseline. In safety-trained Claude models, documents containing prompt injections suffer a sharp drop in recommendation rate, and this suppression propagates beyond the injected document to unmodified documents of the same brand. In Claude Opus 4.6, the target brand drops from a 54% baseline to zero top-2 recommendations across all 50 trials, even though only 1 of 4 brand documents in the corpus contains an injection. The directional pattern is reproduced in counterfactual experiments and across three brands. A contrasting result across the GPT models tested, where the same injection instead increases recommendations, suggests model-family differences in how injection-like context affects recommendation behavior. These findings raise the technical possibility of a reverse-attack scenario in which an adversary embeds injections in a competitor's documents to suppress the competitor's brand via safety-sensitive model behavior.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：随着LLM推荐系统的快速普及（如ChatGPT Search），间接提示注入（indirect prompt injection）成为新兴威胁——攻击者通过在网页文档中嵌入隐藏指令来操控AI推荐。2026年微软安全报告指出已有31家公司遭受此类攻击。为应对攻击，模型提供商采用了安全训练（如OpenAI的RLHF、Anthropic的Constitutional AI）。然而，安全训练与间接提示注入之间的相互作用尚未被充分研究。

- **核心问题**：安全训练是否会因“矫枉过正”而产生意外的副作用？具体而言，当模型检测到文档中的提示注入模式时，是否会过度压制被注入品牌的推荐，甚至波及同品牌其他未篡改文档？

- **整体含义**：论文发现了一个反直觉的“注射悖论”——在安全训练的Claude模型中，提示注入不仅没有提升目标品牌的推荐，反而导致其推荐率显著下降（甚至低于未注入基线）。这揭示了一种新的失败模式：安全训练可能创造新的攻击面，使攻击者可以通过向竞争对手文档中嵌入注入来反向压制对方品牌。

## 2. 方法论

### 核心思想

- 测量不同条件下模型对目标品牌的top-2推荐命中率，对比基线和跨模型差异。

### 关键技术细节
- **触发类型**：直接型（T1：显式推荐指令，隐藏于文档的meta块、编辑注、评估检查表中）；结构型（T2：文档长度扩展3倍）；隐式型（T3点击诱饵、T5情感夸张、T6时间性暗示、T7虚假权威）。
- **条件设计**：
  - **基线**：原始语料（无修改）
  - **隐式条件**：仅使用T5+T6+T7（无注入）
  - **注入条件**：仅使用T1（约10%长度增加）
  - **组合条件**：T1+T2+T3+T5+T6+T7
- **反事实实验**：将被注入文档替换为Sennheiser文档（非目标品牌），形成3个Edifier文档的基线，比较与注入条件下的品牌推荐率差异，以验证抑制是否传播到同品牌未修改文档。
- **跨品牌复制**：对Apple AirPods Pro 3（8个文档）和Samsung Galaxy Buds3 Pro（7个文档）采用相同设计重复实验。
- **防御消融**：在系统提示中增加四种安全指导（D1-D4），依次为通用警告、注入特定警告、注入警告+决策锚点（列举客观规格）、仅有决策锚点。

## 6. 主要结论与发现

1. **注射悖论成立**：在安全训练的Claude Sonnet/Opus中，提示注入导致目标品牌推荐率显著下降（Sonnet -18个百分点，Opus -46个百分点），而GPT模型下相同注入反而提升推荐（GPT-4o-mini +23个百分点）。Haiku处于中间（无显著变化）。

2. **品牌级抑制传播**：虽然语料库中仅1/4的Edifier文档包含注入，但其余3个未修改文档的推荐也被完全压制（Opus组合条件下0%命中率）。反事实实验表明，注入条件（0%）的推荐率甚至低于拿掉被注入文档后的3文档基线（28%），即出现“比不存在更差”效应。

3. **跨品牌可复现**：在Edifier, Apple, Galaxy三个品牌上，抑制方向一致（从Haiku到Sonnet到Opus依次增强），但抑制深度因品牌知名度不同（Edifier -54 pp > Apple -34 pp > Galaxy -14 pp）。

4. **抑制机制非全局性**：当注入移至Apple文档时，对Edifier（依赖语料的品牌）的推荐无显著影响，排除了“语料全局不信任”假说。支持“品牌级信任度惩罚”机制：模型将注入定性为安全违规后，信任下降作用于整个品牌实体。

5. **防御措施效果有限**：四种提示级防御（D1-D4）均无法完全恢复基线。最有效的D4（提供客观规格锚点）在Opus上恢复至36%（仍低于基线54%），在Sonnet上恢复至22%（低于基线26%）。防御效果不跨模型复现，且恢复主要来自外部决策标准，而非注入防御本身。

6. **反向攻击可能性**：攻击者可以通过向竞争对手文档中嵌入注入来利用安全敏感性，实现品牌级压制，且受害者本身文档未被篡改，难以定位根源。

## 8. 不足与局限

- **静态语料假设**：实验忽略检索、分块、重排序等实际RAG流程，真实场景中的效应可能不同。
- **域和语言单一**：仅测试无线耳塞（消费品）和韩语，推广性有限。
- **模型覆盖有限**：仅测试GPT和Claude两个家族，未涉及Llama, Gemini等。Claude内部Haiku与Sonnet/Opus的差异无法分离规模与对齐配置的贡献。
- **机制未充分验证**：提出的“品牌级信任度惩罚”假说仅基于行为观察，缺乏注意力分析等机械解释验证。不能排除表面关键词匹配等更简单解释。
- **防御评估不完整**：仅测试了注入条件（T1）下的防御，未在组合条件（更接近真实攻击）下验证。防御效果也不跨模型一致。
- **统计功效注意**：部分子实验（如Galaxy-Opus的-14 pp）在n=50下未达显著性（p=0.176），需要更大样本量。
- **对抗性假设限制**：假设攻击者仅修改一个文档（2.5%语料），未测试更高比例或更隐蔽的注入方式。
