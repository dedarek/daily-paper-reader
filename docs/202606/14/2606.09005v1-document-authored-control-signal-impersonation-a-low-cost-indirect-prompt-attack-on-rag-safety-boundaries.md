## 摘要
检索增强生成（RAG）系统通常将用户查询、检索文档、元数据、系统标签和任务指令序列化为一个自然语言提示。我们研究了这种设计中的源权威边界失效：攻击者撰写的检索文本可以模仿那些对模型看似具有控制相关性的元数据、来源、权威或披露策略信号。我们将这种模式称为文档撰写的控制信号模仿（DACSI）。DACSI是间接提示注入中一种非命令式、类元数据的载荷子类。其核心教训很简单：文档撰写的标签是数据，而非策略。命令式注入要求模型忽略、覆盖或违反策略；而DACSI则追问，当RAG提示渲染将可信与不可信文本合并到同一自然语言通道时，不可信的文档文本是否可能被错误归因于授权控制信号。

我们在六个模型设置、提示压力级别、注入基线、信号分类、RAG中介管道、系统控制探测、源权威归因探测以及合成金丝雀格式上评估了DACSI。我们按模型体制解释证据，而非作为六个相同的重复：DeepSeek V4 Pro和Qwen3.5-397B提供了最清晰的正向提升，DeepSeek V4 Flash是高易感性设置，GPT-5.5和Gemini 3.1 Pro Low是带有选定残留风险的强边界探测，而GLM-4.7是饱和泄露边界案例。在这些体制中，DACSI值得单独评估，因为它使用了无命令的元数据/来源/策略表面，遵循RAG特定的源权威路径，并对源/通道分离有响应。源权威探测是行为归因证据，而非内部机制的证明。

## Abstract
Retrieval-augmented generation (RAG) systems often serialize user queries, retrieved documents, metadata, system labels, and task instructions into one natural-language prompt. We study a source-authority boundary failure in this design: attacker-authored retrieved text can impersonate metadata, provenance, authority, or disclosure-policy signals that appear control-relevant to the model. We call this pattern Document-Authored Control-Signal Impersonation (DACSI). DACSI is a non-imperative, metadata-like payload subclass within indirect prompt injection. Its central lesson is simple: document-authored labels are data, not policy. Command-style injection asks the model to ignore, override, or violate policy; DACSI asks whether untrusted document text can be misattributed as an authorized control signal when RAG prompt rendering collapses trusted and untrusted text into the same natural-language channel.   We evaluate DACSI across six model settings, prompt-pressure levels, injection baselines, signal taxonomies, RAG-mediated pipelines, system-control probes, a source-authority attribution probe, and synthetic canary formats. We interpret the evidence by model regime rather than as six equal replications: DeepSeek V4 Pro and Qwen3.5-397B provide the cleanest positive lift, DeepSeek V4 Flash is a high-susceptibility setting, GPT-5.5 and Gemini 3.1 Pro Low are strong-boundary probes with selected residual risks, and GLM-4.7 is a saturated leakage boundary case. Across these regimes, DACSI warrants separate evaluation because it uses a command-free metadata/provenance/policy surface, follows a RAG-specific source-authority path, and responds to source/channel separation. The source-authority probe is behavioral attribution evidence, not proof of an internal mechanism.

- **整体含义**：DACSI 是间接提示注入的一个**非命令式、类元数据载荷子类**。它与传统命令式注入的区别在于：传统注入要求模型“忽略、覆盖或违反策略”；而 DACSI 则利用源混淆，让模型误将文档文本当作授权控制信号。该攻击无需显式命令，成本极低（仅需撰写可检索文本），但可能绕过命令词导向的检测器，并在某些模型上造成与命令式注入相当的泄露风险。论文强调**文档撰写的标签是数据，而非策略**，而现有 RAG 系统设计并未在提示渲染层面严格区分这两者。

### 论文的主要结论与发现

1. **可行性**（E1）：DACSI 的效果依赖于**复制压力**——弱提示下泄露不明显，强提示（要求精确复制）下泄漏显著（如 DeepSeek V4 Pro 从 0.5% → 28.0%）。模型-任务条件性，非无条件泄漏。
2. **独立子类证据**（E2）：DACSI 即使**不包含任何命令词**（0 命令词 vs 传统注入 3.25 个命令词），在多个模型上仍可达到与命令式注入接近甚至更高的 UDR（如 DeepSeek V4 Flash：DACSI 均值 89.3% vs 传统 85.3%）。这表明其**行为路径不同**，值得单独评估。
3. **信号家族选择性**（E3）：披露政策、权威、来源标签风险最高；消极控制（“不要引用”）和无意义标签效果弱。Gemini 3.1 Pro Low 仅在“官方注释”类标签下有残留风险（13.5%→0%基线），其他信号无效果，进一步支持分类的语义性。
4. **跨 RAG 管道**（E4）：DACSI 在玩具嵌入、BM25、混合嵌入及 LangChain 式检索中均存活，但检索设计（如分块）会中介风险（BM25 较弱，混合嵌入最强）。LangChain 式重现中 DeepSeek V4 Pro 从 1.5% 升至 34–41.5%，Gemini 仍稳定为 0%。
5. **系统控制有效性**（E5）：**系统-文档通道分离**最稳健（DeepSeek V4 Pro 上 DACSI 从 19%→0.2%）；通用警告可能适得其反（Qwen3.5-397B 上 UDR 从 85.4%升至 93.5%）；检索时**隐秘值红化**可以将泄露降为 0%。
6. **源权威归因证据**（M1）：相同权威语义从系统来源移至文档来源时，DeepSeek V4 Pro 的 UDR 从 15% 升至 25%（系统文本本身也可引起泄露）；但若改为引用文本、中性元数据或显式源分离，则泄露消失。Gemini 互补：仅系统来源引起泄漏，文档来源无效果。这支持**源-权威边界模糊**是主要机制。
7. **凭证格式鲁棒性**（E6）：DACSI 效应不限于某一特定凭证格式；多数格式均有效，但程度因模型和格式而变（如 Gemini 在 database URL 格式上基线泄漏严重，不作为主要证据）。

### 优点

- **创新性**：提出了一个新的提示注入子类（DACSI），填补了现有研究在非命令式、元数据/权威/策略表面攻击上的空白。将 RAG 提示渲染中的源-权威边界失效正式化。
- **实验设计系统**：采用配对基线、多模型体制、多维度变化（压力、家族、检索、防御、格式），并严格统计处理（置信区间、配对检验、多重比较校正）。公开完整 artifact 保证可复现。
- **精确界定**：反复声明 DACSI 是间接提示注入的子类，避免过度推广。结论明确“文档撰写的标签是数据，不是策略”，并给出可操作的缓解建议（通道分离、红化）。
- **鲁棒性证据**：通过 M1 源放置探测、E3 信号选择性、E6 格式鲁棒性检验，排除了“仅凭命令词移除”或“单一格式”的解释。

### 不足与局限

- **合成度量**：UDR 仅衡量合成凭证的字符串再现，不直接衡量真实凭证泄露、账户接管或下游利用。真实系统还有访问控制、输出过滤、监控层，论文未评估这些整体防御栈。
- **外部有效性有限**：覆盖了 6 个模型，但未在完整的端到端企业部署（含生产级向量数据库、重排序器、密钥管理系统等）中测试。模型版本通过 API 访问，版本细节可能变化。
- **提示模板敏感**：作为提示工程实验，结果对措辞敏感。虽已使用匹配基线、负控制、多重表面，但仍不能完全排除特定模板偏差。
- **防御评估深度不足**：系统控制探测（如 Spotlighting、StruQ、CaMeL）仅测试了简化版本，未进行全面的水平基准测试。不声明提出的通道分离优于既有防御。
- **内部机制未解释**：源权威探测提供的是行为归因证据（并非内部归因电路），机制仍然不明确。
- **未覆盖攻击变体**：信号分类虽广泛但可能不完整；未考虑对抗性提示之间的相互作用或更复杂的多步攻击。
