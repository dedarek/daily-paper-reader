## 摘要
代理技能让大语言模型代理能够重用指令、资源、工具和工作流，但它们也为恶意行为创造了新的隐藏空间。一个技能可能在文档或代码中看起来无害，但只有在特定用户请求、本地资产、持久状态或多步工具交互的调用下才会变得有害。这使得纯静态审查变得脆弱。我们提出了运行时技能审计（Runtime Skill Audit, RSA），这是一种动态分析方法，通过询问技能中介代理在目标化运行时条件下实际执行的操作来审计技能。RSA并不用相同的通用任务测试每个技能，而是剖析风险相关接口，准备执行这些接口所需的上下文，并根据产生的轨迹证据分配安全标签。我们在OpenClaw上实现了RSA，并在100个技能上评估了它，与代表性的静态基线进行了比较。RSA实现了90.0%的准确率，真正例率为88.0%，假正例率为8.0%，相比最佳静态基线，准确率提高了13.0个百分点。在自我进化攻击下，静态检测器在一两轮后崩溃，而RSA在每轮中仍能检测到20个恶意技能中的19-20个。

## Abstract
Agent skills let LLM agents reuse instructions, resources, tools, and workflows, but they also create a new place for malicious behavior to hide. A skill may look benign in its documentation or code while becoming harmful only when it is invoked with particular user requests, local assets, persistent state, or multi-step tool interactions. This makes purely static vetting brittle. We present Runtime Skill Audit (RSA), a dynamic analysis method that audits skills by asking what the skill-mediated agent actually does under targeted runtime conditions. Instead of testing every skill with the same generic tasks, RSA profiles risk-relevant interfaces, prepares the execution context needed to exercise them, and assigns security labels from the resulting trace evidence. We instantiate RSA on OpenClaw and evaluate it on 100 skills against representative static baselines. RSA achieves 90.0\% accuracy with an 88.0\% true positive rate and an 8.0\% false positive rate, improving accuracy by 13.0 percentage points over the best static baseline. Under self-evolving attacks, static detectors collapse after one or two rounds, while RSA continues to detect 19--20 out of 20 malicious skills across rounds.

# Runtime Skill Audit: 针对代理技能安全的目标化运行时探测

## 1. 论文的核心问题与整体含义（研究动机和背景）

随着高自主性LLM代理（如OpenClaw、Claude Code、Codex）的兴起，代理技能（agent skills）作为能力扩展、模块化和可复用任务指导的机制被广泛采用。然而，开放技能生态也扩大了攻击面：恶意或受损的技能可能将不安全行为隐藏在看似无害的指令、工作流和工具交互模式背后。**核心问题**是：恶意行为往往是**行为性的**而非纯文本性的——技能文档看起来无害，但仅在特定运行时条件（特定用户请求、本地资产、持久状态、多步工具交互）下才会暴露危险。静态审查（规则匹配、LLM基于工件的判断）无法捕获这种依赖运行时的行为，因为其仅观察工件层面，无法判断规则是否被执行、接触了哪些文件、或良性工作流在运行时上下文中是否变得有害。

**整体含义**：需要一种行为导向的动态分析方法，通过实际执行来揭示技能的潜在恶意行为，从而克服静态审查的局限性。

## 2. 论文提出的方法论：核心思想与关键技术细节

### 核心思想
RSA是一种**动态分析方法**，通过**风险引导的任务生成**、**上下文感知的运行时执行**和**轨迹基底的判决**，将技能审计从工件级审查转变为行为级审计。核心问题是：技能中介的代理在目标化运行时条件下**实际做了什么**？

### 关键技术细节
RSA包含三个设计原则：

1. **风险引导任务生成（Risk-guided tasking）**：首先使用LLM分析技能工件，基于预定义的工具分类法和攻击模式知识，推断技能可能调用或影响的平台工具，并将其映射到安全相关的功能组（如文件访问、shell执行、网络交互、内存访问等）。这生成一个结构化安全轮廓，记录技能的可能良性目的、工具接口、敏感资源以及攻击模式。然后从这些轮廓生成可执行的任务，而非通用任务列表。任务包括合理的用户请求、沙箱准备（实例化相关上下文）以及可重复执行的约束。

2. **上下文感知运行时执行（Context-aware runtime execution）**：在隔离的代理环境中执行任务，并**根据任务生成的要求准备运行时上下文**（如相关文件、目录、模拟凭证、持久状态）。恶意分支往往只有在技能与风险接口期望的资产或状态交互时才变得可见。执行期间记录结构化的轨迹（工具调用、文件操作、网络请求、内存交互等）。此外，使用**自修复循环**：当执行失败（如缺少资源）时，修复LLM诊断失败原因并重新运行（有限预算内）；对于依赖外部应用/平台功能的技能，使用确定性的假应用垫片（如模拟Gmail命令）来记录技能如何使用该接口，而不影响最终标签。

3. **轨迹基底判决（Trace-grounded judgment）**：基于收集的执行轨迹进行标签分配，而非基于工件。评估器检查：良性目标是否完成、探测的风险表面是否被激活、观察到的行为是否表示安全相关的误用。四种判决：`benign executed`、`harmful blocked`（恶意分支存在但被代理阻止）、`harmful executed`（轨迹包含有害行为）、`uncertain`。

### 辅助组件
- **知识库（Knowledge base）**：固定资源，包含工具分类法、风险表面映射、攻击模式模板、标签定义、评估清单（类似MITRE ATT&CK）。
- **记忆模块（Memory）**：从先前执行中更新，存储有效触发器、重复假阳性、漏报、轨迹证据摘要，指导后续任务生成和评估，但不直接决定当前判决。

## 6. 论文的主要结论与发现

1. **RSA显著优于静态基线**：在100技能数据集上，RSA准确率90.0%（88.0% TPR, 8.0% FPR），比最佳静态基线SkillScanner（77.0%准确率）提升13个百分点，且FPR从12%降至8%。
2. **自进化攻击下鲁棒性**：静态检测器（Aguara、SkillScanner）在1-2轮变异后检测率骤降至0，而RSA在全部轮次中稳定检测19-20/20个恶意技能。原因是恶意技能必须暴露恶意分支才能达到目标，而动态执行能捕获行为，不受工件表面形式变异的欺骗。
3. **知识引导有效**：消融实验显示，加入知识引导后TPR从82%升至88%，FPR从16%降至8%，说明知识帮助生成更精准的任务和解释轨迹。
4. **行为基底的判决能区分“有害但被阻止”与“有害且执行了”**：案例中恶意分支存在但被代理阻止，标签为`harmful blocked`，体现了动态审计的细粒度。

## 7. 优点

- **动态分析填补静态审查盲区**：能够发现仅依赖运行时上下文的恶意行为（如环境依赖、触发条件、多步执行），这是静态方法无法做到的。
- **风险引导的任务生成**：根据技能轮廓定制任务，而非使用通用任务，提高了探测的针对性和效率。
- **上下文感知执行**：通过材料化相关资产（文件、凭证、状态等）使恶意分支可被触发，避免因缺少上下文而漏检。
- **轨迹基底判决**：基于实际行为而非工件推测，提供了更可靠的证据；四个判决分类合理，区分“被阻止”与“被执行”。
- **自修复循环与假应用垫片**：增强了执行鲁棒性，减少因环境问题导致的无效运行，并能在缺失外部服务时仍记录意图。
- **知识库与记忆模块**：分离固定知识与运行时经验，引导探测且不直接决定判决增加了可解释性。
- **可复现**：代码开源。

## 8. 不足与局限

- **数据集规模有限**：仅100个技能，且恶意技能部分来自生成和改编，真实世界中攻击多样性可能更高；良性技能覆盖也有限。
- **依赖LLM质量**：轮廓分析、任务生成、判决均使用LLM，模型能力、提示设计、一致性直接影响结果；未进行不同LLM的对比或鲁棒性测试。
- **动态执行非穷举**：生成的测试任务可能未覆盖所有恶意触发条件，特别是需要特定外部状态、长时间序列或罕见输入的情况；安全标签反映了特定探测下的行为，不能完全保证技能绝对安全。
- **平台特定**：实验仅在OpenClaw上实例化，泛化到其他代理平台（如Claude Code、Codex）需要适配工具分类法和执行沙箱。
- **未报告计算成本**：动态执行每个技能需要准备沙箱、执行、可能修复，计算开销高于静态方法；论文未分析时间或资源消耗。
- **未考虑低层沙箱逃逸**：威胁模型假设攻击者无法破坏评估基础设施，但在真实部署中可能存在沙箱逃逸风险。
- **未讨论对抗性提示对RSA自身的影响**：虽然RSA面对自适应变异鲁棒，但若攻击者能干扰RSA的LLM组件（如通过注入），可能影响结果。
